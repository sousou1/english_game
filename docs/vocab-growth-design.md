## 1. 高校卒業語彙の目標値（逆算）

### 1.1 根拠 ― 学習指導要領（2017/2018 改訂・現行課程）

| 段階 | 新語数（現行課程） | 備考 |
|---|---|---|
| 小学校（外国語活動＋教科化） | **600〜700語** | 2020 全面実施。「聞く・話す」中心の受容語 |
| 中学校 | **1,600〜1,800語** | 2021 全面実施 |
| 高校 | **1,800〜2,500語** | 2022 年次進行。科目構成で上下 |
| **高校卒業 累計** | **約 4,000〜5,000語** | CEFR **B1〜B2** 目安 |

- **旧課程（2008/2009）との差**: 旧＝中学 1,200＋高校 1,800＝**約 3,000語**（CEFR A2〜B1）。
  現行課程は **+1,000〜2,000語**（小学校の前倒し＋中高の積み増し）で、到達目標が **B1→B2 寄り**に引き上げられた。
- CEFR 目安（受容語彙）: A2≈1,500 / B1≈2,500 / B2≈4,000〜5,000。**高校卒業＝B1 を確実に超え B2 に手が届く帯**。

### 1.2 本作の目標語彙数（推奨値）

> **推奨＝完走（コンプ）目標 5,000語**（現行課程の上振れ＝B2 まで担保）。**段階目標を 2 つ置く**（下記）。

| 到達点 | 累計語数 | CEFR 目安 | 本作での位置づけ |
|---|---|---|---|
| 本編クリア（30日キャンペーン終点） | **450**（ゲート）/ 供給累計 603 | A2〜B1 | **メイン（物語）の終点**。感情設計の背骨。経済の 15語/日×30日シム終点と一致（不変） |
| 段階目標①「中学卒業相当」 | **2,400** | A2〜B1 | 小700＋中1700。サブクエストで本編外を積む第一里程標 |
| 段階目標②「高校卒業 B1」 | **4,000** | B1〜B2 | 現行課程の下限。実用的な「完走」ライン |
| **コンプ目標「高校卒業 B2」** | **5,000** | B2 | 全語契約＝「原文修復率100%」（arc-plot §6 のコンプ要素と一致） |

**理由**: ①現行課程の上限（5,000）に合わせれば「高校卒業語彙が身につく」を堂々と言える。②本編 450〜603 語は
**A2〜B1 止まり**で、高校卒業には届かない（＝ここが今回の核心ギャップ）。③よって **5,000 は本編だけでは到達不能**。
**サブ（お使いクエスト）という任意供給トラックが残り約 4,400 語を運ぶ**設計にして初めて成立する（§4・§5）。

### 1.3 「身につく」までの所要時間（逆算・15語/日想定）

経済正典の **15語/日** を据え置いて累計語数を時間に換算（`tests/_sim_v6.mjs` と同じ前提）:

| 到達点 | 累計語 | 15語/日での所要 |
|---|---|---|
| 本編クリア | 450 | 約 **30 日**（＝経済シム終点。不変） |
| 中学卒業相当 | 2,400 | 約 **160 日**（5.3ヶ月） |
| 高校卒業 B1 | 4,000 | 約 **267 日**（8.9ヶ月） |
| 高校卒業 B2（コンプ） | 5,000 | 約 **333 日**（11.1ヶ月） |

> **設計上の含意**: 「30日キャンペーン＝物語の感情曲線」と「高校卒業＝約1年の学習旅程」は**時間スケールが一桁違う**。
> この二つを**同じトラックに乗せると破綻する**（物語を1年に薄く伸ばすか、語彙を30日に詰め込むかの二択になり、どちらも壊れる）。
> → **メインとサブの分離が必然**（§4）。物語は30日で完走でき、語彙は約1年かけてサブで積む。両者は独立に進む。

## 3. 語彙 DB 拡張計画

### 3.1 拡張方針（規約）

追加語は **arc-plot §11.7・economy-spec の不変条件を満たす**こと:

1. **実在英単語のみ**（架空語禁止）。`w` キーで一意。**既存782語・追加語間の重複ゼロを機械検証**。
2. **8系統（`f`）に配分**。既存の系統定義（§2.4 scenario-structure と1:1）を踏襲し、偏りを抑える（各系統が最終的に概ね均等＝5,000/8≒625語/系統を上限目安）。
3. **位階（`l`）L1-5 を付与**。語の親密度・CEFR 帯で割り当て（L1=最頻基礎〜L5=上級）。位階＝経済の rank 倍率に直結（§1.1 economy）。
4. **`ex`/`jx`（例文）を全語に整備**。クイズ・読み上げで使う。中学〜高校の語法に沿った平易な例文。
5. **FSRS 整合**: 追加で `srs.js` のカード生成パラメータは変えない（語固有パラメータを持たない設計を維持）。

### 3.2 段階計画（DB を一気に5,000にしない）

| 段階 | DB 到達 | 追加 | 主な位階帯 | 対応する学習到達 |
|---|---|---|---|---|
| 段階0（現状） | 782 | — | L1-5 既存 | 本編＋αを賄える |
| **段階1** | **2,400** | +約1,620 | L1-2 中心 | 中学卒業相当（小700＋中1700） |
| **段階2** | **5,000** | +約2,600 | L3-5 中心 | 高校卒業 B2（コンプ） |

- **段階1 を先に**整えれば、サブクエスト第1弾（中学帯）を連載開始でき、DB 全量を待たずに語彙増トラックを走らせられる。
- 拡張は**機械生成＋人手校閲のパイプライン**を想定（語選定＝コーパス頻度／CEFR リスト、`j`/`ex`/`jx`＝校閲必須）。
  実装フェーズで別途設計（本フェーズは方針確定まで）。

### 3.3 拡張時の検証（実装フェーズで回す回帰）

- 重複ゼロ（`w` 一意）・全語に `p/j/l/f/ex/jx` 充足・系統別/位階別の分布レポート・FSRS カード生成のスモーク。
- 供給を増やすたび `tests/_sim_v6.mjs` 系で**再シミュ**し詰みゼロを確認（§5）。

## 4. メイン / サブの分離方針（本書の核心）

### 4.1 二トラック構成

| | **メイン（物語）** | **サブ（お使いクエスト）** |
|---|---|---|
| 目的 | 物語ペーシング・感情設計（喪失と再生） | **語彙量を稼ぐ任意トラック** |
| 供給 | イベント限定 8語/本・全9章75本＝**累計603語**（不変） | イベント限定 8語/本・**約550本で残り約4,400語** |
| ゲート | 章ゲート＝定着語数 S≥3（15/40/…/450）。**80%ルールはメイン供給累計のみで判定** | **ゲートを持たない**。クリアしてもしなくてもメインは進む |
| 進行制約 | 章順・感情曲線に従う | **メインの節目間に自由に差し込み可。スキップしてもメインは詰まらない** |
| 解禁前提 | ch1（灯の力獲得）＝不変 | **灯の力獲得後（ch1完了後）のみ供給**＝維持設定どおり |
| 物語負荷 | 重い結末・伏線・キャラ声を担う | **軽い独立挿話**（街の住人の頼みごと等）。物語フラグ（yui/gaku/truth/route）に**触れない** |

### 4.2 分離が満たす不変条件

- **80%ルールはメイン供給累計だけで閉じる**（`次章ゲート ≤ 0.8 × メイン供給累計`）。サブはボーナス。
  → サブを一切やらなくても本編は arc-plot §4.1 のとおり完走できる（自己充足）。**サブは任意供給**である数学的保証。
- **章ゲートは settled(S≥3) の総数**で判定＝供給元（メイン/サブ）を問わない。サブで覚えた語も settled に加算され、
  **ゲートは早まりこそすれ遅れない**（§5 の再シムで確認）。＝サブは詰みを作らず、むしろ救済になりうる。
- **誤答は物語を変えない／説教禁止／1シーン≤8行／温度3択**＝サブにも全適用。
- **経済数値（gate段・W(k)・I4・再燃）不変**。サブ供給は「語数を増やす」だけで、経済のチューニングノブ（IDLE_K・注文成長率・銘減衰）に触れない。

### 4.3 データ / ID の分離規約（実装フェーズの指針）

- **語 DB は共通**（`data/words.js` 一本）。メイン/サブの区別は**語ではなくイベント側**に持つ。
- **イベント ID**: メイン＝`cNN_*`（既存規約）。サブ＝**`sub_*`**（例 `sub_errand_kitchen_01`）で名前空間を分離。
- **供給語の重複規約**: サブ供給語も `data/words.js` 実在・**メイン既出語/章内と重複ゼロ**を機械検証（v6 規約をサブにも適用）。
  メインが使う 603 語とサブが使う約 4,400 語は**重複しない**ようプール分割（メイン＝物語に必要な語、サブ＝残り）。
- **ゲート計算**: 80%ルールの「供給累計」は**メイン `cNN_*` の供給だけ**を数える（サブを除外）＝メイン自己充足の担保。
- **物語フラグ**: サブイベントは `yui`/`gaku`/`truth`/`route`/`ch` を**増減しない**（リンタで弾く）。
- **配置**: サブは「ch1完了後・任意の章節目間」に出現。章テーマ（系統）に緩く寄せた語パックをサブに割り当て可（例: ch2＝travel/food のサブ）。

## 5. 供給ペースの再設計 ＋ 再シミュレーション

### 5.1 採用案 ―（c）サブ（お使い）クエストで本編外の供給を上乗せ

指示書の候補(a)連載章増・(b)プレイ日数見直し・(c)サブ供給のうち、**(c) を主軸**に採用する。

- **(a) 章増は不可**: arc-plot は全9章で感情設計が閉じている。章を足すと物語が薄まる。
- **(b) 日数見直しは自然に内包**: 高校卒業（5,000語）は 15語/日で約333日。**プレイ日数の想定は「30日キャンペーン＋長期サブ」**と
  再定義する。経済の 15語/日 自体は不変（増やさない＝詰み・燃え尽き防止）。
- **(c) サブが本体**: 8語/本の v6 規約をそのままサブにも適用し、**約550本のお使いクエストを連載供給**。
  メインの節目間に差し込み、任意・スキップ可。これが残り約4,400語を運ぶ。

### 5.2 再シミュレーション（`tests/_sim_v6.mjs` を壊さず実施）

- **元の `tests/_sim_v6.mjs`（ch1ベースライン）は不変**のまま、その難易度エンジン（`srs`/`battle`/`pool`）を再利用する
  **新規シム `tests/_sim_v6_subquest.mjs`** を追加し、サブ供給を上乗せしても本編が壊れないことを数値確認した。
- モデルは `_sim_v6` と同一（1日30分=1260タップ・焚き火全消化・正答85%・武器/ラッシュ/タップマイルストーン抜きの保守側）。
  サブは**灯の力獲得後（day4〜）**に 1日 +8語 / +16語 を上乗せする3シナリオを比較。

実行結果（`node tests/_sim_v6_subquest.mjs`）:

| シナリオ | c02ゲート(settled40)到達 | day14 供給累計 | day14 settled |
|---|---|---|---|
| ベースライン（サブなし＝_sim_v6相当） | **day 6** | 51 | 51 |
| サブ +8語/日 | **day 6** | 139 | 115 |
| サブ +16語/日 | **day 6** | 227 | 179 |

**確認できたこと**:
1. **ゲート到達日は遅れない**（サブ有無で day6 固定／早まりこそすれ）。＝**任意供給で詰みは構造的に発生しない**。
2. **80%ルール（gate40 ≤ 0.8×供給累計）はサブ供給で供給累計が増えるほど強く満たされる**（破れない）。
3. **settled は供給に比例して単調増加**＝サブが語彙量エンジンとして機能。本編経済の段（gate値・到達日）は不変のまま。

> 注: 経済の A/I 比番人（`economy-spec.md` §5 の就寝放置≈アクティブ比）は別シム（Python）が正典。
> サブは「語数を増やす」だけでチューニングノブ（IDLE_K・注文成長率・銘減衰）に触れないため、A/I 番人の帯域設計を変えない。
> 各章/サブ弾の執筆時に供給語を確定したら、`_sim_v6` 系の詰みゼロ確認をその都度回す（規約）。

## 6. まとめ（フェーズ1の結論）

1. **目標語彙数**: コンプ目標 **5,000語（高校卒業 B2）**、段階目標 2,400（中学卒業相当）/ 4,000（高校卒業 B1）。本編クリアは A2〜B1（450〜603語）。
2. **ギャップ**: DB を **782 → 5,000（+約4,218語）** に拡張必須。本編供給 603 は不変、残り約4,400語をサブが運ぶ。
3. **DB拡張**: 実在語追加＋8系統/位階付与＋ex/jx整備。段階1（→2,400）→段階2（→5,000）の二段。FSRS整合は語追加だけで自動。
4. **供給ペース**: サブ（お使い）約550本＠8語/本を連載。**メイン経済（15語/日×30日・gate段）は不変**。再シムで詰みゼロ・80%ルール堅持を確認。
5. **メイン/サブ分離**: 80%ルールはメイン供給のみで判定（メイン自己充足）、サブはゲート非依存・任意・物語フラグ非干渉。データは語DB共通／イベントID（`cNN_*`/`sub_*`）で分離。
