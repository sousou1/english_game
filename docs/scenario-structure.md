## 2. 世界観

### 2.1 基本設定
大陸グランディアでは、魔法とは「異界の言葉」を声にすることである。数百年前、異界から「渡り人」が現れ、この世界が異界の言葉で"書かれている"ことを伝えた。「shine」と唱えれば光が応え、「open」と唱えれば扉が応える――**世界は異界の言葉だけを聞く**。共通語(=日本語表記)は人間同士の言葉で、世界には届かない。

### 2.2 異界の言葉の規則(5原則)
1. **呪文に文法はない**。単語ひとつ=呪文ひとつ。語彙の広さ=呪文書の厚さ。
2. **魔力の本体は「意味の想起」**。音を出しても意味を思い出せなければ空気の振動(=灯し場のマッチングそのもの)。
3. **言葉は深く根づくほど強く応える**(=FSRSのStability。職位=言葉との絆)。
4. **奪った言葉は必ず腐る**(案3移植)。他人から奪う・一夜で詰め込んだ言葉は発音から腐り落ちる。丸暗記が消える学習科学の世界法則化。**作中で講釈は一切しない**。敵(盗賊・教団)の詠唱が「腐っている」描写でのみ見せる。
5. **誤答の世界観処理**: 綴り違いの詠唱は暴発しない。「世界に聞こえなかった」だけ。**「言葉は胸に戻った。もう一度呼べばいい」**(lapseの無罰叙述、固定文言)。

### 2.3 社会と対立軸
- 言葉は王国の財産。平民が上位語(位階3以上=L3以上)を覚えることは違法(**言葉狩り法**)。王家と学院が「免状」で高位語を独占。
- 冒険者ギルドだけが例外特区。**「確かに思い出せた数」だけが階級になる**実力主義(F→S級)——主人公が成り上がれる理由であり、章ゲート=昇格試験の物語的正当化。
- 敵: **「沈黙(サイレンス)」**――言葉が力を失い人が声を失う現象。元凶は**無言王**。言葉が嘘と戦争と階級しか生まないと結論した最初の渡り人であり、すべての言葉を異界へ"回収"しようとしている。

### 2.5 位階と原文
- 真語は位階一〜五(L1〜L5)。位階=原文の「深さの層」。上位語を根づかせるほど深い章が開く。

## 7. システム統合(既存数値は無改造、すべて「皮」)

### 7.1 大原則
- **一文ルール**「誤答は物語を変えない。選択だけが物語を変える」。
- 誤答=詠唱不発。「言葉は胸に戻った。もう一度呼べばいい」。罰はI4どおり**コンボ切れ+タイル0.4秒ロックのみ**。灯火は1も減らない。
- 重い結末(バルドの声喪失等)は物語選択(truth等)にのみ連動。クイズ成績には一切連動しない。
- 用語は全続投: 言霊/灯火/靄/うとうと/起こす/招く/鐘。**コードの概念名は変えない**(灯火→マナのリスキンは廃止。ランタンの世界に「灯火」はそのまま似合う)。

### 7.2 灯火(回答ポイント)の使途
1. **シナリオ進行**: クエストノード(シーン3〜5束)の解錠に灯火を支払う(受注料・旅費)。**シーン内の読み進めは無料**。解錠価格は `0.6 × W(k_now)`(現在の注文インデックスのW(k)に動的連動=常にその日の1〜2セッション分に正規化)。初回クエストは50%割引。
2. **武器=触媒の購入**(7.4)。
3. **既存の施設・棚・印刷機**: 現行経済をそのまま継続。
- ボス挑戦は**無料**(削り保存・再挑戦自由)。

### 7.3 章ゲート=ギルド昇格試験
扉=「定着語数(S≥3=青銅以上)」建て(§4の表)+灯火少額。物語上はF→E→…→S級の昇格試験としてバルドが課す。確かな想起(due後正答×postLapse==0)以外では進まない既存原則を維持。**復習だけでもゲートは越えられる(新語追加の強制なし)=詰みが構造的に発生しない**。

### 7.4 武器=触媒(声を世界に伝える媒介)
| 表示 | 実体(既存数値のリネーム) |
|---|---|
| 攻(杖・剣) | タップ価値倍率 |
| 連(指輪) | コンボ猶予・維持 |
| 速(外套) | フィーバーゲージ効率 |
| 識(系統特化、例: 精霊の杖=nature語×2) | 分野別倍率→「招く」の方向付けと連動 |
| 貯(灯火庫・インク壺) | 棚上限・「招く」コスト割引 |
- ※案1の「結界(誤タップロック短縮)」は**不採用**(ロックの存在理由を装備で買い崩す設計穴)。
- 武器はストック=売却も破壊もなし。**装備は倍率であって代替ではない**(覚えていない語は何を装備しても撃てない)。
- 「宿=招く課金化」は不採用(案3の逆インセンティブ指摘)。招くは現行どおり無料のプル型。

### 7.5 ステータス(捏造数値ゼロ。全部が実在指標の正直な別名)
| 表示 | 実体 |
|---|---|
| LV | 習得語数 |
| 詠唱(ATK) | V_ref(直近100正解タップの移動平均) |
| 深さ(INT) | 平均Stability(会心率表示に変換) |
| 速詠(AGI) | コンボ・フィーバー統計 |
| 魔力(残灯) | CpS(放置生産) |
| ギルドランクF→S | 確かな想起累計+定着語数 |
| 系統Lv×8 | 分野別習得数。travel=地図スキップ解放/business=店割引/feelings=恋愛の"もう一歩踏み込む選択肢"解錠(**ロック時も回想から再挑戦可**=「英単語が足りなくて恋を逃した」を恒久化させない) |

### 7.6 ボス戦=詠唱バトル(大注文の皮)
- ボスHP=仕事量W(k)。お題(日本語=放ちたい技の意味)→正しいタイルをタップ=詠唱ヒット、獲得灯火がそのままダメージ(獲得と納品が同時=既存構造)。
- 出題デッキ=**期日到来カード優先+習得プール混合**。「ボスの弱点は、お前が忘れかけた言葉だ」(バルドの台詞で一度だけ提示)。FSRS採点は**通常レビュー1回分のみ計上**(故意誤答ファーミング・二重加点を塞ぐ)。
- **ボスの反撃は演出のみ**(画面が揺れる・タイルに靄がかかる)。プレイヤーHPなし・敗北状態なし(案2のHP/気力制は**廃止**、I4準拠)。削りは保存され、放置中も仲間(CpS)が削っておいてくれる。再挑戦は常に無料。緊張感はクリアスタイル称号(S/A/B、見た目のみ)で担保。
- コンボ25連でフィーバー=「限界詠唱」、必殺技名カットイン。弱点系統(章の推奨分野×2)が「今週はnatureを招こう」という学習導線。

### 7.7 学習ステップの物語化(v2用語完全続投)
- **招く**=新しい呪文の写本に契約のインクで署名する。3分後ステップ=インクが乾くのを待つ。
- **鐘**=ギルドの依頼更新と言霊の帰還時刻(朝昼夜)。ノノの手紙も鐘で届く。
- **起こす**=眠った言霊の研ぎ直し。再燃バースト(R=0.85ピーク×3.50)=**オーバーチャージ詠唱**。「忘れかけた言葉ほどよく燃える」――少年漫画の覚醒の文法と忘却曲線の数式の一致(本企画の心臓)。
- **職位**(錆鉄=見習い→星鋼=語り部)=言霊との絆。**黄金(S≥30)以上の語だけ**セリフ・ログ内に英語表記で現れる(v2決定維持。案2の「全習得語が光る+◆◆◆全文タグ付け」は実装コスト過大につき章扉8行限定に縮小)。
- 放置収益=契約済みの言霊たちが留守中もギルドの巡回依頼をこなして稼ぐ(過去の想起の利息)。

## 8. 文体規則

1. **1シーン=2〜8行+選択肢**。8行上限は絶対。強制スクロール演出は禁止。「続きを読む」はプレイヤー起点タップ。全文ログ残留。
2. **チュートリアル文ゼロ**。システムは世界の住人の台詞と行動で教える(鐘=生活、再燃=バルドの一言、等。各装置の言語化は作中一回まで)。
3. **説教禁止**。学習価値の講釈は書かない。法則(奪った言葉は腐る等)は敵の描写と悲劇でのみ見せる。
4. **裏テーマ(沈黙=声に出せない恐怖)は絶対に言語化しない**。
5. 選択肢は常に2〜3択の「温度違い」(ニヤつく/茶化す/黙る)。どれを選んでも直後の同一シーンに合流、差分はセリフ1〜3行。クイズの選択と物語の選択は別物であることを開幕10行で体感させる。
6. **恋愛はPG(手を繋ぐまで)**。直球の告白文を避け、軽口・手紙・沈黙で描く。デート専用回を作らない(共闘と事件の中で進める)。テスト基準=「ニヤッとしたか/スキップしたか」の観察。
7. 闇は残虐でなく様式美(奪う・統べる)。仲間を傷つける選択肢は置かない。闇ルートでも学習行動は完全同一。
8. **物語で詠唱される語は必ず782語データ(`data/words.js`)から選ぶ**。執筆時に `w` キーで実在検証必須(開幕`fire`事故の再発防止)。固有名詞(黙狼・黙字卿等)は対象外。
9. キャラ声の固定: ノノ=短文即レス/セシリア=棘敬語+ですわ/ヴェル=気だるい皮肉・嘘をつかない・きみ呼び/ガイ=熱血短文/バルド=兄貴口調/無言王=叫ばない。
10. 固有名詞(スペルライト/黙字/言葉狩り法等)は全て`terms.json`にdata化し、15歳テストで「ダサい」と言われたら即改名できる構造にする。

## 10. シナリオデータ実装規約(JSON)

### 10.1 ファイル配置(現行story-lines.jsと完全分離)
```
data/scenario/
  chapters/ch1.json … ch9.json   # 章ごと1ファイル(終章=ch9)
  endings.json                    # ED5本
  letters.json                    # ノノの手紙(鐘配信)
  terms.json                      # 固有名詞辞書(改名一括反映用)
  flags.schema.json               # フラグ定義(実装初日に確定、以後変更しない)
```

### 10.2 ID規約
- シーンID: `c{章2桁}_{連番3桁}`。連番は**10刻み**(挿入余地)。例: `c01_040`
- ルート/差分変種: サフィックス `_h/_n/_c/_d`(hero/nono/ceci/dark)。例: `c08_120_d`。行スワップは変種ファイルを作らず `lines` 内の条件行で処理(下記`when`)
- クエストノード: `q{章}_{連番}`。例: `q1_3`
- ボス: `boss_{slug}`。例: `boss_mokuro`, `boss_silent_letter`
- エンディング: `end_hero / end_nono / end_ceci / end_dark / end_friend`
- 選択肢: シーン内ローカル `a/b/c`

### 10.3 フラグ規約
- 名前は小文字snake、§5.1の6個以外**追加禁止**: `nono, ceci, dark, truth, route, ch`
- 効果記法: `"effects": {"nono": 1}`(加算のみ。**負の値は禁止**=リンタで弾く)
- 条件記法: `"when": "nono>=4"`, `"when": "route==dark"`, `"when": "truth"`

### 10.4 シーンスキーマ(例: c01_040)
```json
{
  "id": "c01_040",
  "quest": "q1_1",
  "lines": [
    {"t": "ランタンが訊いている。お題は『輝く』。――灯せ。"},
    {"t": "ヴェル「……へえ。読めるんだ、きみ」", "when": "ch>=4", "swap": true}
  ],
  "battle": {
    "boss": null,
    "scripted": [{"j": "輝く", "w": "shine", "tiles": ["shine","apple","sleep","door","water","tree"]}],
    "deck": null, "theme": null, "lose": false
  },
  "choices": [
    {"k": "a", "t": "……かもな(ニヤつく)", "effects": {"nono": 1}, "goto": "c01_050"},
    {"k": "b", "t": "無我夢中だった", "goto": "c01_050"},
    {"k": "c", "t": "黙ってランタンを見る", "goto": "c01_050"}
  ],
  "next": "c01_050"
}
```
- `battle.scripted`: 演出固定問(`w`は**必ず`data/words.js`に実在**。ビルド時に存在チェックを走らせる=文体規則8の機械化)
- `battle.deck: "due_first"`: 期日優先+習得プール混合の通常ボスデッキ
- `choices`が無いシーンは`next`のみ。`goto`は必ず同章内

### 10.5 コスト・ゲート規約
```json
{
  "id": "q1_3",
  "unlock": {"wMult": 0.6, "firstTimeDiscount": 0.5},
  "gate": null
}
{
  "id": "ch1_gate",
  "gate": {"settled": 15, "tomoshibiWMult": 0.3},
  "narrative": "旅支度――言霊15体を青銅に"
}
```
- `wMult`: 解錠価格=`wMult × W(k_now)`(解錠時の現在注文インデックスで動的評価。常に「その日の1〜2セッション分」に正規化)。標準値: 通常ノード0.3〜0.6、章最終ノード0.6〜1.0。シーン単位の課金は禁止(ノード=シーン3〜5束のみ)
- `gate.settled`: 定着語数(S≥3)。値は§4のペーシング表からのみ取る(15/40/90/150/220/290/360/420/450)
