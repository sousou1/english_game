# v5 イベントモード設計 — 物語×単語×イラストの三位一体

## ねらい
ストーリーが「飛ばされる」のは、読むことに報酬がないから。v5では**物語の山場=イベント**とし、
イベントは「そのシーンに沿った穴埋め詠唱」でしか進めない。埋めた単語は新規習得語として
SRS(学習ステップ)に加わり、シーンのイラストがアルバムに残る。
文脈+物語+画像の三重符号化(dual coding)で記憶に残す。

## イベントの構造(data/events.js)
```js
{ id:'ev_c01_fire', chapter:1, title:'燃える夜', art:'ev_fire', tier:1,
  gate:{ read:'c01_050' },          // このシーンを読むと解放
  intro:['...'],                     // 導入(2-4行)
  beats:[
    { lines:['火が回る。井戸はあそこだ!'],
      teach:'water',                 // 新出語: 意味カード提示→
      cast:{ jp:'水よ、来い——', answer:'water' } },  // 穴埋め詠唱(4択)
    { lines:['倒れた戸が道をふさぐ'], review:true,
      castLine:'おぼえた言葉で、戸を破る!' },        // 既習語の反芻(実行時抽選)
  ],
  outro:['...'],
  reward:{ gold:120, drop:'mid' } }  // 初回クリア時のみ
```

## ルール(学習整合)
- **新出語(teach)**: イベント内で意味提示→文脈詠唱。クリア時に `p.steps[w] = startSteps(now)`
  (=招くと同じ)。以後は焚き火の修行で定着させる。日次上限なし。
- **反芻(review)**: 既習語から実行時に抽選(期日近い語を優先)。**SRSには書き込まない**
  (B7: SRS書き込みは焚き火だけ。イベントは「運用」)。誤答ペナルティなし(やり直しのみ)。
- 既にカード/ステップにある teach 語は再追加しない(反芻として機能する)。
- 選択肢は quiz.js の pickDistractors を再利用(意味重複なし)。

## 難易度の階段(tier) — 「物語が進むほど難しいことができる」
- tier1: 日本語文脈→英単語4択(第1章前半)
- tier2: 6択+教えカードは冒頭のみ(終盤の詠唱は思い出して撃つ)(第1章後半)
- tier3(将来): 2語連続詠唱(動詞+名詞)
- tier4(将来・上級職解禁): **文法詠唱** — 語順並べ替え(I open the door)。上級職=文法学習。

## UI
- メイン画面: イベント解放中は #eventBanner(脈動するグラデ枠+アート縮小版+タイトル)を
  ステージ下に大きく表示。メニュー「物語」にもバッジ。解放時はticker+SE。
- イベント本体: 焚き火と同じ全画面オーバーレイ(プレイヤー起点・タイマーなし・全没収なし)。
  イラスト→行送り→詠唱、の繰り返し。クリアでアートがアルバムへ。
- アルバム: 物語シートのハブにクリア済みイベントのアート格子。タップで再生(報酬なし)。

## シナリオ改稿(第1章)
方向: **夜祭の夜、「静寂の火」が村を焼く。言葉を食う火。** レンとノノは老詠唱士の写本を
持って王都へ逃げ、ゼロから成り上がる。中学生向け語彙・ルビ維持。
- 野心の分岐(フラグ加算のみ): amb_home(居場所を取り戻す)/ amb_revenge(火を放った者を追う)
  / amb_gold(大金で村を建て直す)。第1章では色付け、章が進むと分岐拡大。
- 互換ID維持: c01_010〜c01_180(+c02_000ゲート settled:40)。ノノ合流確定は c01_160
  (王都の宿で働き始める→「さしいれ」が文通メカニクスの新しい皮)。
- 誤答は物語を変えない。選択だけが物語を変える。

## 画像
- 既存22素材: ベスト1/3を選定→ffmpeg(libwebp)で webp 化して assets/img/ へ。
  敵=256px / シーン・イベント=720px / タイトル=1080px。**SWのCOREには入れない**
  (既存fetchハンドラのstale-while-revalidateで実行時キャッシュ)。常にemoji/無画像フォールバック。
- ドット絵検討(訴訟リスク低減): ①後処理(area縮小+palette 32色+neighbor 4x拡大)
  ②プロンプト生成(pixel art指定)の両方を試作し、ユーザーに見せて選ぶ。
- 新規大量生成: イベント絵・新シーン絵・敵リデザイン(ドット)・表情差分(レン/ノノ×4)・
  野心分岐絵・ロゴ再挑戦。各3-5候補、夜間バッチ。

## 数値
- イベント報酬gold: 直近の敵HP相当(enemyHp(kills)×1.5)+ 確定ドロップ(mid卓)。
  恒久乗数には一切触れない(経済不変条件の維持)。
- 新出語5語/イベント。第1章で6イベント=30語(語彙782語の在庫で第9章まで余裕)。
