## フェーズ1: 語彙ターゲットの逆算と拡張設計（先に止める）
1. **高校卒業語彙の目標値を逆算**: 学習指導要領（小600-700＋中1600-1800＋高1800-2500 ⇒ 高校卒業で概ね**4000〜5000語**、
   CEFR B1-B2目安）から、**本作が完走で身につけさせる目標語彙数**を根拠つきで定める（旧課程3000語との差も明記）。
2. **現状とのギャップ評価**: 現 `data/words.js`=782語、本編ゲート450語（30日シム終点）。目標（例5000語）との差を出し、
   **語彙DBの拡張が必要か**を判定。必要なら **実在語の追加・`f`属性（8系統）/位階(L1-5)付与・FSRS整合** の拡張方針と段階計画。
3. **供給ペースの再設計**: 8語/本・80%ルール・15語/日×30日シムを壊さずに目標へ近づける案を出す。
   候補＝(a) 連載章数を増やす (b) 1日の学習語数/プレイ日数の想定見直し (c) **サブ（お使い）クエストで本編外の供給を上乗せ**。
   いずれも `tests/_sim_v6.mjs` で**再シミュ**し、詰み（ゲート未到達）が出ないことを数値で確認。
4. **メイン/サブの分離方針**: メイン＝物語ペーシング（章ゲート・感情設計）。サブ＝お使いクエストで**語彙量を稼ぐ**独立トラック
   （メインの節目間に差し込み可・スキップしてもメインは詰まらない＝任意供給）。データ/IDの分離規約を定める。
   → **フェーズ1成果＝設計ドキュメント**（`docs/vocab-growth-design.md` 等）。**ここで停止して報告**（方針の承認待ち）。

## 制約（死守）
- 詠唱/供給語は `data/words.js` 実在・重複衝突ゼロ（DB拡張する場合も追加語は実在英単語・`w/f` 整合）。
- 8語/本・**次章ゲート ≤ 供給累計8割**・settledゲート・**誤答は物語を変えない**・説教禁止・1シーン≤8行・温度3択。
- 経済数値（gate段・W(k)・I4・再燃）は不変。重い結末は物語フラグのみ連動。**ch1 は変更しない**。
- 供給を増やしたら必ず `tests/_sim_v6.mjs` 系で再シミュし詰みゼロを確認。
