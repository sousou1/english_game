# Clicker Heroes / Tap Titans 2 — 成長曲線の実測分解

■ Clicker Heroes(CH)/ Tap Titans 2(TT2)成長曲線の分解(全て実測値・wiki/ゲームファイル由来)

【1. レベルコスト曲線(支出側)】
- CH ヒーローのレベルアップコスト: cost(L) = floor(BaseCost × 1.07^(L−1))。全ヒーロー共通の×1.07。
  - 1.07^10 ≈ 1.967 → 「10レベルごとにコスト約2倍」
  - 1.07^25 ≈ 5.427 → 「25レベルごとにコスト約5.4倍」
  - 例: Treebeast(BaseCost 50)のLv114→115は floor(50×1.07^114)=111,871ゴールド
- Cid(クリック専用ヒーロー)のみ cost = floor((5+L)×1.07^(L−1))(Lv16以降は20×1.07^(L−1))
- 後期追加のAce Scout級は×1.22/レベル(5レベルで2倍超)という超急勾配+巨大マイルストーン(25レベルごとに×1000)の組み合わせ。
- Tap Titansシリーズ: cost = BaseCost × 1.075^Level(進化後はBaseCost×10×1.075^Level)。ゲームファイル実測。1.075^10≈2.06でCHとほぼ同じ「10レベルで2倍」。
- 新ヒーロー(=新施設)の基礎コスト階段: CH実測で 5 → 50 → 250 → 1,000 → 4,000 → 20,000 → 1e5 → 4e5 → 2.5e6 …(隣接比×4〜6.25)。同時に「コスト/基礎DPS比」が悪化: 10:1 → 11.4 → 13.5 → 16.3 → 20.5 → 26.8 → 36.8 → 53 → …終盤Frostleafは28,113:1。新施設ほど割高だが絶対量で圧倒する設計。

【2. マイルストーン(25/50/100…)の実数値】
- CH アップグレード解禁: 各ヒーローLv10/25/50/75/100(一部125/150)。自己DPS系は合計+1900%(=×20)が標準パッケージで、「Lv1〜200の谷」を埋める設計と明言されている。
- CH レベルボーナス: Lv200から25レベルごとに自己DPS×4。Lv1000〜8000は1000の倍数で×10(それ以外は×4)。Frostleaf以降のヒーローはLv525〜725の区間だけ×5。後期ヒーロー(Xavira以降)は×4.5で×10なし。
- 黄金比: 25レベルあたり ダメージ×4 vs コスト×5.427 → 効率は0.737倍/25レベル(≈0.988^L)。つまりマイルストーン倍率をコスト成長より「わずかに低く」設定し、どのヒーローも最終的に引退して次へバトンタッチする。
- TT2: Sword Master(タップ)もヒーローも「Lv10から20レベルごと」にマイルストーン倍率。高レベルでは間隔が30、40…と開く。SMの上限はLv12,500で×1000ボーナス。ヒーロー武器は1本ごとに×1.5、全37種コンプで追加×5。

【3. 敵HP・ゴールド曲線とボス壁のペース】
- CH モンスターHP: ゾーン1〜140: HP = ceil(10×(z−1+1.55^(z−1)))、ボスは×10。141〜500: 139+1.55^139×1.145^(z−140)ベース、つまり×1.145/ゾーン。以降500ゾーンごとに+0.001(1.146, 1.147…上限1.545)。
- ボス壁のペース: 5ゾーンごとにボス(HP×10、30秒制限、撃破は1体でゾーンクリア)。ゾーン1〜140はボスごとに要求DPS約×10(1.55^5≈8.9)、141以降は×2(1.145^5≈1.97)に意図的に緩和。100ゾーンで×1.145^100≈7.6e5。
- 進捗ゲージ: 通常ゾーンは10体撃破で次解禁(Kumawakamaruで削減可)。最速(即死)状態で100ゾーン≈47秒。
- ゴールド: モンスターの基本価値 = HP/15。ゾーン75以降は×min(3, 1.025^(z−75))ボーナス(z≈120で上限×3)。宝箱は1%出現で×10ゴールド(Doraで出現率最大100倍、Mimzeeで+50%/Lv)。TT2のChestersonも同じく×10ゴールド、マルチスポーンは+2〜4倍。
- 500ゾーンルール(後期テンポ調整): 500ゾーンごとに +0.1体/ゾーン、primal確率−2%(下限5%)、ボスHP+0.4体分、ボスタイマー−2秒(下限2秒)。
- TT1(シリーズ原型)HP式: MaxHP = 18.5 × 1.57^min(stage,150) × 1.17^max(stage−150,0)。ボスHP = MaxHP×[2,4,6,7,10]((stage−1) mod 5で巡回)→ 5ステージ周期の中で壁が2→4→6→7→10と漸増する「ミニ壁リズム」。

【4. タップダメージとDPSの比率推移】
- CH: クリックダメージは序盤Cidの線形成長(クリック強化合計+20900%=×210)→ 中盤以降は「総DPSの0.5%をクリックに加算」するアップグレードが7個、合計でクリック=総DPSの3.5%。
- 比率の意味: 秒10クリック(Clickstorm=10クリック/秒×30秒、CD10分)で総DPSの35%追加。つまり「タップは放置生産の3〜4割のブースト」が基準線。スキル(Powersurge +100%DPS/30秒、Super Clicks +200%クリック/30秒、Lucky Strikes クリティカル+50%/30秒、Golden Clicks クリックごとに敵ゴールド1%)とJuggernautコンボ(+0.01%DPS/クリック累積)を重ねたときだけアクティブが数倍〜10倍に化ける。
- TT1実測の経験則: 「総DPS ≈ タップダメージ×10 + ヒーローDPS」。プレイヤーLv400〜500以降はタップ強化よりヒーロー(放置)へ投資した方がゴールド効率が良くなる、という移行点が明示されている。
- TT2: タップ→ペット→ヒーローへ主役が移る。ペットは「20タップごとに1回攻撃」(タップ依存)→ ペット総レベル200で自動攻撃化(3秒ごと)= タップからの卒業が「資産量」で解禁される。

【5. アクティブ vs 放置のバランス(実数値)】
- CH 放置ボーナス: Siyalatas「+25%アイドルDPS/Lv(10レベルごとに−1%、下限+15%)」、Libertas「+25%アイドルゴールド/Lv(同減衰)」。発動条件は「60秒間クリックなし」。コストは線形(n)で、序盤から雪だるま式に積める。
- CH アクティブ: Fragsworth「+20%クリックダメージ/Lv」、Juggernaut「+0.01%DPS/クリックコンボ」(コストn^1.5)、Bhaal「+15%クリティカル/Lv」。クリティカルは基礎0%、アップグレードで+3%×3=9%。
- 公式Tier表では「Idleビルド: Siyalatas/Libertas/Nogardnitが最上位、クリック系は最下位」「Activeビルド: Fragsworth/Juggernaut/Bhaalが最上位」と完全に二極化し、ハイブリッドが両取り。
- プレステージ通貨: Hero Souls = floor(((zone−80)/25)^1.3)/primalボス(zone100以降25%出現、Atmanで+75%まで)。未使用HSは1個+10%DPS。初回アセンション推奨はzone130(HS≈2〜5個)。zone300で≈17個、zone1000で≈109個と緩い多項式成長(これがメタ進行の通貨)。

【6. コトダマ工房への写像】
- 「お題プール連打」=CHのクリック。タップ価値は独立変数にせず「総生産(灯火/秒)の3.5%×(語彙係数)」のように放置生産に従属させるのがCHの結論。語彙数×職位は CHの「Static×Level」(線形)+「25ごと×4」(職位=マイルストーン)にそのまま対応させられる: 覚えた語数が線形軸、職位昇進(25語/50語/100語…)が×4級の乗数軸。
- お題側(進捗ゲージ)はHP=1.55^n(序盤約10〜30お題)→1.145^n巡航、5お題ごとに「ボスお題」(HP×10、ご褒美つき)が実証済みリズム。

## 推奨パラメータ

- **施設(職人)レベルコスト成長率**: cost(L) = BaseCost × 1.07^(L−1)。10レベルで約2倍(1.967)、25レベルで約5.4倍
  - 根拠: Clicker Heroes全ヒーロー共通の実測式(wiki Formulas/Heroes。例: Treebeast Lv114→115 = floor(50×1.07^114) = 111,871)。Tap Titansも1.075でほぼ同値
- **職位マイルストーン(倍率)**: レベル(語彙)25ごとに×4。序盤(Lv10/25/50/75/100)は×2級アップグレードを散らし合計×20。節目(1000語ごと等)は×10
  - 根拠: CH実測: Lv200+で25レベルごと×4、Lv1000〜8000は×10、自己DPSアップグレード合計+1900%。TT2はLv10から20ごと(間隔漸増30,40…、上限Lv12,500で×1000)
- **マイルストーン倍率とコストの黄金比**: 倍率(×4) < 同区間のコスト成長(×5.427)。効率が0.737倍/25レベル(≈0.988^L)で単一施設が自然に引退する
  - 根拠: CH wiki Formulasの DPS/Cost = Static×0.988^L 導出。1施設を掘り続ける戦略を数式で殺し「次の施設」へ誘導する核心設計
- **次施設の基礎コストと効率**: 基礎コスト=前施設の×4〜6。コスト/生産効率は1段ごとに×1.15〜1.45悪化(10:1→11.4→13.5→…→53:1、終盤28,113:1)
  - 根拠: CHヒーロー表実測(50→250→1000→4000→20000→1e5→4e5→2.5e6)。新施設は割高でも絶対量で旧施設を瞬殺する感覚を作る
- **お題(敵)HP曲線**: 序盤〜140: HP = ceil(10×(n−1+1.55^(n−1)))。以降: ×1.145/段(超長期は500段ごとに+0.001、上限1.545)
  - 根拠: CH Zones/Formulas実測。序盤は×1.55で「最初の壁→プレステージ誘導」、巡航は×1.145(5段で約2倍)
- **ボス壁のペース**: 5段ごとにボス(HP×10、制限30秒、撃破1体でクリア)。要求DPSは序盤ボスごとに×10、中盤以降×2(1.145^5≈1.97)
  - 根拠: CH Zones実測。×10壁は140まで(プレステージ前提のペース)、以降×2に緩和された。常時×10は詰むことを開発が認めた形
- **進捗ゲージ(ゾーンクリア条件)**: 通常10お題撃破で次解禁、ボスお題は1体。短縮手段(Kumawakamaru相当)を後から販売
  - 根拠: CH Zones実測(10 kills/zone、即死状態で100ゾーン47秒)。テンポ調整資源として機能
- **灯火(ゴールド)獲得量**: 1お題の報酬 = そのお題のHP/15。中盤から×1.025^(段−75)ボーナス(上限×3)。宝箱お題1%出現で×10
  - 根拠: CH Formulas/Zones実測(Monster Gold Worth = HP/15、Min(3,1.025^(level−75))、Treasure Chest 1%・×10)。TT2 Chestersonも×10
- **タップ価値とDPS(放置生産)の比率**: 1タップ = 総生産/秒の3.5%(0.5%×7段階で解禁)。秒10タップで放置の+35%。スキル重ねがけ時のみ2〜10倍
  - 根拠: CH実測: クリックダメージ=総DPSの0.5%アップグレード×7=3.5%。序盤専用の線形クリック(Cid、×210)は中盤で死ぬため、%連動が必須
- **語彙=タップ価値のエンジン化(コトダマ工房適用)**: タップ価値 = (総生産の3.5%基準) × 語彙数(線形) × 職位倍率(25語ごと×4)。語彙獲得自体を「1.07^Lコストのかからない無料レベル」として扱う
  - 根拠: CHの DPS = Base×Level(線形)×4^(milestones)構造に語彙(Level)と職位(milestone)を写像。学習がコスト無しの成長軸になりインフレを牽引できる
- **アクティブスキルの基準値**: 自動タップ10回/秒×30秒(CD10分)、生産+100%/30秒、タップ+200%/30秒、タップごとに敵報酬1%獲得/30秒(CD1時間)
  - 根拠: CH Skills実測(Clickstorm/Powersurge/Super Clicks/Golden Clicks)。30秒バースト+分単位CDが連打ゲームの覚醒リズム
- **放置ボーナスの成長率**: 放置生産+25%/Lv(10Lvごとに−1%減衰、下限+15%)、コストは線形。発動条件「60秒無タップ」。タップ資産が閾値(例: 合計200Lv)に達したら3秒ごと自動タップ解禁
  - 根拠: CH Siyalatas/Libertas実測(+25%→+15%、incremental cost=n)とTT2ペット実測(20タップごと1回→総レベル200で自動化)
- **プレステージ通貨(将来枠)**: 獲得 = floor(((到達段−80)/25)^1.3)、1個あたり恒常+10%生産。primal(通貨ドロップ)ボスは段100以降25%出現。初回リセット目安は段130
  - 根拠: CH Formulas/Primal Bosses実測。緩い多項式(z130≈2個、z300≈17個、z1000≈109個)が周回ごとの伸びを保証しつつ暴走しない
- **ボス壁ミニリズム(代替案)**: 5段周期内でボス倍率を[2,4,6,7,10]と巡回させる(TT1方式)。HP成長は序盤×1.57/段→後半×1.17/段
  - 根拠: Tap Titansゲームファイル実測(MaxHP=18.5×1.57^min(stage,150)×1.17^max(stage−150,0)、BossHP=MaxHP×[2,4,6,7,10])。一定×10より起伏のある壁を作れる

## やってはいけない失敗

1. 線形タップの罠(Cid問題): CHのCidはクリック強化合計×210あっても、敵HPが×1.55^nで伸びるため数十ゾーンで無意味化した。開発は後から「クリック=総DPSの0.5%×7」を追加して救済。教訓: タップ価値は最初から放置生産に%連動させる。コトダマ工房なら「語彙×職位」を独立したダメージ表ではなく総生産への乗数にする。

2. 一定×10壁の罠: CHのゾーン1〜140はボスごとに要求DPS×10で、これはプレステージで戻る前提のペース。141以降は×2に緩和されている。最初から終わりまで×10壁を貼ると、プレステージ未実装のコトダマ工房(将来枠)では確実に詰む。巡航は「5段で×2」(=×1.145/段)を上限にすべき。

3. 単一施設深掘りの詰み: コスト×1.07^Lに対しマイルストーン×4(25Lvごと)では効率0.988^Lで必ず腐る(CHではLv199時点で効率がLv1の0.03%)。これは意図された設計だが、UIが「次の施設が答え」と示さないと初心者は1つの職人に全資源を注いで停滞する。CHは新ヒーローの絶対DPSで圧倒させることで誘導した。

4. 数値オーバーフロー: CHはdouble上限1.8e308を超えてゾーン4725で「InfinK HP」と表示されゲームが壊れた(パッチ0.22以前の実害)。HPは×1.145/段なら約5,400段でe308に到達する。PWA(JS)ならbreak_infinity.js等の大数ライブラリか指数+仮数の自前管理を最初から入れる。

5. アクティブと放置の分断・自動化メタ: CHはJuggernaut(+0.01%DPS/クリック累積)とFragsworthがオートクリッカー前提のメタを生み、人力プレイヤーとの格差が問題化。TT2は運営がオートタッパー使用をBAN対象にしている(wiki FAQに明記)。タップ報酬は「総生産の3.5%/タップ」のような飽和する形にし、秒間タップ数に上限(CHのClickstormは10回/秒)を置く。学習ゲームでは特に「正答できる速度」が上限になるので、誤答ペナルティではなく正答ボーナス側で差をつける。

6. ボス時間制限と学習の二重苦: CHの30秒タイマーは「DPS不足なら戻って育てろ」の装置で、500ゾーンごとに−2秒(下限2秒)まで縮む。語想起に時間がかかる学習ゲームで短いタイマーを課すと「知識不足」と「DPS不足」の失敗が混ざり、何を直せばいいか分からなくなる。タイマーは緩く始めて延長手段(CHのChronos: +1秒/Lv、最大+30秒)を資源で買わせる方が安全。

7. 報酬/コストのギア比破壊: CHは報酬=HP/15で固定し、中盤の救済ボーナス(×1.025/段)にも上限×3を設けている。救済ボーナスを無上限にすると、敵HP成長(×1.145)と掛け合わさって収入がコスト(×1.07^L)を恒久的に追い越し、ゲームが自壊する。倍率系の収入ボーナスには必ずキャップを。

8. 後付けナーフの炎上リスク: CHの「500ゾーンルール」(モンスター数+0.1/500段、primal−2%等)は後から追加された減速装置で、既存プレイヤーの体感を悪化させた。個人開発では成長カーブの上限側(減速の仕組み)を最初に数式へ織り込み、後出しナーフを避ける。
