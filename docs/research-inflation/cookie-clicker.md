# Cookie Clicker — 成長曲線の実測分解

## Cookie Clicker 成長曲線の分解(全て wiki.gg / Fandom wiki / Kongregate "The Math of Idle Games" で確認した実数値)

### 1. 建物コスト成長率 ×1.15 の意味
価格式は Price = BaseCost × 1.15^N(N=所持数)。1.15 の含意:
- 1.15^5 ≈ 2.01 → 「5個買うごとに価格2倍」
- 1.15^10 ≈ 4.05、1.15^25 ≈ 32.9、1.15^99 ≈ 約103万倍(100個目は基本価格の~100万倍)
- 生産は所持数に線形(+1個=+基本CpS)なのでコストだけ指数。同じ建物を掘り続けると限界回収時間が5個ごとに2倍化し、自然に「次の建物/アップグレードへ視線が移る」。これが×1.15の本当の役割=**プレイヤーの注意を横(新ティア)に流すポンプ**。
- 比較: AdVenture Capitalist は ×1.07(+25/50個ごとに生産×2のマイルストーン)。建物種が少ないゲームほど低い成長率+マイルストーン倍率で1種を深く掘らせる設計になる(Kongregate Part I)。

Cursor→Prism(14ティア)の幾何平均: **価格ジャンプ ×12.25/ティア、生産ジャンプ ×6.4/ティア → 回収時間が毎ティア約×1.9(ほぼ2倍)**。「次の施設は約12倍高いが約6.5倍しか稼がない」が黄金比。これにより新施設は常に魅力的だが、買った瞬間にゲームが壊れない。後期建物(JS Console以降)は意図的にジャンプを×150~280に広げて天井を演出。

### 3. アップグレードの出現リズム
- 建物別ティアアップグレード: 所持数 **1 / 5 / 25 / 50 / 100 / 150 / 200…(以降50刻み、600まで)** で解禁。効果はほぼ全て「その建物のCpS×2」。
- 価格は建物基本価格の **×10 → ×50 → ×500 → ×50,000 → ×5M → ×500M…(ティア3以降は×100ずつ)**。つまり「建物を積む(線形成長)→閾値到達→×2倍率(垂直ジャンプ)」の階段が約2倍周期で来る。これが多項式成長を指数成長に引き上げる装置(Kongregate Part I:「指数コストに線形生産だけでは進行が対数化する」)。
- グローバル系: フレーバークッキー(各+1~5%CpS、乗算)、**キトン**(実績数×4%=ミルク、係数0.1~0.2で K=∏(1+milk×係数) を乗算。例: ミルク200%でhelpers+engineers → 1.2×1.3=×1.56)。実績(=やり込みの可視化)が直接CpSインフレに変換される点が重要。
- クリック系: マウスアップグレード15段(Plastic mouse: 累計手作り1,000枚で解禁・50,000で購入 → Iron: 10万枚・5M → Titanium: 1,000万枚・500M…解禁×100ずつ/価格×100ずつ)。各「**クリックに+CpSの1%**」。中盤でクリック=CpSの17~35%相当となり、タップが最後まで死なない。

### 4. CpSインフレの実測カーブ
- 体感ペース(攻略ガイド実測): 10億クッキー到達がアクティブで約2~4時間、初回プレステージ解禁(累計1兆枚)が数日~数週間。累計枚数は序盤「時間の4~5乗」的な多項式で伸び、キトン/フレーバー/新ティアの倍率が入るたびに指数へ遷移する。
- インフレの圧縮装置: プレステージレベル = (累計枚数/1兆)^(1/3) の**立方根**。+1レベル=+1%CpS。推奨初回アセンションは365レベル=累計約4.9×10^19枚。指数インフレを立方根で圧縮してメタ進行を線形に見せる。
- CpS実績は×10刻み(1→10→100→1,000…)、累計枚数実績は×100~×10刻みで、「次の実績まで常に1~2時間以内」になるよう刻まれている。

### 5. ゴールデンクッキーの瞬間倍率設計
- 出現間隔: 300~900秒のランダム。Lucky day / Serendipity が各々間隔を×0.5(両方で75~225秒)、Get Lucky が効果時間を×2。
- 効果(確率/倍率/時間): **Frenzy ~40%: CpS×7・77秒** / **Lucky ~40%: min(銀行の15%, CpS×900[=15分ぶん])+13** / **Click Frenzy ~3%: クリック×777・13秒** / Building Special ~8%: 建物数×10%・30秒 / Cookie Chain・Storm 各~0.7%。
- リスクオプトイン枠: グランマポカリプス中のWrath Cookieで **Elder Frenzy ×666・6秒**(約6%。6秒で約66分ぶんの収入)。
- 設計原則が明瞭: **倍率×継続時間×出現率の積をほぼ一定に保つ**(×7は77秒・高頻度、×777は13秒・3%、×666は6秒+デメリット契約込み)。コンボ(Frenzy×Click Frenzy=クリック×5,439)を「重なったときだけ」許すことで上振れの快感を演出。Luckyの「銀行15% vs CpS×900の小さい方」キャップは、貯蓄プレイに報酬を与えつつ無限スケールを禁止する(銀行6,000×CpS以上保持が最適、という貯金メタ戦略を意図的に生む)。

## 推奨パラメータ

- **施設コスト成長率(同一施設の買い増し)**: ×1.15^N(施設が8種以上ある場合)。施設が5種前後なら×1.07~1.10+「10/25/50個でその施設の生産×2」マイルストーン併用
  - 根拠: Cookie Clicker実測: 1.15^5≈2.01で5個ごとに価格2倍、20種の建物があるから成立。建物種が少ないAdVenture Capitalistは1.07+25個ごと×2で1種を深掘りさせる(Kongregate Math of Idle Games Part I)
- **施設ティア間ジャンプ比**: 次ティアの価格×10~12、生産×6~6.5(=回収時間が毎ティア約2倍)。式: cost_i≈cost_0×12^i, prod_i≈prod_0×6.4^i
  - 根拠: Cookie Clicker全建物の実測幾何平均(Cursor 15/0.1 → Prism 2.1Q/2.9B): 価格×12.25/ティア、CpS×6.4/ティア、回収時間150s→724,138sでほぼ毎段×1.9
- **アップグレード出現閾値とコスト**: 施設所持数 1/5/25/50/100… で「その施設の生産×2」を解禁。価格=施設基本価格×10→×50→×500→×50,000(以降×100ずつ)
  - 根拠: Cookie Clickerのティアアップグレード実測値(wiki.gg Upgrades)。線形の積み上げに約2倍周期で垂直ジャンプを挟み、指数カーブを維持する装置
- **タップ価値の式(語彙=インフレエンジンの核)**: タップ価値 = 基礎値×2^(序盤強化数) + 放置CpS×1%×マウス級強化数(累計タップ数1,000/10万/1,000万…で解禁、価格は×100ずつ)。コトダマでは「マウス級強化数」を語彙数の段階(例: 50語/200語/500語…)に、係数を職位倍率に置換
  - 根拠: Cookie Clicker実測: 初代の固定+1方式はタップが即無価値化し、現行は15段の+CpS1%方式で中盤タップ=CpSの17~35%を維持(wiki.gg Cookies per Click)
- **グローバル乗算(覚えた語数→全体インフレ)**: 知識ミルク = 習得語数×4%。乗算K = ∏(1 + ミルク×係数)、係数0.1~0.2で5~8段(語数13/25/50/75/100…語ごとに解禁)
  - 根拠: Cookie Clickerのミルク=実績×4%+キトン係数0.1/0.125/0.15/0.175/0.2の実測(wiki.gg Milk)。「実績→CpS」の変換を「語彙→灯火」にそのまま写像できる
- **瞬間ボーナス(ゴールデンクッキー相当)の倍率×時間×頻度**: 出現間隔300~900秒(強化で×0.5を2段)。通常枠: 全生産×7・77秒(出現の40%)/お題ボーナス: min(所持灯火の15%, CpS×900)+固定13(40%)/タップ枠: タップ×777・13秒(3%)。リスクオプトイン枠は×666・6秒
  - 根拠: Cookie Clicker実測値そのまま(wiki.gg Golden Cookie, Fandom Wrath Cookie)。倍率×時間×頻度の積を一定に保つのが設計原則。コトダマなら「制限時間内に正答連打でClick Frenzy発動」が学習と相性が良い
- **瞬間報酬のキャップ**: 一時金 = min(銀行の15%, 放置CpS×900秒ぶん)。上限は常に「N分ぶんの生産」で表現する
  - 根拠: Cookie ClickerのLucky実測。初期版は銀行20%無上限で貯蓄放置が支配戦略化→900×CpSキャップが追加された経緯(Fandom Golden Cookie)
- **マイルストーン実績の刻み**: 累計資源は×100刻み(1,000→10万→1,000万…)、生産速度は×10刻み。常に「次の目標が1~2時間以内」に見えるよう配置
  - 根拠: Cookie Clickerの実績ラダー実測。10億到達~2-4時間、累計1兆(プレステージ解禁)が数日、というペーシングの骨格
- **(将来枠)プレステージ変換式**: プレステージP = (累計灯火/基準値)^(1/3)、1Pごとに+1%恒久ボーナス。初回リセット推奨はP≈200~365相当
  - 根拠: Cookie Clicker実測: Heavenly chips=(累計枚数/1兆)^(1/3)、+1%/level、初回365レベル推奨(wiki.gg Ascension guide)。立方根が指数インフレの圧縮装置

## やってはいけない失敗

1. **施設が少ないのに×1.15を使う**: Cookie Clickerの1.15は「20種の建物に次々乗り換える」前提の値。施設4~5種の小規模ゲームで1.15にすると数時間で全施設の回収時間が爆発し壁化する。AdVenture Capitalistは施設が10種しかない代わりに×1.07+25個ごとの生産×2で1種を深く掘らせた。コトダマで施設が少ないなら1.07~1.10+マイルストーン方式を選ぶこと。

2. **指数コストに線形生産だけをぶつける**: コストが1.15^Nで生産が+1ずつだと、進行速度は対数的に減衰して「永遠に同じ画面」になる(Kongregate Part Iの中心命題:指数は多項式を必ず追い抜く)。Cookie Clickerは所持数1/5/25/50/100…で「生産×2」の垂直ジャンプを約2倍周期で挟むことで指数成長を維持している。倍率アップグレードの階段を最初から設計に入れないと死ぬ。

3. **瞬間報酬を銀行残高%の無上限にする**: Cookie Clicker初期のLuckyは「銀行の20%・無上限」で、「買わずに貯めてゴールデン待ち」が支配戦略になりタップも購入も止まった。現行はmin(銀行15%, CpS×900)+13にキャップ。一時金は必ず「放置生産N分ぶん」を上限にする。

4. **タップ価値を固定値で実装する**: 初代Cookie Clickerのクリック強化は固定値加算で、CpSが1桁伸びた瞬間にタップが無意味化した。現行は「クリック=+CpSの1%」を15段積ませて中盤でもタップ=CpSの17~35%を保つ。コトダマの「タップ価値=語彙数×職位」はこの教訓に合致するが、必ず放置CpSに比例する項を含めること(語彙数だけだと放置生産に置いていかれる)。

5. **大倍率を長時間・無条件で配る**: ×777は13秒のみ・出現3%、×666は6秒+グランマポカリプスという「呪いとセットの契約」でしか出ない。倍率×継続時間×出現頻度の期待値を管理せずに大倍率を配ると、通常時の生産が「ボーナス待ちの消化時間」に堕ちる。

6. **ティア間で価格と生産を同率ジャンプさせる**: 両方×10にすると旧施設が即ゴミ化し買い戻す理由が消える。Cookie Clickerは価格×12/生産×6.4で回収時間を毎段2倍にし、さらに旧施設にも×2アップグレードを与えて延命する。「新施設は強いが、旧施設も積めば閾値報酬がある」状態を維持すること。

7. **(コトダマ固有の注意)学習の正答をボーナス倍率の必須条件にしすぎる**: Cookie Clickerのゴールデンクッキーは「気づいて押すだけ」で、失敗ペナルティがない。瞬間倍率イベントに難しい想起課題を重ねると、ボーナスが「失敗体験の増幅器」になる。倍率イベントは易しい既習語プールから出すか、失敗しても0ではなく縮小報酬にする。
