## 1. タップ価値の式

```
V(w) = B0 × 2^rank(w) × 2^wm(w) × K_total
B0 = 1 灯火
K_total = K_coll × K_milk × K_mei × K_nukumi      … 恒久全体倍率
実際の獲得 = V(w) × コンボ × フィーバー × 鮮度 × 言霊バフ … 瞬間倍率(別掛け)
```

### 1.1 語の職位 rank(w)(SRS=学習接続の核)
復習間隔を空けた正解で昇格。**昇格=その語のタップ価値が恒久×2**。

| rank | 職位 | 倍率 | 昇格条件(習得からの最短経過+正解) |
|---|---|---|---|
| 0 | 見習い | ×1 | 初習得 |
| 1 | 弟子 | ×2 | +1日後の正解 |
| 2 | 職人 | ×4 | +3日後の正解 |
| 3 | 師範 | ×8 | +7日後の正解 |
| 4 | 名工 | ×16 | +14日後の正解 |
| 5 | 匠 | ×32 | +30日後の正解 |

期限超過のペナルティはなし(昇格が止まるだけ)。分散学習の間隔(1/3/7/14/30日)がそのまま経済の成長スケジュールになる。

### 1.2 語ごとのタップマイルストーン wm(w)
その語の累計正解タップ数: **10 / 50 / 250 / 1,000 / 5,000 回で各×2**(最大×32)。閾値×5刻みなので同じ語を掘る効率は逓減し、自然に新しい語(=学習)へ視線が流れる。

### 1.3 恒久全体倍率 K_total の内訳
| 項 | 式 | 解禁/取得 |
|---|---|---|
| K_coll(語彙コレクション) | 累計習得語数 25/50/100/200/300/500/750/1000 で×2(累積) | 自動(無料) |
| K_milk(知識の灯) | 灯明 = min(語数,600)×2%。K_milk = ∏(1+灯明×係数) | 「灯心」購入5段: 係数0.10/0.125/0.15/0.175/0.20、解禁50/100/200/350/500語、価格5e3/2e5/5e6/2e8/1e10灯火 |
| K_mei(工房の銘) | 1〜5個目×2、6個目以降×1.5(累積) | 注文10ごとに1個購入権、価格=直近大注文要求の×2 |
| K_nukumi(炉の温み) | 施設総所持数 25/50/100/200/400/800台 で×2 | 自動(無料) |

**銘の減衰(×2→×1.5)は必須。** 健全性条件: `ln(銘倍率) / (銘間隔10注文 × ln(注文成長率)) < 1`。後期(成長率1.13)で ln1.5/(10×ln1.13)=0.33<1 → フィードバック暴走しない(×2のままだと後期に発散する。計算済み)。

## 3. 進捗ゲージ(注文)と解禁チェーン

注文k = 「灯火W(k)ぶんの仕事量」。**手動正解タップのV(バフ込み)がそのままゲージに乗る**(獲得と納品が同時=CHのダメージ=ゴールド構造)。タイマーなし。

```
W(k) = 30 × 1.35^(k−1)            (k ≤ 40)
W(k) = W(40) × 1.13^(k−40)        (k > 40)   ← 二段勾配(CH 1.55→1.145の写像)
大注文: 5注文ごとに要求×3。報酬: 一時金 W×0.5+解禁+新テーマ語パック(20〜30語)
```

検証済みペース: Day1に9注文、以後**4〜6注文/日で一定**(=毎セッション約1解禁、大注文ほぼ毎日1回、銘2日に1回)。

解禁チェーン(抜粋): 注文3=施設T1 / 5(大)=フィーバー / 8=語パック#2 / 10(大)=施設T2+銘#1 / 15(大)=金の言霊 / 20(大)=施設T3+灯心#1 / 30(大)=施設T4 / 35=自動配り手I / 50(大)=施設T5 / 70(大)=施設T6 / 100(大)=プレステージ予告(将来枠)。
大注文は「直近パックの語70%縛り」の学習チャレンジにする(タイマー・失敗なし)。

## 5. 放置生産とアクティブ:アイドル比(本仕様の核心の発明)

放置CpSを独立変数にせず、**タップ価値に従属+log圧縮**で定義する(CC「クリック=CpSの1%」の逆向き適用+ソフトキャップ第4類型):

```
CpS = V_ref × 0.0035 × log2(1 + P_total)
V_ref = 直近100正解タップのVの移動平均(瞬間バフ除く、下限 = B0×K_total)
P_total = Σ(q_i × 所持数_i × 所持マイルストーン倍率_i) + 自動配り手の寄与
```

- **目標: 就寝8時間の放置 ≈ アクティブ1〜3セッションぶん(10〜30分)**。比 A10分/I8時間 ∈ [0.3, 3.0] を番人とし、log圧縮が構造的に保証する(30日シムで2.5→0.38に収まることを確認)。
- 1正解タップ ≈ 放置14〜70秒ぶんで推移(I2を満たす)。
- オフライン蓄積は**100%・無上限**(Pecorella「オフライン上限は離脱要因」)。
- お帰りボーナス: 復習期限語数×2%のタップブースト30分(上限×2)→ **復習が再訪のトリガー**。

この構造により「放置がタップを追い越して学習が無意味化する」事故が**数式レベルで起きない**。

## 6. 語彙=インフレエンジンの接続(新語1つの価値)

新しい語を1つ覚えると:
1. **即時**: 一時金 = CpS×300秒ぶん+その語は初日タップ×2(新鮮ボーナス)
2. **恒久(線形項)**: 灯明+2% → K_milk が約+1.0〜2.5%(灯心の段数による)
3. **恒久(指数項)**: 25/50/100/200/300/500/750/1000語の節目で全体×2(K_coll)
4. **時限爆弾**: 1/3/7/14/30日後の復習正解で×2ずつ、最終×32(rank)

→ 1日15語+復習で **V_ref が約×1.9/日** 成長(シム実測)。これが経済全体のインフレ率の主成分であり、施設・銘はその増幅器。**学習を止めると rank昇格・灯明・K_coll が全停止し、成長が銘+施設のみ(×1.1/日以下)に失速する=「語彙が燃料」が体感になる。**

## 8. 壁の設計 ―「壁を越える手段=学習」

壁は3種、すべて**学習が最安の突破手段**になるよう配置:

1. **大注文(5注文ごと、要求×3)**: 現ペースで7〜10分ぶんの仕事量=その日のセッションでは越えにくい高さ。突破手段の価格比較: 施設=灯火コスト指数 / **学習=灯火コストゼロ**(新語15個→翌日rank1昇格×2+灯明 → V_ref×1.5〜2.5)。→ 壁は「寝て、復習して、翌朝越える」設計になり、**分散学習の生理(spacing effect)とゲームのペーシングが一致する**。
2. **施設ティア(価格×12)**: 約1〜2日ごとに新ティアが視界に入る中期目標。
3. **後期勾配の保証**: 注文成長1.13/段(5注文で×1.84)< 学習継続時のV成長×1.9/日 → **学習を続ける限り構造的に詰まない**。逆に学習停止時のみ壁が立つ。

詰み防止の安全弁: 注文タイマーなし / 正解密度下限10% / 復習期限超過の減衰なし / 大注文の語縛りも失敗概念なし(I4)。

## 9. 30日シミュレーション(検証済み・実行可能)

以下は実際に実行して§5・§7の数値を得たコード(Python、`/tmp/kotodama/sim2.py` に保存済み):

```python
# 定数: WPD=15(語/日) ACTIVE_S=1800(秒/日) TAP_RATE=0.7 ACTIVE_MULT=1.8(コンボ等平均)
# WM_TH=[10,50,250,1000,5000] COLL_TH=[25,50,100,200,300,500,750,1000]
# TOSHIN=[(50語,0.10,5e3),(100,0.125,2e5),(200,0.15,5e6),(350,0.175,2e8),(500,0.20,1e10)]
# NUKUMI_TH=[25,50,100,200,400,800] OWN_MILE=[10,25,50,100,200] IDLE_K=0.0035
# FAC=[(100,1,1.10),(1200,3,1.09),(14000,9,1.08),(170000,27,1.08),(2e6,81,1.07),(2.5e7,243,1.07)]
# W(k)=30×1.35^(k-1) (k≤40) → ×1.13/段 (k>40)、5注文ごと×3

for day in 1..30:
    cohorts.append(新規15語)                       # 語彙は線形
    K_tap = K_coll(語数) × K_milk(灯明×灯心) × K_mei(銘: ×2^min(5)×1.5^超過) × K_nukumi(施設総数)
    taps=1260 を配分(新規20% / 復習期限{1,3,7,14,30日}30% / 全体50%)
    各語: V = 2^rank(経過日数) × 2^wm(累計タップ) × K_tap
    income_tap = ΣV×ACTIVE_MULT;  V_ref = ΣV/taps
    P = Σ(q×台数×所持マイルストーン);  CpS = V_ref × 0.0035 × log2(1+P)
    income_idle = CpS × 86400                       # オフライン100%無上限
    work += income_tap; 注文を cum_W(k) と照合して解決; 銘 = 注文//10
    支出: 灯心(条件達成時)→ 施設を貪欲購入(銀行の70%まで、最安値回転)
    検証: 0.3 ≤ (income_tap×1/3)/(CpS×28800) ≤ 3.0   # 番人(A10分/I8時間)
```

実行結果(キャリブレーション済み): 解禁ペース4〜6注文/日で一定、A10m/I8h = 2.5→0.38(帯域内)、Day30累計5.2e13。**チューニングノブは3つだけ**: ①IDLE_K(放置の重さ) ②注文成長率1.35/1.13(解禁ペース) ③銘の減衰点(後期インフレ率)。
