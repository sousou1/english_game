## 0. 設計原理 — パチンコから「何を借り、何を借りないか」

### 0.1 パチンコ演出の正体

パチンコの大当たりは**始動入賞の瞬間に抽選が完了**しており、その後の数十秒の演出はすべて「すでに決まった結果を、どの順番・どの濃度で開示するか」という**開示のドラマツルギー**である。脳科学的な裏付けは3つ:

1. **ドーパミンは報酬でなく予告(cue)で発火する**(Schultz)。だから本体は当たりではなく先読み・リーチ=予告の階層
2. **期待のピークは確率50%付近**(Fiorillo)。「ほぼ確」でも「ほぼ無理」でもない中間期待度の演出が最も滞留させる
3. **ニアミス効果**(Clark)。「あと一歩」は敗北なのに継続動機を生む

### 0.2 本作との根本的差異(ここを間違えると全部腐る)

パチンコは**運のゲームに物語を後付け**する。本作は**スキルと記憶のゲーム**であり、結果はプレイヤーの指が今リアルタイムに決めている。よって:

- **借りるもの**: 期待度の色階層文法 / 報酬の物理化(出玉感)/ 開示の二段化(昇格)/ 状態遷移の祝祭(確変突入)/ 音階上昇
- **借りないもの**: 偽の抽選 / ハズレの捏造(煽って外す)/ LDW=負けを勝ちの音で包む演出 / スキップ不能の長尺 / 射幸の不透明化

本作の幸運な事実: スキルゲームでは**ニアミスが無料で自然発生する**(コンボ24でミス、注文残り1声で時間切れ)。捏造する必要がなく、**可視化するだけでよい**。

### 0.3 3層アーキテクチャ(全演出の土台)

```
[入力+状態層] #pool ほか      … タップ→16ms以内に数値・タイル確定。演出を一切待たない
[演出層]      #fxLayer (新規)  … position:fixed; inset:0; pointer-events:none; z-index:55
                                 粒子・カットイン帯・暗転vignette・魔法陣はすべてここ
[音層]        audio.js         … 発火のみ。進行を保持しない(音が止まってもゲームは進む)
```

演出層は**レイアウトに参加しない**(transform/opacityのみ)。メイン画面ノースクロール方針の前提=演出がリフローを起こさないこと。

## 1. パチンコ核装置の分解 — 心理機構

| 装置 | パチンコでの仕組み | 心理機構 |
|---|---|---|
| **保留ランプ・先読み予告** | 待機中の抽選玉が点灯し、色(白→青→緑→赤→金→虹)で期待度を予告 | 予告階層の学習。cue時点でドーパミン発火。「未来の保証された試行」が離席を防ぐ |

| **プレミア演出・法則** | 超低頻度の特殊演出=当確。攻略誌・コミュニティの語り草 | 知識の収集欲。文法を知る者だけが気づく=熟達の報酬。物語の外部化(人に話す) |
| **音階上昇** | 回転音・連続演出の音程が半音ずつ上がる | 聴覚による期待勾配の連続表現。上昇=接近のメタファーは前注意的に伝わる |

### 2.1 RPGバトルへの接続(別WFのシナリオ設計と握る境界)

- **詠唱=タップ**: お題(日本語)=唱えるべき呪文の意味、タイル正解=詠唱成功。攻撃力=タップ価値(2^職位×マイルストーン…)をそのまま再スキン
- **敵HP=仕事量ゲージの再スキン**: 注文W(k)を敵HPと呼び替える。雑魚=通常注文、ボス=大注文(×3)。経済モデル無改造
- **敵の攻撃=時間経過のみ**: 一定間隔でプレイヤーHPを削る=本物の時限バトル(賭けのあるタイマーはOK)。**誤タップをダメージ化しない**(誤答没収禁止の規則。コンボ切れ+0.4秒ロック=詠唱中断で罰は十分)
- **敗北=敵が逃げる**: 機会損失のみ。資源・S値・HPアイテムの没収なし(ストック非減少原則)。「この3語を速く詠唱できれば倒せた」の敗因表示(v1の悔しさ設計を継承)
- **武器購入=施設の再スキン**: 杖=鈴(×2)、鎧=棚、等。SRS(記憶)を金で代行する武器は作らない

## 3. 音設計 — WebAudioオシレータ仕様(アセットゼロ)

既存 `js/audio.js` の `tone(freq, t0, dur, type, gain)` に2つのヘルパーを追加すれば全部作れる:

```js
// 周波数スイープ(キュイン・上昇予告の本体)
function sweep(f0, f1, t0, dur, type = 'sawtooth', gain = 0.1) {
  const o = ctx.createOscillator(); const g = ctx.createGain();
  o.type = type;
  o.frequency.setValueAtTime(f0, ctx.currentTime + t0);
  o.frequency.exponentialRampToValueAtTime(f1, ctx.currentTime + t0 + dur);
  g.gain.setValueAtTime(0.0001, ctx.currentTime + t0);
  g.gain.exponentialRampToValueAtTime(gain, ctx.currentTime + t0 + 0.01);
  g.gain.exponentialRampToValueAtTime(0.0001, ctx.currentTime + t0 + dur);
  o.connect(g).connect(ctx.destination);
  o.start(ctx.currentTime + t0); o.stop(ctx.currentTime + t0 + dur + 0.02);
}
// ホワイトノイズ(払い出しの「シャラ」。バッファ生成=アセット不要)
function noise(t0, dur, gain = 0.06) {
  const buf = ctx.createBuffer(1, ctx.sampleRate * dur, ctx.sampleRate);
  const d = buf.getChannelData(0);
  for (let i = 0; i < d.length; i++) d[i] = (Math.random() * 2 - 1) * (1 - i / d.length);
  const src = ctx.createBufferSource(); src.buffer = buf;
  const g = ctx.createGain(); g.gain.value = gain;
  src.connect(g).connect(ctx.destination); src.start(ctx.currentTime + t0);
}
```

### 3.2 ミキシング規律

- master GainNode(0.8)を1本通し、設定の音量スライダーに接続
- TTS発話中(speechSynthesis.speaking)はSFX gain×0.5(ダッキング)。listen出題の保護
- `payout`再生中は`combo`音を gain半減(音の渋滞防止)。同種SFXの同時発音は3まで
- フィーバー中のみ2秒毎の柔らかいアルペジオ(523/659/784を90msずらし、gain0.03)=BGMの代替。15秒で必ず終わるから飽和しない

## 4. 「待たせない」原則 — 全演出共通規則(出荷チェックリスト)

v1監査の最大の教訓: **ランの42%が演出待ちだった**。これを二度とやらない。

1. **結果先行・演出後追い**: 数値・タイル・ゲージは16ms以内に確定状態へ。祝祭はその後を追いかける。演出の完了を待つロジックを書かない(`await`演出は全面禁止)
2. **入力レイヤー不可侵**: 演出はすべて`#fxLayer`(pointer-events:none)。入力を塞いでよいのは既存の0.4秒誤タップロックと描画直後300msガードだけ
3. **レイアウトシフトゼロ**: 演出はtransform/opacityのみ。width/height/marginをアニメしない。ノースクロールUIの前提条件
4. **偽抽選・偽ハズレ・LDW禁止**: 煽って外す演出を作らない(スキルゲームではニアミスが自然発生するので不要)。誤答・コンボ切れに祝祭音を付けない。色階層は実倍率に正直対応
5. **タップ=早送り**: 多段演出(ラウンド払い出し・二段開示)はタップで最終状態へ即ジャンプ。キャンセルではなく完成(獲得額は1円も変わらない)
6. **尺の上限**: 毎タップ級≤150ms(音≤80ms)/節目級≤600ms/セッション級≤2.5s/記念級≤5s。すべて並行実行
7. **頻度の階層管理(演出インフレ防止)**: 毎タップ(音階+spark)/10タップ毎(発光段階)/セッション1〜3回(フィーバー・注文)/週1回以下(昇進・プレミア)。下の階層の演出を上の頻度で鳴らさない。全部が特別=何も特別でない
8. **学習保護**: インライン想起カード(起こす)展開中はfxLayer演出を抑制(spark・カットイン停止、音は`ok`/`bad`のみ)。SRSの想起に祝祭ノイズを混ぜない。SRS誤答への否定演出は現状(ログ1行)から強化しない
9. **安全とアクセシビリティ**: フラッシュは輝度差を抑え3回/秒以下。`prefers-reduced-motion`で粒子・フラッシュ・回転を停止(色変化と数字のみ)。設定に演出強度3段階「ひかえめ/ふつう/ドバドバ」
10. **演出予算**: 同時DOM粒子≤40(オブジェクトプール化、現行のcreateElement+setTimeout方式を改修)。rAFループは既存tickの1本に統合。フレーム16ms超過の演出は削る

**監査運用**: 全演出PRでこの10項目+v2設計書§7の11項目をチェック。違反0件が出荷条件。
