## 0. 設計原則(全節に優先する差分の不変条件)

| # | 不変条件 | 由来 |
|---|---|---|
| B1 | **敵HP = 注文の要求仕事量 W(k) の完全リスキン**。`orderTarget()`の式・数値は一切変更しない | 検証済みペーシング(討伐4〜6体/日)を丸ごと継承 |
| B2 | **ダメージ = 既存のタップ獲得 `gain` そのもの**。「攻撃=収入」なので戦闘行為自体が常にプラス収支 → 敗北しても損失が構造的に発生しない | I4(没収禁止) |
| B3 | 誤答はプレイヤー被ダメ・敵ゲージ加速に**一切接続しない**。罰は現行どおりコンボ切れ+0.4秒ロックのみ | 学習罰の禁止 |
| B4 | プレイヤーHPが意味を持つのは**プレイヤー起点で開始する「討伐チャンス」のみ**。雑魚に反撃はない | 偽タイマー禁止/本物の賭けタイマーのみ可 |
| B5 | Lv/EXPは**攻撃力に乗らない**(HP・会心のみ)。火力は既存の語彙経済(職位・コレクション・武器)が独占 → 「学習が火力、プレイ量は生存力」 | 学習=燃料の維持 |
| B6 | 討伐数(=解禁チェーンのクロック)は**手動タップでのみ進む**。放置・からくりの手は敵HPを削らない | 既存invariant「注文進捗=手動のみ」 |
| B7 | バトル層はSRS(`srs.js`/`schedule.js`)に**1バイトも書き込まない** | SRS汚染禁止 |
| B8 | 演出はCSS/絵文字/シルエット/パーティクルのみ。全画面割込みなし、描画直後300ms入力ガード、16ms応答 | 個人開発+フィール規則 |

## 2. 敵: HP・出現サイクル・報酬

### 2.1 HP(pool.jsへの差分: 名称変更のみ、式は不変)

```
HP_enemy(n) = orderTarget(n)                       … 現行式そのまま
  = 30 × 1.35^(n)            (n+1 ≤ 40)
  = W(40) × 1.13^(n+1−40)    (n+1 > 40)
  × 3   if (n+1) % 5 == 0    (中ボス)
  × 5/3 if (n+1) ∈ CHAPTER_BOSSES  (章ボス: 中ボスと重なるので計×5)
```

### 2.2 出現サイクル

- **常時1体が画面上部に居る**(現行の注文行を敵エリアに置換)。倒すと300ms のスポーン演出(この間は既存規則どおり入力ガード)で次の個体が出現。**敵不在の時間ゼロ**(タップが宙に浮かない)。
- 雑魚・結界へのダメージは**セッションを跨いで永続**(= `order.got` の現行挙動)。タイマーなし。
- 章ごとの雑魚ロスター(絵文字+CSSシルエット、シナリオWFに命名フックを渡す):
  - 章1(村): 🐀野ねずみ / 🦇洞こうもり / 🕷森ぐも / 👺ゴブリン、中ボス🦌森の主、章ボス🐉黒霧の竜(20体目)
  - 章2(都会): 🐍路地の毒蛇 / 🃏ペテン妖精 / 🗡辻盗賊 …(50体目=章ボス)
  - 以降90/140体目(恋愛編クライマックス/闇堕ち分岐)はシナリオWF側で確定

### 2.3 討伐報酬(経済差分: ここだけ既存から変更)

```
ゴールド報酬 G(n) = round(HP_enemy(n) × 0.5)      … 現行 orderReward の「灯火→ゴールド」置換
EXP報酬           = +10(雑魚) / +150(中ボス・章ボス)
```

- 現行の一時金(灯火 W×0.5)は**廃止し全額ゴールド化**。注文ペース(4〜6/日)はゲージ充填=タップ収入で決まるため不変。施設購入ペースのみ僅かに低下する → **補正ノブ: Day7シムで施設ペースが遅れたら `IDLE_K` 0.21→0.24**。
- 討伐演出: 敵シルエットが崩れて灯火パーティクル化→ゴールドが弾けてカウンタへ吸い込まれる(800ms、タップでスキップ可)。

## 3. ダメージ式(詠唱攻撃)

既存 `wordValue × comboMult × 鮮度` に**武器係数と会心**だけを掛け足す:

```
DMG = B0 × 2^rank(w) × 2^wm(w) × K_coll × K_ring × K_weapon   … 恒久部(wordValueの差分)
      × combo × fever × 鮮度 × crit                            … 瞬間部
K_weapon = Π(所持武器の倍率)        … §5。新規の恒久乗算スロット(economy-spec §1.3 の K_mei の実装)
crit     = 2.0(会心時)/ 1.0
獲得灯火 = DMG(ダメージと獲得は同一値。現行 gain の意味を変えない)
敵HP進捗 = DMG(手動タップのみ。auto(使い魔)は乗らない — 現行どおり)
```

`p.vref` は現行式(`gain / comboMult` の移動平均)のままで K_weapon を自動的に含むため、**放置生産も武器に追従しつつ log2 圧縮で従属が保たれる**(I2「1タップ=放置14〜70秒」は不変)。

| 見切り(ピンとき)正解 | +25 | |

| 新語を招く | +10 | |
| 雑魚討伐 | +10 | |
| ボス討伐 | +150 | |

想定: 約2,300 EXP/日(タップ1,260+学習層)。

### 4.2 レベル曲線

```
次のLvまで: N(L) = round(100 × L^1.4)
累計:       E(L) ≈ 41.7 × L^2.4
```

ペース実測(シム前提値で): **Day1→Lv5、Day7→Lv12、Day30→Lv22、Day90→Lv35**。序盤は1セッションで複数回レベルアップ(ドーパミン)、巡航0.5〜1Lv/日。

### 4.3 ステータス

```
HPmax = 80 + 20 × Lv          (Lv5=180, Lv12=320, Lv22=520)
会心率 = 4% + 0.2% × Lv(Lv由来上限+6%)+ 武器特性(総上限20%)
会心倍率 = ×2(ダメージ=獲得灯火も×2)
攻撃力表示 = 現在の平均詠唱力 ≒ p.vref(新規ステータスではなく既存値の表示)
```

- レベルアップ: HP+20、会心+0.2%、**全回復**(ボス戦中なら回復演出=逆転の高揚)。
- 会心の経済影響: 平均×1.05〜1.2。`ACTIVE_MULT` 想定を 1.8→1.9 に補正(シム再計算ノブとして明記)。

## 5. 武器(ゴールド経済) — 「銘」スロットの実装として

### 5.1 位置づけ: 施設と**共存・通貨分離**

- 施設=灯火建て(拠点・放置)、武器=ゴールド建て(戦闘・タップ倍率)。**ひとつの財布を奪い合わない**。
- 武器の攻撃倍率は economy-spec §1.3 で計画済み・未実装の「工房の銘(K_mei)」をそのまま武器に充当。**×2が5本、6本目以降×1.5** — 健全性条件 `ln(1.5)/(10討伐 × ln(1.13)) = 0.33 < 1` を継承(後期フィードバック暴走なし、計算済み)。

### 5.2 購入規則と価格

```
購入権: 討伐10体ごとに1本(銘の「注文10ごと」を継承)
価格:   P_weapon = HP_enemy(直近の中ボス) × 1.0   (ゴールド建て)
```

価格検算: 討伐10体分のゴールド収入 ≈ 0.5×ΣW ≈ 3.0〜4.6 × W(直近ボス)。価格=3×W(k)(中ボスHP) → **約2日で1本買える**(序盤ややカツカツ→後期に余裕。余剰ゴールドは将来のポーション/装飾のシンク枠)。

### 5.3 武器テーブル(初期値)

| # | 名 | 解禁(討伐数) | 倍率 | 特性(装備中のみ有効) |
|---|---|---|---|---|
| 0 | 樫の杖 | 初期装備 | ×1 | なし |
| 1 | 旅装の短剣 | 10 | ×2 | 心得: 誤タップ1回までコンボ保持(60秒に1回回復) |
| 2 | 蒼火の魔杖 | 20(章1ボス後) | ×2 | 会心+4% |
| 3 | 風詠みの弓 | 30 | ×2 | 詠唱ラッシュ+3秒(上限21秒に拡張) |
| 4 | 月鋼の剣 | 40 | ×2 | 鮮度の速窓 2秒→3秒 |
| 5 | 竜骨の大杖 | 50(章2ボス後) | ×2 | ボス戦の被ダメ−25% |
| 6+ | (10討伐ごと) | 60,70,… | ×1.5 | 特性1〜5の強化版ローテ |

**倍率は所持累積(鍛冶の継承)、特性は装備中の1本のみ**。→ 買う=恒久成長、装備=プレイスタイル選択。型落ち武器が死なない。

## 6. 時限フィーバー『詠唱ラッシュ』(pool.js CURVE差分)

### 6.1 突入: ゲージ式に変更(現行 combo===25 トリガを置換)

```
ゲージG ∈ [0,25]: 正解+1 / 誤答−3 / 無操作1秒−1(灯し場アクティブ中のみ減衰)
G=25 → 自動突入(1秒のリーチ演出=バナー+画面縁発光。入力は生きたまま。全画面割込みなし)
```

誤答−3はeconomy-spec §2で監査済みの規定値(資源没収には当たらない: ゲージは貯蓄でなく勢い)。

### 6.2 本体: 延長型の本物のタイマー

```
基本時間 10秒、正解1回ごとに +0.4秒、合計上限 18秒(武器3で21秒)
倍率 ×3 × 連鎖補正、タップ価値・与ダメに乗算(現行 feverMult の系譜)
フィーバー中の正解はゲージに乗らない(G=0から再充填)
```

「正解し続ける限り炎が消えない」= 賭けが本物の時限。偽タイマー禁止規則に適合。

### 6.3 連鎖(チェーン) — 『もう一回』の機構化

```
終了から45秒以内に次のラッシュへ突入 → 連鎖+1: 倍率 ×3.0 → ×3.25 → ×3.5 → ×3.75 → ×4.0(上限)
45秒経過で連鎖リセット(×3.0に戻る)
残り火: 終了時 G=6 からスタート(25まで正解19回 ≈ 27秒 → 45秒窓に間に合う緊張感)
```

### 6.4 終了時の精算演出(1.6秒、タップで短縮、表示直後300ms入力ガード)

1. 獲得灯火がスロット式ロールでカウントアップ(吸い込みパーティクル)
2. 最大コンボ/会心数をスタンプ(ドンッ)
3. **「連鎖継続 ×3.25 — あと45秒」のリアルタイムタイマーゲージ点火**(これが導線。本物の賭けタイマー)
4. プール画面内のパネルで完結。全画面モーダルにしない

### 6.5 経済検算

サイクル ≈ 充填27秒+ラッシュ平均14秒 → 稼働率 ≈ 34%(現行 ≈ 29%)。`ACTIVE_MULT` 1.8→1.9〜2.0 へ微増。**調整ノブ: 残り火G(6→0)と基本時間(10s→8s)**。

## 7. プレイヤー被ダメと敗北: 『討伐チャンス』

### 7.1 構造(中ボス・章ボスのみ。雑魚は無反撃)

```
ボスHP = 結界 75% + 本体 25%
結界:   通常プレイで削る(タイマーなし、ダメージ永続 = 現行大注文と同一)
本体:   結界破壊後、「討伐チャンス」ボタンが点灯(プレイヤー起点でのみ開始)
```

### 7.2 討伐チャンスの戦闘

```
本体HP    = 0.25 × HP_enemy(boss)
ボス攻撃  = 5秒ごとに ATK(b) ダメージ(章ボスは4.5秒)
ATK(b)    = round( HPMODEL(b) / 13 )
HPMODEL(b)= 80 + 20 × (55.4 × max(1, b−1))^(1/2.4)   … b=通算ボス番号。
            「順調ペースのプレイヤーが13発≈65秒生存する」よう正規化(55.4 = 想定EXP/日 2310 ÷ 41.7)
プレイヤーHP = 開始時に全回復(戦闘外でHPは常に満タン。回復薬グラインドを作らない)
```

- 攻略の核: **必要DPSは「順調ペースのV × 0.8(ラッシュ込み)/ × 1.7(ラッシュなし)」**。→ ゲージを満タンにしてから挑むのが定石になり、§6と噛み合う。
- 火力不足で勝てない場合の正規ルートは**学習**: 新語+復習で翌日 rank昇格×2 → V_ref ×1.9/日(シム実測)。「寝て、復習して、翌朝竜を斬る」— 分散学習の生理が攻略法になる(economy-spec §8の壁設計を継承)。

### 7.3 敗北 = 軽い撤退(没収ゼロの証明)

- HP0 or 撤退ボタン → 「体勢を立て直した」。**失うものは無し**: 戦闘中の与ダメ=獲得灯火は既に手元(B2)。結界は壊れたまま。SRS・コンボ資産・ゴールド不変。
- 本体HPは全回復するが、**撤退1回ごとに本体HP上限−5%(累積上限−30%)**=「竜も傷つく」。負け続けても見える前進があり、絶望が発生しない。
- 再挑戦は即時可(クールダウンなし。ゲージ充填が自然な間合いになる)。
- 誤答は被ダメ・敵攻撃周期に**無関係**(B3)。時間圧は存在するが、ミスへの二重罰は置かない。
- コピー例: 「無理は禁物だ。村に戻って湯を浴びよう。(灯火とゴールドはそのまま)」

## 9. 放置との整合(留守中の自動戦闘)

- **実体は現行 `settle()` のまま変更ゼロ**。留守の灯火収入を「仲間の見回り戦果」としてRPG語で再叙述する:
  - 帰還精算: 「留守の間、火守りの妖精たちが魔物の群れ ×N を追い払った(+X灯火)」 N = ceil(gained / 現雑魚HPの5%) の純演出値。
  - 偽タイマーではない: 表示は真実の精算結果の言い換えであり、数値を1つも捏造しない。
- **討伐数は留守中に進まない**(B6)。敵を倒すのは主人公=プレイヤーだけ。解禁ペースが学習から切り離されない。
- うとうとの言霊→熟成マナの仕組みも不変(「眠った呪文が夢で力を蓄える」とリスキン)。
