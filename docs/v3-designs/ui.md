# メイン画面設計仕様 v3.0 『詠唱バトル画面』— ノースクロール 390×844

- 版: 2026-06-12 設計確定版
- 対象: RPG化方針に基づくメイン画面の全面再設計(縦1カラムスクロールUIの廃止)
- 基準解像度: 390×844(iPhone 12/13/14級、セーフエリア込み: 上47px / 下34px、実効763px)
- 前提資料: `docs/v2-design.md`(フィール規則§7)、`docs/economy-spec.md`(タップ価値・注文・フィーバー数値)、`docs/game-feel-audit.md`(ゴーストクリック等17件の教訓)

---

## 1. 設計原則(本仕様の全節に優先)

1. **スクロールゼロ**: メイン画面の全要素が844pxに常時収まる。`overflow:hidden`。情報の追加はシート(§8)かステージ内インライン表示で行い、縦に積まない。
2. **新しい経済を発明しない**: RPGの数値はすべて既存経済の再スキン(§2)。敵HP=注文の仕事量、攻撃力=タップ価値。二重帳簿を作らない。
3. **下半分は指の領土、上半分は目の領土**: y411以降(お題・プール・メニュー)は親指圏の入力ゾーンで、演出を一切被せない。y411より上(ステータス・ステージ)は演出シアターで、入力を要求しない。
4. **フィール規則(v2設計書§7)全11項目を継承**: 偽タイマー禁止(本物の賭けのタイマーは可)/16ms応答/描画後300ms入力ガード/全画面割込み禁止/誤答没収禁止/SRSへのゲーム圧汚染禁止。
5. **個人開発の描画予算**: 敵・シーンはCSSグラデーション+絵文字+シルエット+パーティクルdivで構成。スプライト量産はしない。

## 2. RPG転写マップ(既存システム → RPG語彙)

| RPG概念 | 既存システム | 備考 |
|---|---|---|
| 魔法詠唱 | 灯し場の正解タップ | 英語=異界の言葉。お題「凍らせる」→ `freeze` をタップ=詠唱 |
| 攻撃力 ⚔ | タップ価値 V_ref(2^職位×語マイルストーン×コレクション×施設) | ヘッダに常時表示。語彙が育つ=攻撃力が上がる、が毎タップ見える |
| 敵HP | 注文の仕事量 W(k)=30×1.35^(k−1) | **撃破=注文達成=解禁**。ゲージを敵の身体に乗せただけで経済は無改造 |
| ボス | 大注文(5注文ごと要求×3) | 語パック70%縛り=「弱点属性」として翻訳。失敗概念なしは維持 |
| ゴールド 💰 | 灯火 | 表記は万進(経済仕様§7) |
| プレイヤーHP ❤ | **新規**(敵の実タイマー攻撃でのみ減少) | **誤答では1も減らない**(§9)。HP0=「後退」で没収ゼロ |
| Lv / EXP | 累計「確かな想起」数建て | 時間建てアンロック禁止規則に適合 |
| 武器・防具(武器屋) | 施設・灯心・銘の再スキン | 武器=タップ系乗数 / 防具=被ダメ軽減・HP上限 / 従者=自動配り手 |
| 呪文書 | 言霊リスト | rank(見習い→匠)=呪文の位階。新語の「招く」もここ(「新しい頁を編む」) |
| 修行 | 起こす(FSRS期日想起)+鐘 | バトルから物理的に隔離(§8.3)。SRS汚染禁止の砦 |
| シナリオ | 物語シート+ステージ背景の差し替え | 村→都会→…はCSS背景シーンの交換で表現 |

## 4. 推奨: 案B『冒険者の手元』

理由:
1. **「タップ→ダメージ→敵HP→報酬→攻撃力成長」の因果連鎖が1画面で閉じる**。本作の核「語彙が燃料」(economy-spec§6)は数字が見えて初めて成立し、案Aはこれを失う。
2. ステージ276pxは、絵文字96px敵+ダメージポップ+魔法陣+セリフ1行に十分で、案Cの220pxでは会心演出(40px数字)が窮屈。
3. 案Cの常設情報のうち「実際に毎秒判断に使う」のはコンボ/フィーバー/敵HPのみ。残りは武器屋・呪文書シートで需要時に見せれば足りる(プログレッシブ開示)。
4. 救済措置: 設定に「詳細表示」トグルを置き、ONで案Cの要素のうち**バフ行(28px)と詠唱価値プレビューのみ**を案Bに追加する(ステージを248pxに縮めて吸収)。廃人と15歳の両取り。

### 5.2 各エリア仕様

**A: ステータス(44px)**
- 左→右: `Lv12`(14px) / HPバー140×10px+`84/100`(12px) / `⚔1.7億`(16px太字、=V_ref万進表記) / `💰1.2億`(16px太字、tabular-nums)。
- ゴールドはrAFで常時カウントアップ(放置収入で微動=偽ゲージ禁止規則の「本物の数字は常に動く」)。
- 獲得時は加算前→後のオドメーター回転(450ms)。⚔は昇格・解禁時に金色フラッシュ+数値ロール。

**B: ステージ(276px)**
- 背景: シナリオ進行で差し替わるCSSシーン(線形グラデ空+SVGシルエット2層+靄のradial-gradient)。村→都会の進行はここで無言で語る。
- 敵: 絵文字96px+`drop-shadow`+`hue-rotate`でバリエーション生成。idle時は2.4s周期の浮遊アニメ。被弾でknockback 120ms+白フラッシュ。
- 敵HPバー: 300×12px+`62%`(12px)。注文ゲージの転用。**正解タップのV(バフ込み)がそのままダメージ**。
- 敵攻撃テレグラフ: 右上に`⏳敵の詠唱 7`(本物のカウントダウン、毎フレーム動く円形リング)。0で攻撃→プレイヤーHPに実害。**本物の賭けがあるタイマーなので表示可**。
- セリフ/ログ行: 下端1行14px。物語・精算・昇格通知はすべてここを流れる(全画面割込みなし)。タップで直近8行をステージ内オーバーレイ表示(プレイヤー起点なので可)。
- 敵タップ: 弱点と報酬予告の吹き出しを2秒表示(任意操作。入力必須にしない)。
- バトル間はシーン会話モード: 敵の位置に人物シルエット+セリフ。シナリオの短い会話はステージ内で完結させる。

**C: コンボ/フィーバー帯(44px)**
- 左: `🔥×1.42`(18px太字イタリック、+2%/連・上限×2)。更新はオドメーター。コンボ切れは赤フラッシュ120ms+減衰アニメ(無言でリセットしない)。
- 中: フィーバーゲージ206×16px、5目盛ごとに刻み、`18/25`(12px)。誤タップ−3/無操作1秒−1(既存仕様)で常時動く。
- 右: 満タン時のみ`[⚡解放]`ボタン56×44px出現(脈動)。**自動突入ではなくプレイヤー起点**。満タン保持は30秒まで、以降−1/秒で減衰(炎が燃え尽きるアニメ、止まって見えるゲージ禁止)。

**D: お題(64px)**
- 日本語お題24px太字中央。切替時は120msスライドイン。
- 右端: 鮮度リング◔(2秒以内×1.5/5秒以内×1.2の実ボーナス窓を排水リングで正直表示。減点はないためリング消滅=等倍に戻るだけ)。
- お題の文言はRPG命令形に統一:「凍らせる」「borrow=借りる、を唱えよ」等のテンプレ。

**E: 詠唱プール(258px)**
- タイル179×78px×6(2列3行)。英単語20px、12文字超は16pxまで自動縮小。1お題につき正解1枚(密度17%)。
- 正解: タイルが光→敵へ飛翔する弾パーティクル(CSS transform 240ms)→着弾ポップ。タイルは即座に新語が補充(120msフェードイン+**入力ガード300ms**)。
- 誤タップ: 震えshake 240ms+彩度落ち0.4秒ロック+コンボリセット。**没収・HP減・SRS記録は一切なし**。
- フィーバー中: タイル枠が燃える。金の言霊イベントは正解タイルの虹色化で表現(プール外に要素を足さない)。

**F: メニューバー(56px)**
- 5スロット各78×56px(タップ判定はスロット全域=44px以上を満たす): `📜物語` `⚔武器屋` `📖呪文書` `🔔修行` `⚙設定`。アイコン28px+ラベル10px。
- バッジ: 修行=FSRS期日数(本物の数のみ)。武器屋=購入可能品がある時に`!`(買える事実ベース)。偽の赤丸禁止。

## 6. プレイヤーHPとバトルの誠実な実装(フィール規則との両立)

- **HPが減る経路は敵の実タイマー攻撃のみ**。誤タップ・遅い回答ではビタ一文減らない(誤答没収禁止の拡張)。
- 敵の攻撃間隔は敵種ごとに固定(雑魚12秒/ボス8秒)。テレグラフは常時可視・毎フレーム駆動。シート展開中とバックグラウンド中は凍結し、ステージに「敵は様子を見ている」と正直に表示。
- 防具(武器屋)で被ダメ軽減・HP上限増。HPは時間で自然回復(戦闘外)。
- **HP0=「後退」**: 敵が攻撃を止め、回復までプールが「休息モード」(タップで深呼吸=回復加速)になるだけ。ゴールド・敵HPダメージ(=注文進捗)・コンボ由来の獲得分は**一切失わない**(ストック単調性)。敵HPは回復しない。
- 時間圧とSRSの分離: メイン画面のプールは**既習語の運用(流暢性訓練)**でありSRS記録に触れない。FSRS想起は修行シート(§8.3)でのみ行い、そこにはタイマー・敵・コンボが存在しない。

## 7. フィーバー時間制限仕様(「連続プレイさせる」の正体)

1. ゲージ25正解で満タン(誤タップ−3、無操作1秒−1 ← 既存値を維持)。
2. 満タン→`[⚡解放]`点滅。**30秒以内に点火しないと−1/秒で減衰開始**(本物の賭け。残量は燃え尽きる炎アニメで常時可視)。
3. 点火→**15秒間 ×3**+テンポ×1.5+コンボ不断。カウントダウンは敵頭上の魔法陣リング+C帯のバーで常時駆動(本物のタイマーなので表示可)。
4. **チェーン**: フィーバー中に正解15回でゲージ再満タン→終了瞬間に自動で2段目`×4`(15秒)→3段目`×5`(上限)。「もう1段」の追求が連続プレイの動機。チェーン失敗のペナルティはなし(獲得分は全部残る)。
5. 終了時: C帯とログに獲得明細1行(`フィーバー +8.2億`)。明細なしの花火だけ、を禁止。

## 10. 入力規則(game-feel-audit 17件の教訓の継承)

1. 全タップ要素はpointerdownで16ms以内にpressed状態(scale 0.96)。
2. プール再配布・選択肢系の描画直後300msは入力無効+120msフェードインで「まだ押せない」を可視化(監査#13)。
3. **reveal後に押し始めたポインタのみ回答として受理**(pointerdown起点判定。ゴーストクリック対策、監査#6/#12)。
4. 正解後の入力デッドタイムは作らない: 演出は非同期で完走させ、次のお題は即タップ可能(監査#7/#14の反転)。
5. タイル内で指が転がっても離した位置が同タイル内なら成立。フリック誤爆判定はしない(監査#8/#9)。
6. 数字とゲージの二重表示は単一ソースから同フレーム更新(監査#3/#15)。

## 11. 画面サイズ可変対応

## 12. 実装メモ(描画予算)

- 敵: 絵文字1字+CSS(`drop-shadow` 2層、`hue-rotate`で色変異、`clip-path`の影シルエット)。1敵=定義JSON 5行(絵文字・色相・HP・攻撃間隔・セリフ)。
- シーン: CSSグラデ空+SVGシルエット2層+靄(radial-gradient、`opacity`アニメ)。1シーン=CSS 30行。
- パーティクル: 使い回しのdivプール(最大40個)、`transform`+`opacity`のみ(リフロー禁止)。
- ダメージポップ: ステージ直下のオーバーレイ層に挿入し、画面遷移と無関係に完走(監査#17の教訓)。
