**読み解き:** 各審査員のトップ票は三者三様(中毒性→スミス、学習効果→リコレクト、実現性→工房)に割れたが、**全視点で8.0以上を取ったのはスミスただ1本**。審査全体を貫いた分水嶺は次の3点だった。

1. **想起がコアバーブか、幕間の儀式か** — 想起そのものが毎秒の操作になっている案(スミス、リコレクト)が強く、戦闘がフィラーになる案(サバイバーズ、バザール)は学習密度で脱落した。
2. **報酬曲線の「形状」** — 忘却に単調増加の報酬を払う案は「わざと放置して高レアを狙う」逆インセンティブを抱える。リコレクトだけが「望ましい困難のピークで倍率最大」の山型を明示した。
3. **D30以降を「運営」で支えるか「自己再生」で支えるか** — ソロ開発のシーズン運営は例外なく息切れする。プレイヤー自身の忘却履歴がコンテンツを無限生成する装置(ハントの宿敵、スミスの賞金首)を持つ案だけが数ヶ月スケールで生き残る。

## 2. トップ3企画 詳細紹介

### 第1位 『コトダマ・スミス 〜錆と忘却の塔〜』(総合 8.50)

**コンセプト:** プレイヤーは言霊鍛冶師。英単語=言霊カードを燃料に炉を焚き、Balatro式の規定スコア超えで塔を登る。忘却は「錆」、想起は「炎」。

- **コアループ:** 1ラン約6分。手札5枚の言霊を「フリック想起」(正解方向へフリック=火花とチップ加算音)で流し、正解ごとに決定論的に燃料が加算→鍛冶具(ジョーカー相当)が乗算増幅→火柱と桁上がりでノード突破。ノード間に報酬3択のガチャ快感、ボスは自分の忘れかけ単語の集合体「忘却獣」。8案で唯一、**想起がスキップ不能なコアバーブそのもの**(毎分5回前後)であり、L1の快感と学習密度が完全に一致している。
- **SRSの溶かし込み:** 裏でFSRS。想起確率Rの低下をカードの「錆」として描画し、復習期日は「錆びてきたカードを磨きたい」収集欲に翻訳。配牌の約50%をdue語に不可視リグし、出題選択権はプレイヤーに渡さない。誤答は「暴走」→同ノード内の遅延再挑戦、2連続失敗語は「賞金首」化して次ラン以降のエリート敵になる。
- **ハクスラ要素:** **レアリティ=FSRSの記憶安定性そのもの**(錆鉄→銅→銀→金→星鋼)。課金でも周回でも上がらず、正しいタイミングの想起だけが上げる——「図鑑の総合力=本人の語彙力」が文字通り成立し、学習のオマケ化が構造的に不可能。運はドロップ・鍛冶具のアフィックス側に完全隔離。鍛冶具シナジーは品詞・接頭辞・語根に掛かり、ビルド探索=語彙知識の構造化になる。
- **なぜ続けたくなるか:** 敗因単語トップ3(「この3体を鎮めれば突破できた」)が「勝ちたい」と「覚えたい」を同一欲求に縫合する。1ラン6分+再開コストゼロ+デイリーシード炉+週次の個人賞金首レイド+焼入れラダーと、D1/D7/D30の全時間軸に専用装置が漏れなく配置され、敗北日もアンロックが進む。
- **弱点と対策:** ①フリック想起が選択肢即時提示で純粋想起ウィンドウを欠く→**選択肢の遅延提示(想起の間)を1工程追加**(後述ハイブリッドで吸収)。②「燃料∝経過間隔」の単調増加が放置farmを誘発→**山型カーブへ修正**。③ジュース品質(火柱・加算音)への依存が高い→演出工数をここに集中投資。④レアリティ降格の損失回避→降格幅圧縮+即復帰演出+床値保証。⑤6〜9ヶ月スコープは楽観的→初期は2,000語・3形式・鍛冶具30種に圧縮。

### 第2位 『コトダマ工房 〜眠る言霊と忘却の獣〜』(総合 7.33)

**コンセプト:** 英単語=工房に住む小さな従業員「言霊」。覚えるほど世話なしで働き続ける、逆転の錬成放置ゲーム。

- **コアループ:** 1日2〜4回×5〜10分。起動即オフライン生産を回収→「覚醒の儀」(眠そうな言霊をタップ→想起クイズ→熟成マナ一括放出+ドロップ判定)→経営フェーズ(装備・施設)→新人スカウト→次の閾値を見て閉じる。任意の「言霊遠征」が30分コースの受け皿。
- **SRSの溶かし込み:** 8案中**最も美しい概念翻訳**——「記憶の定着=自動化アンロック」。FSRSの復習間隔が伸びる=その言霊が一人で働ける時間が伸びる、なので「覚える」と「放置効率が上がる」が同一の行為になる。完熟ウィンドウ(R≒0.9)での覚醒にボーナス、早すぎる覚醒は生産一時停止のペナルティ——**既知語の高速周回という学習回避の最頻エクスプロイトを定義レベルで殺した唯一の案**。FSRS期日を朝市・昼市・夜市の3窓に量子化し、分散学習を生活リズムに固定する。
- **ハクスラ要素:** 正解想起1回=採掘1振り。レアリティ床は「自立度(間隔の長さ)×完熟命中×コンボ」で上昇し、30日語の完熟覚醒はレア確定。装備は4アーキタイプ(多読型/熟成型/放置型/遠征型)のビルドを成立させるが、全装備は想起でしか生まれない「記憶燃料」が無いと出力ゼロ。
- **なぜ続けたくなるか:** 放置ジャンルの再訪文法(回収のために開く)とSRSの分散学習リズムが構造レベルで一致しており、Duolingoの「1日1回の義務」と質的に異なる生活組み込みができる。離脱=仕込み、起動=収穫の負コスト設計はD1〜D7習慣形成に強い。技術的実現性は8案中最高(全部品が枯れている)。
- **弱点と対策:** ①瞬間の快感が8案中最弱で、調整を誤ると「Ankiの督促」に退化→覚醒演出と完熟ボーナス比率の数値テストに専用工数を確保。②FSRS間隔成長×放置経済乗数という二重非線形系のバランス調整地獄→片側(放置経済)を意図的に単純な線形+キャップに倒す。③ミッドゲーム砂漠が運用頼み→生成型コンテンツ(日替わりモディファイア、合成)へ早期に重心移動。④画面の地味さ=ストア映えの弱さは未解決のまま受け入れる必要がある。

### 第3位 『言ノ刃(コトノハ)〜忘却のダンジョンと語彙の鍛冶場〜』(総合 7.00)

**コンセプト:** 忘れかけた単語ほどよく斬れる。デイリー習慣化×戦利品経済の翻訳。

- **コアループ:** 1セッション基本5分。「今日の遠征」=5部屋固定ダンジョン(敵編成は今日のSRS期日語から自動生成)で、手札の単語カードを敵の弱点(意味/穴埋め文)へドラッグして斬る。締めは昨日の宝箱の「鑑定」と、文字タイル並べの「今日のボス単語」(絵文字グリッド共有)。
- **SRSの溶かし込み:** 8案を通じて**最もエレガントな一行の発明**——「ドロップした未鑑定装備は翌日まで開封できない」。"明日まで存在しない宝箱"がSRSの第1間隔そのものになり、アポイントメント機構と分散学習が単一の仕掛けで実現される。XPを忘却確率に比例させて「スコア最適化=学習最適化」を強制一致させるグッドハート対策の明示も誠実。
- **ハクスラ要素:** レアリティ=実際のコーパス頻度(低頻度語ほど強いが錆びやすい=本当に忘れやすい、というリスク・リターンに嘘がない)。強化値+1〜+8=SRSステージ。語源セットボーナス(spec-/port-/dict-)でビルド構築が接辞・語根学習そのものになる。+8「殿堂入り」で錆び対象から外れ、後半ほど復習負荷が軽くなる(挫折曲線と逆相関)。
- **なぜ続けたくなるか:** 4枠の皿(遠征/鑑定/ボス/復讐戦)、儀式的なセッション終了、野営式ストリーク(0リセットなし+復帰追い風バフ)と、デイリー習慣化装置が教科書的に完備。D7リテンションの確度は8案中最高。サーバほぼ不要のスコープ規律も最上位。
- **弱点と対策:** ①「罪悪感ゼロで5分で切る」思想ゆえに中毒性へ自ら天井を張っており、エンゲージメント総量がDuolingo型の下限に張り付く→「つい30分」側の導線を別腹でなく本線に組み込む再設計が要る。②手札からのドラッグ式は消去法に最も脆い→複数敵同時マッチング+同領域ディストラクタで弥縫しつつ、ボス以外にも産出形式を増やす。③Duolingoの本丸(デイリー×ストリーク×5分)と正面衝突するポジショニング→ハクスラ経済の深さで差別化を立てる。④3,000語分の「絵」は最高コストのデータ選択→MVPでは絵を捨てる。

## 3. ハイブリッド最強案 『コトダマ・リフォージ 〜再燃の塔と熟成の宝箱〜』

### 3.1 統合方針

ベースシャーシは総合1位の**スミス**(想起=コアバーブ、決定論燃料×乗算増幅、6分ラン、レアリティ=記憶安定性)。そこに審査で指摘された弱点を、他案の実証済みパーツで1つずつ潰す。

| スミスへの審査指摘 | 移植するパーツ | 出典 |
|---|---|---|
| 選択肢即時提示で純粋想起ウィンドウがない | 「想起の間」2秒+見切りベット(選択肢提示前の確信宣言) | サバイバーズ/ハント |
| 燃料∝経過間隔の単調増加→放置farm誘発 | R値の山型「再燃バースト」カーブ | リコレクト |
| 新出単語の第1間隔が弱い | 「翌日まで開かない熟成宝箱」=新出導入装置 | 言ノ刃 |
| レアリティ降格=損失回避の離脱トリガー | 床値保証+降格幅圧縮+「亡霊からの奪還」イベント化 | ハント/言ノ刃 |
| ソロ運営のコンテンツ供給切れ | 宿敵(自分の忘却履歴から自動合成される無限ボス) | ハント |
| 4択まぐれ正解のSRS汚染 | 「パス」の期待値設計(山勘<正直なパス<挑戦) | サバイバーズ |
| ストリーク切れ→離脱 | 野営式ストリーク(0リセットなし、復帰追い風バフ) | 言ノ刃 |
| iOS SafariのIndexedDB追い出しでデータ全損 | 軽量クラウド同期を初期リリース必須要件に格上げ | 審査指摘 |
| 類義語セット編成は記憶干渉を起こす | セットボーナスは語根(形態)ベースに限定 | 審査指摘(Tinkham 1993) |

### 3.2 コアループ(時系列)

**1ラン=6分、塔1区画(通常ノード4+エリート1+ボス)。完全ターン制・時間無制限・全操作オートセーブ。**

- **0:00–0:15 入炉。** 起動即「出発」タップ→開始鍛冶具3択(例:「動詞の鎚」「re-のふいご」)。最初の操作がビルド方針の意思決定。
- **0:15–4:30 ノード攻略(40〜90秒×5)。** 手札5枚の言霊カードを1枚ずつ処理する。1枚の流れ:
  1. **想起の間(2.0秒):** カードをタップすると英単語(または日本語/音声)だけが表示される。選択肢はまだ出ない。ここで**上スワイプ=「見切り!」**(確信宣言)、**下スワイプ=「パス」**(正解開示+ラン内再挑戦予約、ペナルティなし)。
  2. **フリック想起:** 2秒後、4方向に意味タイルが扇状展開。正解方向へフリックで火花+チップ加算音。
  3. **燃料加算(決定論):** 正解=基礎燃料を必ず満額獲得。遅く答えても満額(時間圧は報酬側のみ)。見切り宣言+正解=倍率ボーナス。山勘誤答=バースト権喪失。**期待値上、「分かりそうなら見切り>普通に挑戦>正直なパス>山勘」が成立**し、25%まぐれ正解のSRS汚染を抑える。
  4. **再燃バースト(山型):** その単語のFSRS想起確率Rに応じた倍率が乗る(後述の式)。「忘れかけの単語を思い出した瞬間」が最大火力=最大ドロップ。
  - 誤答語は「暴走」し、同ノード終盤に易形式で再挑戦が割り込む(ラン内遅延再テスト)。鍛冶具群が燃料を乗算増幅し、規定スコア突破で火柱+桁上がり。
- **ノード間(各10秒):** 報酬3択——(a)鍛冶具、(b)**熟成宝箱**(新出単語。開封=初回想起は**翌日以降のみ可能**。今日の渇望を明日に持ち越すアポイントメント機構=第1間隔)、(c)強化素材。
- **4:30–5:30 ボス「忘却獣」:** 自分の忘れかけ単語の集合体。週1回は**宿敵**(直近の誤答上位10語が合成された個人専用ボス、討伐で確定レア)。
- **5:30–6:00 リザルト:** 敗因単語トップ3(「この3体を鎮めれば突破できた」)、言霊のランクアップ演出(錆→銀→金)、アンロックゲージ(累計想起数駆動=敗北日もゼロにならない)、「明日開く宝箱2個」「焼入れ+1で再挑戦」を提示して閉じさせる。

**日次の枠:** デイリー炉(シード固定ラン)+朝の「鑑定タイム」(昨日の熟成宝箱の開封=新出の初回想起)。due消化の隠しクォータ超過後は安定語のみの「巡礼モード」に切替え、過学習とdue渋滞を防ぐ。ストリークは野営式(途切れても食料が減るだけ、復帰時は追い風バフ)。

### 3.3 SRSアルゴリズム実装方針

- **エンジン:** `ts-fsrs`(MIT)をそのまま採用。自作しない。`desired_retention = 0.90` から開始。カードごとに Stability / Difficulty / 最終レビュー日時 / 状態+レビューログをIndexedDBに永続化。
- **評価のマッピング(2値+補助情報→Rating):**
  - 誤答 → `Again`
  - ヒント購入後の正解 → `Hard`
  - 通常正解 → `Good`
  - 見切り宣言+正解 → `Easy`
  - パス → `Again` 扱い(ただしゲーム側ではペナルティなし=「正直な申告」を保護)
  - 形式重み: 4択正解はチャンク組み立て正解より弱い成功としてログに記録し、低重み形式の連続成功だけでは形式昇格させない。
- **再燃バースト(山型カーブ):** 出題時点のRを `ts-fsrs` から算出し、倍率を
  `burst(R) = 1 + 2.5 × exp( −(R − 0.85)² / (2 × 0.08²) )`
  とする(R≈0.85でピーク約3.5倍、R>0.97の直近語はほぼ1倍、R<0.6の放置語も逓減)。**忘却への単調報酬を排除し、「効率厨の最適周回=FSRSの最適復習タイミング」を曲線の形状で強制一致させる。** ラプス時のレアリティ降格は1段まで+床値保証(言ノ刃方式)。
- **配牌(不可視リグ):** ノード手札は R昇順のdue語 約55% / 新出(熟成宝箱経由のみ)20% / 安定語(演出用)25%以下。出題選択権はプレイヤーに渡さない。1日の新出数は復習負債量に応じて自動絞り込み。
- **出題形式の自動昇格:** 英→日4択(想起の間つき)→ 日→英4択 → 音声→意味 → 音節チャンク組み立て(タップのみの産出近似)。昇格は「進化の試練」として演出。ディストラクタは本人の所持語+混同履歴(confusion matrix)から生成し、**初期は人手検収済みディストラクタ銀行(1,000語分)を必須工数として予算化**する。
- **データ保全(審査指摘の必須対応):** iOS SafariはPWA未インストール時にIndexedDBを追い出しうるため、**匿名IDによる軽量同期(Cloudflare Workers + D1/KV、レビューログの追記同期のみ)を初期リリース必須**とする。`navigator.storage.persist()` も要求する。
- **音声:** Web Speech APIに依存せず、**事前生成TTSをCDN配信**(初期1,000語分)。iOSの`speechSynthesis`は品質・挙動が不安定。
- **KPI:** D1/D7/D30に加えて**「7日後・30日後の遅延再テスト正答率」**と「火力寄与率(想起:鍛冶具増幅=6:4目安)」をテレメトリで常時計測。これがない限りエンゲージメント最適化圧に必ず屈する。

### 3.4 MVPスコープ(最初に作るもの)

**Phase 0(2〜3週)— データパイプラインこそ本体:**
- 語彙1,000語(NGSL上位帯)。語義・例文(Tatoeba由来)・事前生成TTS・人手検収済みディストラクタ・品詞/接頭辞/語根タグ。ライセンス確認込み。
- `ts-fsrs` 統合のクイズエンジン単体(想起の間→フリック→Rating記録)を最初に作り、自分で1週間使う。

**Phase 1(6〜8週)— コアラン:**
- 塔1区画(ノード5+ボス)、出題3形式(英→日/日→英/ボスのみチャンク組み立て)、鍛冶具20種、再燃バースト、熟成宝箱、暴走再挑戦、リザルト(敗因トップ3)。
- ジュースは「加算音・火柱・桁上がり」の3点に集中投資(ここがチープだと算数ドリルに堕ちる——演出予算の最優先項目)。
- PWA(オフライン動作+同期)、縦持ち片手、タップ/スワイプ/ドラッグのみ。

**Phase 2(4週)— 継続装置:**
- デイリー炉(日付シード)、野営ストリーク、図鑑(頻度帯×レアリティのマトリクス)、焼入れLv0〜3、巡礼モード。

**v1.1以降(運用):** 宿敵ボス、武器種=形式分担(ハント方式)、語彙帯追加(2,000語→5,000語)、焼入れLv4〜10、成果モード(CEFR/TOEIC網羅率の読み替え表示——「勉強と気づかれない」ことの副作用への二枚看板)。

**作らないもの:** リアルタイム戦闘・弾幕(サバイバーズの失敗土俵)、オートバトラーの僅差バランス(バザールの失敗土俵)、シーズン制リセット運営(ソロでは息切れ確定)、固有キャラアート量産(リコレクトの失敗土俵)、確率課金。

## 結語

本コンペの最大の収穫は、勝敗を分けたのが「アイデアの派手さ」ではなく**①想起をコアバーブに据える規律、②報酬曲線の形状、③ソロ開発で自己再生するコンテンツ装置**という3つの構造判断だったことだ。ハイブリッド案『コトダマ・リフォージ』はスミスのシャーシにリコレクトの山型カーブ、言ノ刃の熟成宝箱、ハントの宿敵、サバイバーズの想起の間を移植し、審査が挙げた弱点を個別に潰してある。最初の一歩は派手な戦闘でも演出でもなく、**Phase 0のクイズエンジン+1,000語データパイプライン**である。ここを誠実に作れば、残りはすべてその上に積める。
