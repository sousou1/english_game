## 0. 統合裁定 — 3設計の矛盾と確定判断

3設計は経済リスキン方針・フィール規則の解釈で一致しており、衝突は以下の12点のみ。各裁定が本仕様の正本。

| # | 論点 | UI案 | バトル案 | 演出案 | **裁定** |
|---|---|---|---|---|---|
| 1 | フィーバー突入 | 手動点火`[⚡解放]`+満タン30秒で減衰 | ゲージ25で自動突入 | — | **手動点火**。プレイヤー起点原則に最も忠実で、30秒保持減衰が「本物の賭け」になる。バトル案の1秒リーチ演出は「満タン到達時」の予告に転用 |
| 2 | フィーバー時間 | 15秒固定×3 | 延長型: 基本10秒+正解+0.4秒(上限18秒) | — | **延長型**。「正解し続ける限り炎が消えない」がPO要件「時間制限で連続プレイ」の本体。平均≈14秒で経済影響はUI案とほぼ等価 |
| 3 | チェーン構造 | フィーバー中15正解で自動連鎖 ×3→×4→×5 | 終了後45秒窓で再突入 ×3.0→×4.0、残り火G=6 | 余韻15秒・閾値25→10 | **3案を合成**: 終了後45秒連鎖窓+残り火G=6+余韻15秒は正解+2(6+10正解×2=26≥25、すなわち演出案の「10連でもう一度」と数学的に等価)。倍率はシム検算済みの ×3.0→×4.0 を採用(×5は不採用) |
| 4 | 雑魚の反撃 | 雑魚12秒/ボス8秒で常時攻撃 | 雑魚は無反撃。被ダメはプレイヤー起点の討伐チャンスのみ | 時間経過ダメージは本物の時限バトルのみ | **バトル案**。メイン詠唱ループへの常時時間圧は「焦り学習」を誘発しSRS隔離の精神に反する。シート展開時のテレグラフ凍結処理も不要になり実装が軽い |

| 7 | 修行(SRS)の置き場 | 100%フルシート「焚き火モード」で物理隔離 | インラインカード現行維持 | 起こす展開中は演出抑制 | **UI案**。「同じ画面に敵がいない」物理隔離がSRS汚染禁止の最強保証。カードコンポーネント(遅延選択肢・ピンときた・時間無制限・1窓8〜12体)は**無改造**で焚き火内に移設。演出案の抑制規則は「焚き火中はfxLayer全停止」として吸収 |
| 8 | プレイヤーHP表示 | ヘッダ常設 | — | — | **討伐チャンス中のみステージ内表示**。裁定4により戦闘外HPは常に満タン=常設表示は死に情報。空いたヘッダ枠に魔素✨とゴールド💰の2通貨を並べる |
| 9 | 演出の描画領域 | y411より上+画面端オーラのみ | — | 全画面#fxLayer | **両立**: #fxLayerは全画面・`pointer-events:none`のレイヤーだが、**描画(被せ物)はy411より上+画面端8pxオーラに限定**。タイル枠の発光・色変化は「状態スタイル」であり被せ物に当たらず可 |
| 10 | リーチ発火条件 | 敵HP残10% | — | 残り≤タップ価値1〜3撃分 | **残りHP ≤ V_ref×3** に統一。「あと数撃で本当に倒せる」という実事実ベース(%は嘘をつきうる) |
| 11 | メニュー構成 | 5スロット(武器屋に施設込み) | 4項目(村=施設が独立) | — | 裁定6に従い**6スロット**: 📜物語 / ⚔武器屋 / 🏘拠点 / 📖呪文書 / 🔔修行 / ⚙設定(各65×56px、44px最小を満たす) |
| 12 | 復活演出 | — | — | 章ボスの「二度立つ」(事前種明かしの実フェーズ) | 採用するが**章ボスのみ・Phase 6オプション**。偽ハズレ禁止は全員一致で維持 |

不変条件(3設計一致、本仕様の全節に優先):
- **B1** 敵HP=`orderTarget(n)`の完全リスキン。式・数値・bigOrder×3は一切変更しない(討伐4〜6体/日のペーシング継承)
- **B2** ダメージ=タップ獲得`gain`そのもの。「攻撃=収入」なので敗北しても構造的に損失ゼロ
- **B3** 誤答は被ダメ・敵ゲージ・SRSに一切接続しない。罰は現行どおりコンボ切れ+0.4秒ロックのみ
- **B5** Lv/EXPは攻撃力に乗らない(HP・会心のみ)。火力は語彙経済が独占=「学習が火力、プレイ量は生存力」
- **B6** 討伐数(解禁クロック)は手動タップでのみ進む。使い魔(auto)は敵HPを削らない
- **B7** バトル層・演出層はSRS(`srs.js`/`schedule.js`)に1バイトも書き込まない
- **B8** 偽タイマー・偽抽選・偽ハズレ・LDW禁止/16ms応答/描画後300ms入力ガード/全画面割込み禁止/スクロールゼロ/CSS+絵文字+パーティクルのみ

### 1.2 入力規則(監査17件の継承・確定)

1. 全タップ要素はpointerdownで16ms以内にpressed(scale 0.96)

## 2. バトル・武器・ステータスの確定数式(既存コードへの差分)

### 2.1 敵HP(pool.js: **式の変更ゼロ**、参照名の追加のみ)

```
HP_enemy(k) = orderTarget(k)                       … 現行コード(pool.js L98-102)そのまま
  = 30 × 1.35^(k−1)                    (k ≤ 40)
  = W(40) × 1.13^(k−40)                (k > 40)
  × 3      if k % 5 == 0               (中ボス = 現行bigOrder)
  × 5/3    if k ∈ [20,50,90,140]       (章ボス: 中ボスと重なり計×5) ← 唯一の追加
```

- 常時1体が画面に居る。撃破→300msスポーン演出(入力ガード)→次の個体。敵不在時間ゼロ
- 雑魚・結界へのダメージはセッションを跨いで永続(=`order.got`の現行挙動)。タイマーなし
- 中ボスの現行「語パック縛り」は**弱点属性**と呼び替え(判定・数値無改造)
- ロスター: 章1(村)🐀🦇🕷👺+中ボス🦌+章ボス🐉(20体目)/章2(都会)🐍🃏🗡(50体目)/90・140体目はシナリオWF確定待ち(フック: `onChapterClear(c, choice)`)

### 2.2 ダメージ=獲得(pool.js `wordValue`への乗算1個の追加のみ)

```
DMG = tapBase × tierMult[rank] × milestoneMult^(語マイルストーン) × globalMult(コレクション×施設)
      × K_weapon                                  … 新規。武器倍率の累積積(§2.4)
      × comboMult × feverMult(連鎖込み) × 鮮度(×1.5/×1.2) × crit(×2 or ×1)

獲得魔素   = DMG     (現行gainの意味を変えない。攻撃=収入)
敵HP進捗  += DMG     (手動タップのみ。使い魔autoは乗らない=現行どおり)
```

`p.vref`(gain/comboMultの移動平均)はK_weaponを自動内包するため、放置生産 `CpM = V_ref × IDLE_K × log2(1+P)` は無改造で武器に追従し、log2圧縮で従属が保たれる(I2「1タップ=放置14〜70秒」不変)。

### 2.3 討伐報酬(経済差分はここだけ)

```
ゴールド G(k) = round(HP_enemy(k) × 0.5)    … 現行の灯火一時金(0.5W)を全額ゴールドに置換
EXP           = +10(雑魚)/+150(中ボス・章ボス)
```

注文ペースはゲージ充填=タップ収入で決まるため不変。施設購入ペースのみ微減 → **補正ノブ: Day7シムで遅れたら IDLE_K 0.21→0.24**。

### 2.5 プレイヤーステータス(Lv/EXP/HP/会心)

```
EXP:  タップ+1 / 想起+15 / 見切り+25 / 卒業+40 / 招く+10 / 雑魚+10 / ボス+150 (≈2,300/日)
Lv:   次LvまでN(L)=round(100×L^1.4)   → Day1=Lv5 / Day7=Lv12 / Day30=Lv22 / Day90=Lv35
HPmax = 80 + 20×Lv                      (討伐チャンスでのみ意味を持つ。戦闘外は常に満タン)
会心率 = 4% + 0.2%×Lv(Lv由来上限+6%)+ 武器特性(総上限20%)、会心倍率×2(獲得も×2)
攻撃力表示⚔ = p.vref(新規ステータスではなく既存値の表示)
レベルアップ: HP+20・会心+0.2%・全回復(ボス戦中なら回復演出=逆転の高揚)
```

会心の経済影響 ≈ 平均×1.05〜1.2 → **ACTIVE_MULT想定 1.8→1.9〜2.0 に補正(シムノブ)**。

### 2.6 討伐チャンス(被ダメと敗北。中ボス・章ボスのみ、雑魚は無反撃)

```
ボスHP   = 結界75% + 本体25%
結界:     通常プレイで削る(タイマーなし、永続=現行大注文と同一)
本体:     結界破壊→「⚔討伐チャンスに挑む」点灯(プレイヤー起点でのみ開始)
本体HP   = 0.25 × HP_enemy(boss)
ボス攻撃 = 5秒ごと(章ボス4.5秒)に ATK(b) = round(HPMODEL(b)/13)
HPMODEL(b) = 80 + 20×(55.4×max(1,b−1))^(1/2.4)   … 順調プレイヤーが13発≈65秒生存する正規化
開始時:   プレイヤーHP全回復(回復薬グラインドを作らない)
必要DPS ≈ 順調ペースV×0.8(ラッシュ込)/×1.7(ラッシュなし) → ゲージ満タンで挑むのが定石
```

**敗北=撤退(没収ゼロの証明)**:
- HP0 or 🏳撤退 → 「体勢を立て直した」。戦闘中の与ダメ=獲得魔素は既に手元(B2)。結界は壊れたまま。魔素・ゴールド・SRS・コンボ資産すべて不変
- 撤退1回ごとに本体HP上限−5%(累積上限−30%)=「竜も傷つく」。負けても前進が見える
- 撤退時に**敗因表示**: 「この3語を速く詠唱できれば倒せた」(直近戦闘で詠唱に最も時間を要した実3語)
- 再挑戦は即時可。火力不足の正規解は学習(新語+復習で翌日V_ref×1.9 → 「寝て、復習して、翌朝竜を斬る」)

## 3. 時限フィーバー『詠唱ラッシュ』確定仕様(裁定1〜3の統合形)

PO要件「時間制限で連続プレイさせる」の本体。**表示するタイマーは全て実害・実益のある本物のみ**。

1. **充填**: ゲージG∈[0,25]。正解+1/誤答−3/無操作1秒−1(灯し場アクティブ中のみ)。誤答−3は監査済み規定値(ゲージは貯蓄でなく勢い=没収に当たらない)
2. **満タン**: `[⚡解放]`が脈動出現(C帯右端、300msガード)+1秒のリーチ演出(画面縁発光、入力は生きたまま)。**30秒以内に点火しないと−1/秒で減衰**(燃え尽きる炎アニメで常時可視=本物の賭け)
3. **点火(手動・プレイヤー起点)**: キュイン音+魔法陣展開500ms → **基本10秒、正解1回ごと+0.4秒、上限18秒**(武器3で21秒)。倍率 **×3×連鎖補正** がタップ価値・与ダメに乗算。テンポ×1.5。タイマーは敵頭上の魔法陣リング(conic-gradient、rAF連続減少)+C帯バーの2箇所で常時駆動。ラッシュ中の正解はゲージに乗らない
4. **連鎖**: 終了から**45秒以内**に次の点火で連鎖+1: **×3.0→×3.25→×3.5→×3.75→×4.0(上限)**。45秒経過で×3.0にリセット。チェーン失敗のペナルティなし(獲得分は全部残る)
5. **残り火と余韻**: 終了時G=6から再スタート+**余韻15秒間は正解+2でゲージが貯まる**(銀リング+「余韻 — あと10詠唱でもう一度」表示)。余韻内なら10正解で再満タン(6+20=26)=演出案「時短」の実装形。余韻後は+1に戻り、45秒窓には19正解≈27秒でぎりぎり届く緊張感
6. **精算(1.6秒、タップで早送り、表示直後300msガード)**: ①獲得魔素のロールカウント+吸い込み粒子 ②最大コンボ/会心数スタンプ ③「連鎖継続 ×3.25 — あと45秒」のリアルタイムゲージ点火(これが導線) ④プール内パネルで完結、全画面モーダル禁止。**明細なしの花火だけ、を禁止**
7. **経済**: サイクル≈充填20〜27秒+ラッシュ平均14秒 → 稼働率≈34%。調整ノブ: emberG(6→0)/baseMs(10s→8s)/afterglowGain(2→1)

検証指標: ラッシュ終了→余韻中の再突入試行率 >70%(PO要件の実測)。
