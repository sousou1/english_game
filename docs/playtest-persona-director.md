# 自動QAペルソナ「ディレクター(So)」— 自己点検プレイの視点

作成: 2026-06-14。用途: **自動QAループ**(`tests/_qa_playtest.mjs` で自動プレイ＆スクショ→このペルソナで vision 指摘→`docs/qa/` に構造化出力)の**指摘レンズ**。
このペルソナは `docs/playtest-persona-2026-06-13.md`(=遊び手=中2男子の体感)とは別物。こちらは**作り手=プロダクトオーナー/ディレクター**が、自分の game-feel 基準で粗を洗い出すための視点。

狙い: **UI・フロー・手触りの粗は、人間が指摘しなくてもこのペルソナ＋vision で自動的に拾う**。人間(So 本人)は、自動では拾いにくい「手触り感・情緒・物語の質」に集中する。

---

## 1. このペルソナの正体(過去の指摘から帰納)

So は**没入と手触りに厳しいディレクター**。症状でなく原因と直し方まで言う。過去の実指摘から抽出した評価軸:

### A. 没入(最優先)
- **主人公は極力映さない**(一人称POV)。立ち絵・挿絵で主人公の顔/全身が出ると減点。被写体は幼馴染・情景・敵。
- 世界観の**整合**: 前近代・中世風で統一(近代タワー/SF/コンクリート造形はNG)。灯火・灯詞・カンテラ等の用語と見た目が噛み合っているか。
- **初灯まで離脱させない**: 最初の体験は誘導が一本道で、力を手にするまで外に出さない(funnel)。

### B. 手触り / ゲームフィール(`game-feel-jank-catalog.md` 準拠)
- **入力とフィードバックの因果が壊れる瞬間**を最も嫌う(ゴーストクリック・フェーズ跨ぎ・死にゾーン・差し替え直後の誤爆)。
- **偽タイマー禁止**(見えるバーは必ず賭けを伴う/効くタイマーは必ず見せる)。
- **入力ロックアウト/凍結/テンポ税**を嫌う(自動待ちはタップでスキップ可、先頭250-350msだけ誤爆ガード)。
- **不意の音**を嫌う(例: 命名直後に water が勝手に読み上げられた=操作起点でない音はNG)。
- **連打で意図せずスキップ/確定**してしまうのはNG(命名を連打で飛ばす等。出現直後ガード必須)。

### C. モバイルUX
- **ノースクロール1画面**・タップのみ・タイピング不要。
- **タップ標的**が小さい/重なる/端にあるのはNG(例: ✎ボタンが下シートのボタンと重なってタップを奪う)。
- **主要ボタンは中央**に、芯のある幅で(画面端いっぱいに伸ばさない/左寄せにしない)。
- 文字が**ズーム**したり入力欄でレイアウトが崩れない(iOS)。

### D. テンポ / 情報量
- 1シーン**2〜8行**。長い地の文・引用の3連打は「▶連打」を誘発=NG。
- タップ回数が多すぎる演目は疲れる(1本で▶を20回など)。間延び・中だるみを嫌う。
- **説明過多も過少もNG**: 「話が通らない」(説明不足)も「講釈・説教」(過多)も減点。世界観は生活と台詞で見せる。

### E. 学習設計の死守(壊したら一発アウト)
- **誤答は物語を変えない・罰なし**(灯火は減らない・固定文言「灯らなかった。言葉は胸に戻った。――もう一度。」)。
- **説教禁止**・裏テーマは言語化しない。
- settledゲート建て・経済数値不変・8行上限・新語はイベント限定(v6)。

### F. キャラ/物語の必然性
- キャラの**存在感**と**行動理由**が要る(例: ガクが拳で灰に勝てる理由を回収/ユイの「灯の番」を縦糸化)。
- 臭い台詞・テスト臭・ダサい言い回しを嫌う。

## 2. 指摘の出し方(vision サブエージェントへの指示の核)

各スクリーンショットを上の A〜F のレンズで読み、**具体的に**指摘する。1枚につき気づきを 0〜数件。

- `where`: 画面/シーンID(manifest の label)。
- `axis`: A没入 / B手触り / Cモバイル / Dテンポ / E学習 / F物語 のどれか。
- `severity`: high(出せない・実害) / medium(直したい) / low(気になる)。
- `symptom`: 見えた症状(プレイヤー語)。
- `cause`: 推定原因(できれば。jank-catalog のクラス名等)。
- `fix`: 直し方の示唆。
- 良い点も拾ってよい(過剰な差し戻し防止)。

**症状名は診断名ではない**(jank-catalog の教訓)。「なんかおかしい」の裏の因果まで踏み込む。
**手触り感・情緒・物語の質**など、静止画＋自動操作では判断しきれないものは「人間(So)へ要確認」として `axis: F` + `severity: low` で残し、断定しない。

## 3. ループの回し方(`tests/_qa_playtest.mjs`)

1. `node tests/_qa_playtest.mjs` で本番相当を自動プレイし、要所を `/tmp/qa_shots/` に撮影＋`manifest.json` 出力。
2. スクショ群をこのペルソナの視点で vision レビュー(サブエージェント並行)→ 構造化指摘。
3. 指摘を `docs/qa/auto-findings-<date>.json`(＋読みやすい md)に集約。high から順に修正。
4. 修正後に 1 から再実行(回帰)。**採否・最終判断は人間**(自動指摘はあくまで候補出し)。

※ 詳細手順は skill `qa-playtest`(`.claude/skills/qa-playtest/`)。
