# ペルソナ・プレイテスト(2026-06-13 / v14)

ペルソナ「中2男子・ゲーム好き・英語苦手寄り・長文は飛ばしがち」になりきった通しプレイの体感レポート。
実プレイは playwright で v14 本番相当(localhost:8347)を駆動。スクショ /tmp/persona_*.png。
※ 自動操作+ペルソナ視点の合議レポート。前任エージェントが操作スクショまで残し、本レポートは
そのスクショ群とデータ静的集計をもとに作成。

## 概要(到達点)
- チュートリアル3語(water/apple/sun)→ 詠唱バトル → 第1章シーン読み → イベント6本(白い火の夜〜
  夜空への誓い)まで通しで到達。Lv4・討伐2体・所持金3.9万・語彙51語まで進んだ。

## 良かった点(ペルソナの本音)
- **複数語ビートが「ちょっと頭使う」感じで良い**。空欄が2つ並んで、左を埋めると緑になって、使った
  選択肢が薄くなる(グレーアウト)。「お、2個セットなのね、で左から、はいはい」とリズムに乗れる。
  (multiword スクショ: 「あの星よ、村まで続く夜の空に…」star→sky の順、star選択後グレーアウト)
- **クリア画面のごほうびが渋滞してて嬉しい**。「💰+6,200 / 武器SSR / ⚡ことだまが満ちた / 📖あたらしい
  言葉×8」が一気に出る。「SSRきた!ラッシュも溜まった!8語ゲット!」と派手で、中2は声出る。
- **会話に丸い顔アイコンが付くようになった**のは地味に効く。「だれが喋ってるか一瞬でわかる」。レンと
  ノノの掛け合いがLINEっぽくてテンポよく読める。
- バトルの「数字がドカン(-100)」+タイル光るのは原神/パズル系に慣れた手に馴染む。

## だるい・つまらないと感じた瞬間
- **イベント前の地の文が、飛ばしたくなる長さのものがある**。「写本の手がかり」シーンとか、紙の引用が
  3連続で出ると「うっ、文字…」となって▶を連打しがち。中2は2〜3行で次のボタンが欲しい。
- **シーン解放の『9必要 ― 魔物を倒そう』ゲートで一回手が止まる**。「え、読み進めたいのに戦えって
  こと?」と文脈が切れる。バトルに戻る導線がもう一押し欲しい。
- イベントを連続でやると **intro→lines→teach→cast の「つづける」タップが多い**。1本で15〜20回くらい
  ▶を押す。中身は良いけど「タップ回数で疲れる」瞬間がある。

## バグ・違和感リスト
- **(軽微)立ち絵は『normal』表情固定**。por_nono_smile/determined 等の差分が webp にあるのに、
  会話の感情に関わらず常に normal。「ノノいつも真顔だな」と気づく余地あり(改善提案2)。
- **(軽微)シーンによっては挿絵なし**で真っ黒の上部ステージが寂しく見える瞬間がある(挿絵未割当の
  c01_030/060/130/150)。絵が出る回と出ない回の落差。
- **(仕様確認)呪文書の「あたらしい言葉は、物語のイベントで手に入る。」案内は明快**。「招くボタンが
  消えた」ことへの不満は出なかった——むしろ「イベント進めれば増えるのね」と素直に納得。新語供給の
  動機づけとしてイベントが機能している。
- クリティカルなバグ・操作不能・進行不能・誤字は今回の通しでは検出されず。テスト73+プローブ13グリーン。

## 改善提案トップ5(優先度順)
1. **イベント間・シーンの地の文を「2〜3行で1ボタン」に刻む or 長い引用を畳む**。飛ばし読み対策。
   特に手がかり系シーンの3連引用は1画面に詰めず段階送りに。
2. **立ち絵の表情差分を会話に連動**。セリフ末の感情(! → surprised/determined、…→ sad/worry、
   笑い → smile)で por_*_表情 を出し分け。アセットは既に webp_ready にある。話者検出ロジック
   (ui.js PORTRAITS)を「名前+表情キー」に拡張するだけ。
3. **シーン解放ゲート(『N体倒そう』)からバトルへの導線を強化**。ゲートボタンを押したら自動で
   バトル画面へ戻す/「あとN体」をバナーで常時表示。
4. **tier2イベントの語数を8→6に下げる検討**(感情語は重い)。難易度シミュ(docs/v6-multiword-spec.md)
   上、c02ゲートは settled40 で供給48→44でもバッファあり。要・再シミュ。
5. **挿絵未割当シーンに wide_* ストックを割り当て**。webp_ready の引き構図(wide_capital_panorama
   等)で上部ステージの「黒余白」を埋め、絵が出る/出ないの落差を減らす。※絵の採否は人間レビュー。
