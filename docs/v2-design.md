## 2. 確定コンセプト

### 2.1 世界観

ことばが「忘却の靄(もや)」に呑まれ、人々が何も話せなくなった世界。プレイヤーは廃墟の工房で目を覚ました「灯(とも)し手」。唯一できることは「思い出す」こと。

思い出された英単語は「言霊(ことだま)」という小さな働き手として工房に住み着き、**灯火(資源)を生産して靄を押し返す**。言霊が深く記憶に根づくほど(=FSRSのStabilityが伸びるほど)**長く独りで働けるようになり**、工房は村に、村は町になり、物言わぬ人々が一語ずつ言葉を取り戻していく。

### 2.2 核となる転写(このゲームの契約)

> **単語を覚える = その言霊が世話なしで働ける期間が伸びる = 放置効率が上がる。この3つは同一の行為である。**

- 中核資源(灯火)と生産能力を生むのは**想起だけ**。放置生産はすべて「過去の想起の利息」。
- 放置ゲー最大の快感「手動→自動化」を「記憶の自立」に直訳する。星鋼(S≥90日)の言霊は90日以上独りで働く=実質自動化。ただし**出題からは絶対に卒業しない**(§5.6)。
- 忘却=失うことではなく「再燃バーストの充電」。これはsrs.jsの数式上の事実であり、捏造された放置ボーナスは一切ない。

### 2.3 物語の構造

- 物語は**背骨ではなくスパイス**。背骨は昇進・経済・注文の生成系。物語が尽きてもゲームが回ることを実装段階で確認する。
- 配給チャネル: ①見出しの無言の名詞置換 ②アクティブ中3〜6分毎のランダムイベント1行 ③閾値到達の一行。手書きは約200イベントで有限と割り切る。
- 固有の叙述装置: **プレイヤーが実際に覚えた語だけが、この世界で発話される**。住人のセリフに学習済みの語が再登場し、黄金以上の語は英語表記のまま現れる。
- 終盤の遅延開示(ネタバレ厳重管理): 靄の正体は「人類が捨てた言葉たちの澱(おり)」であり、灯し手自身が最後のひとつの言霊——『remember』——だったと明かされる。全行動が一斉に再解釈される。

### 2.4 トーンと文体規則

- 静かで温かい。詩的になりすぎない(灯守への15歳の指摘)。1文は短く。
- **難読漢字は使わない**。使う場合は必ず `<ruby>` でルビ(設定でON/OFF、デフォルトON)。
- 主要用語集(これ以外の独自用語を増やさない): 言霊(ことだま)/灯火(ともしび)/靄(もや)/うとうと/起こす/招く/鐘(かね)/たね火/明日ひらく宝箱/ピンときた

## 4. コアループ仕様

### 4.1 在席時(1セッション = 90秒〜3分。3分で自然に切れることを設計目標とする)

```
開く → (1)精算 → (2)起こす → (3)経営 → (4)招く(任意) → (5)閉じ際 → 閉じる
```

**(1) 精算(0〜10秒)**: 起動した瞬間、ログに帰還台帳が流れる(モーダルではない)。
「留守のあいだに灯火 +8,420(棚いっぱいで停止: 23分前)。3体がうとうとしている——起こせば、ためた力が弾ける。」
必ず**正の報酬が先頭**。回収はワンタップ。

**(2) 起こす(60〜120秒)**: うとうと言霊(=FSRS期日到来語、鐘で量子化)を1体ずつ§3.3のカードで想起。**1窓あたりの提示上限8〜12体**、超過分は「次の鐘で」(セッションを3分以内に切るための上限)。正解→熟成マナ一括放出+再燃バースト+自立期間の伸長表示「この子は次、7日間ひとりで働けます」。

**(3) 経営(20〜40秒)**: 灯火で施設を1〜2個購入。価格は等比×1.12で「買える→買えない→もうすぐ」の自動リズム。

**(4) 招く(任意、30〜60秒)**: 新語3候補(注文と設定レベル帯から)→1〜2体を選んで契約。初出は**提示のみ**(語+意味+例文+🔊)でクイズではない。同セッション内に学習ステップ1の再想起が自然にキューへ混ざる(§5.2)。

**(5) 閉じ際**: 「つぎの鐘は19時ごろ。いま起こしても、まだ眠りが浅い」+グレー予告の確認。覚えたて連打はburst式の裾で旨味ゼロなので、ビンジは報酬構造自体が止める。

### 4.2 放置中の進行式(すべて実時間タイムスタンプから決定論的に計算)

放置中に進むのは3レーン:

1. **灯火が貯まる**: 覚醒中の言霊が `rate(word)` で生産(§5.4)。棚(貯蔵上限)でキャップ。time-to-full=チェックバック間隔で、初期45〜60分→増築で2h→8h(就寝)→24hと**プレイヤー自身が購入して伸ばす**。溢れは機会逸失のみ。
2. **バーストが充電される**: Rが下がると言霊は「うとうと」し、生産が緩やかに低下(下限-40%)。低下分は熟成マナとして籠に積もり(上限あり§5.5)、想起成功で一括放出。**放置=力が増す、が忘却曲線の数式上の事実として成立する**(本作の心臓部)。
3. **世界が動く**: 注文が1〜2件積まれる。明日ひらく宝箱が熟す。イベントの種が積まれる。

**オフライン精算の実装規則**: `settle(fromTs, toTs)` を5分刻みのdelta-timeシミュレーションで実装し、**オンラインの毎フレーム計算と同一の関数**を使う(数字は嘘をつかない)。オフライン上限24時間。

**チェックバックの理想リズム**: FSRS期日を朝・昼・夜の3窓「鐘」に量子化(§5.3)し、初週は1日3〜4回×2〜3分→1ヶ月後は1日1〜2回。間隔の伸長=灯火庫の成長=語彙の安定化が三位一体で伸びる。

### 4.3 戻った時の精算体験(表示順を仕様として固定)

1. 灯火の回収(必ず正の報酬が先頭。完全明細: 停止時刻まで表示)
2. 「3体がうとうとしている——いま起こせば ×2.1で弾ける」(宿題ではなく**値上がりした収穫物**として提示)
3. 宝箱の開封(あれば)
4. ログ1行の物語イベント

**鉄則**: 精算は「工房が動いていた」ことだけを語る。**不在中に語彙が増えたと読める表現は1ピクセルも出さない**(教育的詐欺の禁止。文言レビュー必須項目)。

何日空けても**たね火は絶対に消えない**(野営式ストリーク継承)。溜まったうとうとは上限つき提示+超過分は自動で「冬眠」し(見た目の負債を隠す)、靄の大波(§6)の素材になる。復帰コストは常に負。

## 5. 忘却曲線との重なり — 数式レベル仕様

### 5.1 対応表(全対応が学習科学的に正直であることを設計原則とする)

| FSRS(srs.js実装) | ゲーム内 | 備考 |
|---|---|---|
| Stability S(想起でしか上がらない) | **職位** = 自立期間。「あと◯日ひとりで働ける」 | RARITY流用: 錆鉄=見習い(S<3)/青銅=職人(3≤S<10)/白銀=親方(10≤S<30)/黄金=長老(30≤S<90)/星鋼=語り部(S≥90) |
| R(t)=(1+(19/81)·t/S)^(-0.5) | **眠気**。R<0.9(=due到来)で「うとうと」 | `retrievability()` 無改造 |
| due(R≈0.9到達時刻) | うとうと開始時刻 → **次の鐘に切り上げ量子化** | 切り上げなので想起時R∈[0.85,0.90]=完熟窓に自然に入る |
| burst(R)=1+2.5·exp(−(R−0.85)²/(2·0.08²)) | **再燃バースト**(報酬倍率)。R=0.85で×3.50、R=0.99で×1.54、R=0.5で×1.00 | `burst()` 無改造。早起こしも放置しすぎも旨味ゼロ=効率プレイが最適間隔反復に一致 |
| 成長 g=13·((11−D)/10)·S^(−0.05)·(e^(1.8(1−R))−1)、easy×1.35 | 「忘れかけほど大きく育つ」の本体。**報酬式はこのgに比例させる** | `review()` 無改造 |
| lapse: S←max(0.3, 0.4√S)、postLapse=2(成長上限1.2) | 「靄に呑まれかけた」状態。4択まぐれ対策 | `postLapse` 無改造 |
| rating: 0=again/1=hard/2=good/3=easy | 誤答/遅い正答/正答/**ピンときた成功** | 自己申告なし。§5.7 |

### 5.2 学習ステップの追加(v2唯一のスケジューラ拡張。必須実装)

**根拠**: 無改造srs.jsでは初回good後 S=2.5日・20分後R=0.99935 のため、「Day1夜にもう一度出る」が成立しない(審査で検証済みの事実誤認)。嘘の演出で誤魔化すことを禁じ、コトダマ案の学習ステップを正式採用する。

新規ファイル `js/schedule.js`(srs.jsは無改造のままラップする):

```
新出語のライフサイクル:
  招く(提示のみ。クイズなし)
    → step1: due = now + 3分(同セッション内にキューへ混ざる)
    → step1成功 → step2: due = 次の鐘(実時間4〜8時間後)
    → step2成功 → 卒業: srs.review(newCard, rating) を初回実行(S確定、以後FSRS管轄)
  ステップ中の誤答 → step1へ戻る(罰なし、ログ1行)
  ステップ中の語は職位なし(「ひよこ」表示)。生産レートは見習いの50%
  ステップ中はburst・熟成マナの対象外(まぐれ・連打の入口を塞ぐ)
```

卒業時のrating: step2を「ピンときた」成功で通過=easy(S=7、**いきなり青銅**)/通常正答=good(S=2.5、錆鉄)。

### 5.3 鐘(期日の量子化)

```
quantize(due) = min{ w ∈ 窓時刻 | w ≥ due }   窓 = {7:00, 12:30, 19:30}(設定で変更可)
```

- SRSの期日が1日中バラバラに散る問題を、生活の3窓に量子化して解決する(コトダマ案の発明)。
- 切り上げ方向のみ(真のdueより早く出題しない)。切り上げによりR は0.9をわずかに下回り、完熟窓(burst×2〜3.5)に入る——量子化自体が報酬と整合する。
- 学習ステップ(step1の3分後)のみ例外で窓に縛られない。
- 通知はこの鐘の時刻のみ・世界内の文言のみ・**既定オフ**。

### 5.4 経済の基本式(初期値。週0シミュレータで較正する)

```
語の生産レート: rate(w) = base[職位] × clamp(R(w), 0.6, 1.0)   [灯火/分]
  base = { ひよこ:0.03, 見習い:0.05, 職人:0.2, 親方:0.6, 長老:1.5, 語り部:4.0 }
総レート = Σ rate(w)。資源行に実数値で常時表示
棚(貯蔵上限): 初期容量 ≈ 総レート×45分。増築ごとに容量×1.6
施設価格: cost(n) = base × 1.12^n(リピート購入)
```

- **減衰は必ずフロー(レート)のみ。ストック(灯火残高・施設・S・職位)は絶対に減らない**(Kittensの鉄則。全企画一致)。
- うとうとの下限-40%(clampの0.6)は、数百語規模で帰還時のレート低下が「サボった罰」に見えないための floor。プレイテストで罰と読まれたら0.7へ即緩和(廃人の指摘への保険)。

### 5.5 熟成マナ(減衰の収穫予約化)+ ビンジ対策

```
マナ蓄積: mana(w) += (base[職位] − rate(w)) dt
上限:     mana(w) ≤ base[職位] × 12時間分      ← 無限銀行化の禁止(科学者指摘)
放出:     想起成功時 報酬 = (基本灯火 + mana(w)) × burst(R) × 見切り係数
誤答時:   manaは保持(消えない)。放出されないだけ
```

「全部寝かせて週1ビンジ」は、①マナ上限 ②burstの山型(R<0.7で×1.0) ③フローの下限-40% の三重で期待値的に損になる。**シミュレータの回帰テストに不変条件として入れる**(§9.3)。

### 5.6 自動化の正直な実装(星鋼卒業の禁止)

- **どの語も出題から卒業しない**。S上限365日(実装済み)により、星鋼=語り部でも期日は90〜365日ごとに必ず来る。
- 「自動化」の体感は正直な事実で作る: 語り部は1年近く手がかからず最高レートで働く=実質放置。印刷機(Week1解禁)は「青銅以上の語の例文を頁として刷る」装置で、頁=恒久乗数素材+終盤の英語読み物(燈書館の頁資産を統合)。**生産の乗数は買えるが、記憶の代行は買えない**。

### 5.7 採点(「確かな想起」)とまぐれ対策

- 自己申告なし。rating自動算出: 見切り成功=easy(3)/正答=good(2)/誤答=again(0)。回答レイテンシは**非表示で**計測し、極端に遅い正答(>15秒)は成長gに×0.8の内部係数(UIには一切出さない)。
- **確かな想起** = 「due到来後(鐘の窓内)の正答」かつ「postLapse==0」。アンロック閾値・章ゲートはすべてこの「確かな想起数」と「定着語数(S≥3=青銅以上)」建て。当てずっぽう連打では何も進まない。
- 誤答への対処: postLapse(成長上限1.2、実装済み)+学習ステップ差し戻し+**固定ボーナスの上乗せ禁止**。誤答語の将来価値はS低下→早期再来→burst という数式の自然な結果のみ。

### 5.8 出題形式のエスカレーション(語別Stability建て)

`quiz.js` の `pickType()` を reps建てからS建てに改修:

| 職位(S) | 出題形式プール |
|---|---|
| ひよこ/見習い | e2j(英→意味) |
| 職人(S≥3) | + j2e(意味→英)、listen(🔊→意味) |
| 親方(S≥10) | + cloze(例文穴埋め)。j2e/clozeの重み増 |
| 長老/語り部(S≥30) | 産出方向(j2e/cloze)が主。セリフ内英語表記の受容反復が加わる |

新規語にいきなり高難度形式が来ない(燈書館の時代建ての欠陥を修正)。listen はTTS利用可能時のみ・音オフでも全進行が成立すること(15歳要件)。

## 7. ゲームフィール規則(絶対禁止リスト)

リリース前チェックリストとして運用し、**全UI変更でレビューする**。違反0件が出荷条件。

| 3 | **全画面割込み** | モーダル・画面遷移・強制演出ゼロ。誤答処理/昇進/イベント/精算はすべて同一カラム内のインライン展開とログ行 |
| 4 | **無応答タップ** | 全タップに16ms以内の視覚応答。押せないボタンは死なせず「あと灯火80」を即応表示。連打しても壊れない |
| 5 | **偽ゲージ・嘘の数字** | 全数値は実時間delta-timeの純関数。rAFで常時微動。満タン時は「止まって見える」のではなく「満杯」表示+次の一手を1行で提案。オフライン精算は完全明細(停止時刻まで表示) |
| 6 | **後退するストック** | 灯火残高・施設・S・職位・語数は絶対に減らない。失うのは機会のみ(棚の溢れ、完熟窓の通過、見切り失敗) |

| 10 | **誤答への固定ボーナス** | 誤答経由の期待報酬が正答継続を上回る経路を作らない(シミュレータ回帰テストの不変条件) |
| 11 | **読めない漢字** | 難読漢字禁止+ルビ既定ON。セッションは3分以内で自然に切れる構造を維持 |

### 9.2 「勉強臭が消えた」の判定基準(MVP合否)

**定量(テレメトリはローカル集計。主指標に学習KPIを必ず併置する):**

1. **3日目に自発的に1日3回以上開いたか**(鐘の通知オフのまま)
2. 誤答直後10秒以内の離脱率 < 10%(v1は誤答→全画面罰→離脱が頻発)
3. 報酬:要求の提示比 ≥ 1:1 をイベントログから実測(v1は1:17)
4. **学習KPI**: 定着語数(S≥3)の曲線が右肩上がり/2週間後の再走(靄の大波)正答率=本物の保持測定/黄金・星鋼数
5. セッション長の中央値 ≤ 3分、かつ1日合計 ≥ 8分

**定性(プレイテスト観察):**

1. 3秒テスト: 初見の画面を見て「教材だ」と言うか(言われたら見出し・動詞・第一画面を改稿)
2. 誤答時に嫌な顔をしたか(v1の合否基準を継承)
3. 「これ単語帳じゃん」発言の**タイミング**: 発言自体は不可避(15歳の前提)。発言後もその日のうちに再度開いたら合格
4. 精算文言が「溜まった宿題」に読めるか(読めたら文言改稿)

**フィール監査**: §7の11項目=違反0件(偽タイマー0・無応答タップ0・モーダル0・後退する数字0を含む)を出荷条件とする。

### 9.3 シミュレータの回帰テスト不変条件(S0で実装、以後CI相当で常時実行)

1. **誤答経由の累積期待報酬 < 正答継続の累積期待報酬**(全状態で。故意誤答ファーミングの根絶)
2. 早起こし(R>0.97)の期待利得 ≤ 鐘待ちの期待利得(ビンジ抑止)
3. 無限放置の期待利得 < 適期収穫(mana上限+burst裾+floorの三重防御の検算)
4. ストック量は全イベントで単調非減少
5. Day1台本の各アンロック到達時刻が目標±20%以内(想定正答率60/80/95%の3ペルソナで)
6. 学習ステップ込みで「Day1夜に初うとうとが必ず発生する」ことの数値証明
