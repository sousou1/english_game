## 2. 世界観（ゼロから再設計）

### 2.1 基本設定 ―「灯詞（ともしことば）」
この世界の物は、**正しい名（＝灯詞）を想起して呼ぶ**と応える。「sun」を本当に思い出して呼べば光が灯り、
「open」を思い出して呼べば扉が開く。**世界は、想起された灯詞だけを聞く**。日常の言葉（＝日本語表記）は
人同士の言葉で、世界には届かない。灯詞を本当に思い出せる者を **「灯詠み（ひよみ）」** と呼ぶ。
ほとんどの人は灯詞を“読める”が“灯せ”ない。主人公は灯せる、それも強い、稀な灯詠み。

### 2.2 灯詞の規則（5原則。作中で講釈せず、住人の台詞と現象で見せる）
1. **灯詞に文法はない**。語ひとつ＝灯ひとつ。覚えた語の数＝灯せる灯の数。
2. **力の本体は「意味の想起」**。音を出しても意味を思い出せなければ、ただの空気の震え（＝灯し場のマッチングそのもの）。
3. **深く根づいた語ほど強く応える**（＝Stability。職位＝言葉との絆）。
4. **溜め込んだ言葉は灰になる**。誰にも交わされず、一箇所に集められた灯詞は、内側から灰へ崩れる。
   ——丸暗記・独占が消える学習科学の世界法則化。**作中で講釈しない**。敵（集める翁／灰の手先）の灯詞が
   「灰を吐く・くすんでいる」描写でのみ見せる。
5. **誤りの世界処理**: 綴り違いの想起は暴発しない。「世界に聞こえなかった」だけ。
   固定文言 **「灯らなかった。言葉は胸に戻った。――もう一度。」**（無罰叙述）。

### 2.3 「翳り（かげり）」と社会
- いま大陸を蝕む現象 **「翳り（かげり）」**。土地から灯詞が抜け、物が応えなくなり、**人が言葉を忘れていく**。
  翳りに呑まれた町は灰色になり、住人は固有名詞から順に失語する。
- 灯詞は財産。位階の高い語（L3以上）を平民が灯すことは、都の **「灯札（あかりふだ）」法** で制限される。
  都と学び舎が免状で高位語を独占する。例外は **灯詠みギルド（流れ灯詠みの互助団）** ——
  「**確かに思い出せた数**」だけが階級になる実力主義。主人公が成り上がれる理由＝章ゲート（昇格）の正当化。
- 翳りの元凶＝後述の **トワ（集める翁）**。世界一の灯詠みだったが、救えなかった一人のために
  **「もう二度と失わないため」全ての灯詞を集め始めた**。集積が世界を翳らせている（§3キャラ表）。

### 2.4 8分野＝8系統（全782語の `f` 属性と1:1対応。システム＝不変）
| 分野(データ) | 系統 | 役割 |
|---|---|---|
| nature | 灯系 | 攻撃（sun, shine, wind, star…） |
| daily | 暮らし系 | 基礎・便利（wake, door, kitchen…） |
| food | 恵み系 | 回復（water, bread, drink…） |
| feelings | 心系 | 強化・弱体・そして絆（hope, trust, friendship…） |
| school | 識系 | 鑑定・解析（read, study, knowledge…） |
| travel | 道系 | 移動・脱出（arrive, walk, map…） |
| business | 約定系 | 取引・開錠（open, money, sign…） |
| society | 律系 | 結界・法・命令（protect, rule, people…）。護りの基礎語と人を縛る上位語が同居 |

### 2.5 位階と原文（章扉の灯詞修復率）
- 灯詞は位階一〜五（L1〜L5）。深い語を根づかせるほど深い土地が開く。
- **世界の原文は782語で綴られている**。章扉に原文断片が1行。黄金（S≥30）以上にした語は英語表記、
  未到達語は「◆◆」。**章扉8行のみの限定実装**（全脚本タグ付けはしない）。修復率＝契約語数/782を併記。
  終章で全行が英語で読める＝**学習の最終目標と物語の最終目標が同一**（段階開示の一括回収）。

### 4.2 章ごとの要点（感動設計込み）
- **1章「灯が消えた夜」**: 夜祭の晩、村の**大灯（おおあかり）**が消え、翳りが柵を越える。人生初の灯＝`sun`。
  村人が言葉を抜かれる（失語）。ユイと共に村を追われる起点。章末の小さな喪失＝**老灯守の失語は戻らない**
  （完全勝利にしない予行）。詳細は `ch1.md`。
- **2章「追われて、街道」**: 行く当てのない二人。ガク（灯詠み不能の好漢）と出会い共闘。灰の手先ドゥドの
  くすんだ灯詞で法則4を見せる。章末の小さな喪失＝助けた隊商の一人が、もう仲間の名を呼べない。
- **3章「灯札の街」**: ギルドにF級登録。灯札法で上位語の露店灯しを咎められる。カガリ（兄貴分）が飯を奢り、
  戦闘の語り部に。ガクと再会・共闘。**カンテラの声が初めてまとまった文を喋る**（謎の本格化）。
- **4章「禁書の灯」**: 灯詠み資格試験。深夜の禁書庫で封じられた灯詞に触れ、**カンテラの声の正体に最初の手触り**。
  この章でユイの**最初の小さな違和感**（言いかけた言葉が出てこない一行）を置く＝喪失の予兆の散布開始。
- **5章「声のない町」**: 試験で向かった町が丸ごと喋れない（ホラー回）。翳りの正体の断片を声が囁く。
  **章末＝真実を都に報告するか握り潰すか（`truth`）**。
- **6章「帰れない村」**: 戻ったナギ村は灰。**全ルート完全共通章（差分ゼロの休憩区間）**。ここでユイの忘却が
  はっきり進む（村の思い出の固有名詞から抜けていく）。撃退後、ユイの「弱音」が初めて漏れる。
  モチーフ**「大灯」**が灰の中で**消えたまま**＝中盤の“灯が消えかける”底（終章の再点灯が効く）。
- **7章「都の取引」**: 王家が翳りを兵器転用しようとする。**カンテラの声が初めて“お願い”をする**
  （「灰都の底に、あたしの欠片がある。取りに行こう」）。**章末＝光／闇の分岐**（§5）。
- **8章「灰都へ」**: トワの正体・祖父との因縁・`lonely`の手触りが開示。**カガリが一行を庇い、灯詞の半分を失う**
  （`truth`報告済なら一命確定／隠蔽なら生死不明のまま終章へ）。ボス＝灰の門番。
- **終章「きみの名前で」**: トワ戦。出題は全習得プール＋期日優先、語り部級ほど高威力。終盤は**「途切れた灯を継ぐ」**
  形式＝トワの灯詞が一語だけ灰に呑まれて途切れ、正しい一語をタイルで埋め返す（既存ex/jxのクローズ形式。
  **「穴埋め」とは絶対に呼ばない**＝テスト臭対策）。**最大の喪失＝この直前、ユイがアキの名前を忘れる**。
  とどめ＝最高Stability語の自動選択＝**プレイヤーが30日かけて最も根づかせた言葉**（代価＝積んだ30日そのもの）。

- 旧canonの `dark`/`ceci` は廃止（恋愛多ルートを畳み、**幼馴染ユイを感情の核に一本化**＝seed準拠で愛着を集中）。

## 7. 物語全体の起伏（泣きゲー文法／earned-catharsis 5機構の適用）

`docs/nakige-emotional-design.md` のテンプレ（開幕の緩を厚く→小さな喪失の分散→モチーフの意味変容回帰→
完全勝利にしないほろ苦）を全体に通した。

| 機構 | 本作での設計 |
|---|---|
| **①振幅制御** | 開幕の緩＝ナギ村の夜祭・ユイとの軽口を**厚く**置く（1章）。**小さな喪失を各章末に分散**（1章=老灯守の失語が戻らない／2章=隊商の一人／…）。最大の喪失＝**終章直前、ユイがアキの名を忘れる**まで取っておく。結末先出し＝カンテラの声が序章から「灯が消えるのはきみだけど」と零す→以後の軽口すべてが予兆に変わる（裏テーマは言語化しない範囲で） |
| **②モチーフ反復（意味変容）** | §8の表。**回帰の方向を予測不能に**して“あざとさ”を回避 |
| **③段階開示** | 章扉の原文断片＋修復率で可視化。終章で全行が英語で読める＝一括回収。撒いた違和感（カンテラの声／トワ／`lonely`／祖父）は§9の表で全回収を担保 |
| **④奇跡の正当化** | 「溜め込んだ言葉は灰になる」を序盤から事前提示（敵描写のみ）。終章の勝利＝**最高Stability語の自動選択＝積んだ30日そのもの**が代価（無償の大団円にしない）。カガリの声半失＝救済に代価。隠しEDの`lonely→friend`＝対句反転 |
| **⑤愛着の入口** | アキの軽口の一人称（共犯化）＋ユイの**毎日のさしいれ／鐘**（present相棒）。終章でユイが名を忘れる直後、**アキの地の文から軽口が一度だけ消える**（説明せず文体で見せる） |

**毎日の接触を“賑やかな日常の本体”にする**（シーン予算を増やさず愛着を積む／nakige.md 壊れる条件＝尺不足対策）:
- **ユイのさしいれ（鐘配信）**: 1通2〜3行。30日×複数通＝実シーンより遥かに多い接触。口癖・好物・弱音の予兆を反復刷り込み。
  序盤＝軽口（「アキ、また無茶したでしょ」）→中盤＝小さな違和感（「……べつに。なんでもない」）→6章で初めての「助けて」。
- **鐘（朝昼夜）に感情の時刻**: 夜の便りだけ一段トーンを落とす等、生活リズムに同期した微振幅をシーンを使わず生む。

## 11. 文体規則（新canonでも不変）

1. **1シーン＝2〜8行＋選択肢**。8行上限は絶対。「続きを読む」はプレイヤー起点タップ。全文ログ残留。
2. **チュートリアル文ゼロ**。システムは住人の台詞と行動で教える（各装置の言語化は作中一回まで）。
3. **説教禁止**。学習価値の講釈は書かない。法則（溜め込んだ言葉は灰になる等）は敵の描写と悲劇でのみ。
4. **裏テーマ（交わさない言葉は失われる）は作中で絶対に言語化しない**（本書の設計メモ限り）。
5. 選択肢は常に2〜3択の**温度違い**（茶化す／意地を張る／黙る）。どれでも直後の同一シーンに合流、差分はセリフ1〜3行。
6. **PG表現**（手を繋ぐまで）。直球の告白を避け、軽口・さしいれ・沈黙で描く。デート専用回を作らない。
7. **灯詞は必ず `data/words.js` 実在語**（`w` キーで検証）。固有名詞（灰狼・カンテラ等）は対象外。
8. キャラ声の固定（§3）。
9. 固有名詞は全て `terms.json` に data 化し、15歳テストで「ダサい」と言われたら即改名できる構造にする。
