# A Dark Room 分析メモ — 極小テキストの叙述力
対象: Doublespeak Games『A Dark Room』(2013年ブラウザ版、ソースコード実測値含む)/ 用途: 英単語想起ゲーム企画の設計インプット
前提: vault既存分析(idle-incremental.md)のドーパミンループ・セッション構造・SRS接続の一般論は既知とし、本メモは**テキスト叙述に固有の機構**とその差分のみ扱う。

---

## 1. テキストだけで面白さが成立する仕組みの分解

**(a) 動詞の経済 — UIそのものが語彙である**
画面の全要素がボタン=動詞(「火を焚べる」「木を集める」「罠を見る」)と名詞リスト(在庫)。開始時の動詞は**1個**。動詞が増える=「できること」が増える=世界が広がる、が完全に等価。グラフィックゲームでは「新エリア」「新スキル」が別レイヤーだが、ADRでは**新しい言葉の出現自体がコンテンツ追加**。Townsendは旧パーサーADVの「どの動詞が通じるか分からない」探索感を意図的に再現したと語っている。プレイヤーは説明されずに新ボタンを押す——押す前の「これは何だ?」が報酬の前借り(期待形成)になる。

**(b) 名詞の置換による無言の語り(本作最大の発明)**
状態変化を説明文でなく**ラベルの書き換え**で語る:
- タブ名: 「A Dark Room」→火を点けると「A Firelit Room」
- 村の見出し: a silent forest → a lonely hut(小屋1)→ a tiny village(2–4)→ a modest village(5–8)→ a large village(9–14)→ a raucous village(15+)
- そして決定打: 人口管理画面の「workers」がある時点で無言で**「slaves」**に変わる。説明ゼロ。プレイヤーは自分が作った経済の正体を、UIの単語1個の置換で突きつけられる(PopMattersの複犯性分析の核)。
1語の置換が数千字の説明に勝つ。テキストゲームでは**語の選択=演出予算のすべて**。

**(c) 情報の負空間 — 待ち時間が想像の余白になる**
builderは何者か、なぜ世界は滅んだか、一切説明しない。Wikipediaにある通り「環境的手がかりのみで物語る」設計。重要なのは、idleゲーム特有の**待ち時間が「考察する時間」として転用されている**こと。開発者自身が「遅いペーシングが、開示と開示の間にプレイヤーが脳内に世界モデルを構築する時間を与える」と説明している。次の開示はそのモデルを壊すために置かれる(生存サバイバル→中世的隷属→ポストアポカリプス→征服された星の残骸、というzoom-out)。**普通のidleの待ち時間は空白、ADRの待ち時間は伏線の発酵期間**。

**(d) ジャンルシフト=章構造**
クリッカー(火)→村経営(人口配分)→ローグライク(dusty pathの61×61マップ遠征)→最後にシューティング(船での脱出)。メカニクスの変化自体が「章の変わり目」を告げる。テキスト量を増やさず「ルールが変わった」ことで物語の転調を体感させる。

**(e) 正体の遅延開示による全行動の再解釈**
終盤、主人公=異星から来て星々を滅ぼす種族「wanderer」だと判明。それまでの奴隷化・虐殺・資源収奪が一斉に再解釈される。テキスト総量は極小でも、**既に起きたことの意味を書き換える**ことで物語の質量を稼ぐ。安いテキストで高い物語密度を出す技術の教科書。

**(f) 声の間欠投下**
ランダムイベント(物乞い、盗賊、謎の修道士など)は実装上**3〜6分に1回**の抽選。これがゲームの「声」の全てで、沈黙のベース進行に対する句読点になっている。テキストが常時流れないからこそ1イベントが読まれる。

## 2. アンロックと数字成長のリズム(実測値ベース)

ソースコード(GitHub公開)からの正確な数値:

| 時間軸 | 内容 |
|---|---|
| **10秒** | 火を焚べるクールダウン。労働者の生産tickも10秒(builderは木2/10秒) |
| **30秒** | 室温更新・builder状態進行のtick。点火→「窓から光が漏れる」→「ぼろをまとった見知らぬ者が倒れ込む」→回復、の各ビートが約30秒刻み |
| **60秒 / 90秒** | 木を集める60秒、罠を見る90秒。序盤の手動ループの心拍 |
| **0:00–3分** | ボタン1個→火→builder到着→「木が尽きた」(15秒遅延)で森=2画面目がアンロック。**最初の3分で動詞が1→4個、画面が1→2枚に** |
| **5分** | 火が1段階冷える(roaring→dead は放置約20分)。世話の義務サイクル |
| **0.5–3分ごと** | 小屋があれば人口+1〜(小屋1軒=4人、最大20軒80人)。「夜、見知らぬ者が現れる」系の到着テキストが流れる |
| **3–6分ごと** | ランダムイベント1回。セッション中、数分に1度は必ず「読む」瞬間が来る |
| **30–60分** | 交易所→コンパス購入→**dusty path解禁(第1のジャンルシフト)**。ここまでが「経営の学習」フェーズ |
| **1–3時間** | 遠征反復。水1/歩・肉1/2歩・戦闘20%/歩が遠征距離を制限し、「もう一歩先へ」のために村側の生産強化へ戻る往復構造。鉄→石炭→硫黄鉱山(半径5/10/20)が同心円状に配置され、**地図の半径=進行度** |
| **約4.5時間** | 船(半径28=ほぼ外縁)発見→離陸→ループしてタイトルへ。コミュニティ計測の総プレイは約4時間半 |

設計上の要点: **アンロック密度が前半に極端に偏っている**。最初の10分で世界が2倍に2回なる一方、後半は1つの大型開示(遠征、船)を1時間単位で引っ張る。序盤=分単位の構造的報酬、中盤=10分単位、終盤=時間単位、という対数的な間延びを、物語の謎の深化で埋めている。また**オフライン進行が無い**(タブを閉じれば止まり、開いていれば火が消える)点は現代idleと真逆で、後述の再訪構造に直結する。

## 3. プレイヤーが戻ってくる理由の構造

vaultで整理した「開けば回収」型の再訪(オフライン収益)をADRは**持っていない**。代わりに3層:

1. **世話のフィクション(タマゴッチ型)**: 火は5分で1段階弱まり、消える。村は襲撃される。「貯まるから開く」ではなく**「死ぬから戻る」**。義務が物語(寒い世界で火を守る)と完全に一体化しているため、ペナルティなのに嫌われない。
2. **物語の未解決(クリフハンガー型)**: 再訪動機の主燃料は数字でなく「builderは何者か」「この骨は何か」という謎。数値目標(あと木200で小屋)は短期、謎は長期のフックとして二重化されている。
3. **共犯の感情負債**: slaves表記、builderが去り際にロケットを残して消える等、プレイヤーに「自分が選んだ」罪悪感を蓄積させる。感情的に清算されていない状態は離脱しにくい。

ただし総量4.5時間で物語は尽き、ループ構造(墜落して最初に戻る)はあるが実プレイのリピートは弱い。**ADRの再訪設計は「短期決戦の没入維持」であり、月単位のリテンション装置ではない**。ここがvaultで扱った量産型idleとの最大の構造差。

## 4. 「英単語の想起(4択)×忘却曲線」との相性(ADR固有の差分のみ)

### 良い部分
1. **動詞アンロック=語彙獲得のメタファーが直訳できる**。「単語を覚える→できること(動詞ボタン)が増える→世界が広がる」は、ADRの核とテーマレベルで同型。「言葉を知ることが力になる」ゲームを言葉の学習アプリで作る、という整合は他ジャンルでは得られない。
2. **アクション粒度の一致**。stoke 10秒/gather 60秒のリズムは4択想起(1問5〜10秒)とほぼ同テンポ。「想起=薪をくべる」「火の減衰(5分)=短期忘却」の置換が無理なく成立し、忘却曲線をチュートリアル無しで体感則にできる。
3. **開示密度の流用**。「3〜6分に1イベント」≒「10〜30問に1度の物語片」。想起の単調な連打に対し、低頻度・高価値のテキスト報酬を句読点として打つ較正値がそのまま使える。
4. **制作コスト構造の一致**。テキストオンリーなのでアセット制作が要らず、個人〜少人数開発の学習アプリと同じ土俵。名詞置換(workers→slaves型)の演出は実装コストほぼゼロで感情を動かせる。
5. **負空間による説明コスト削減**。チュートリアルを書かず「新しいボタンを押させて学ばせる」方式は、学習アプリで嫌われる説明過多UIの解毒剤になる。

### 悪い部分
1. **燃料の寿命が違う**。ADRの物語は4.5時間の使い切り。SRSは数ヶ月〜年。手書きの物語開示をエンジンにすると、**テキストが尽きた日=引退日**になる。物語は序盤の点火装置と中間マイルストーンに限定し、長期は数字・乗数系(vault設計)に引き継がせる必要がある。
2. **損失駆動の緊張が学習不安に転化する**。ADRは「火が消える」「遠征で死ぬ」損失で駆動するが、誤答→火が消える、を直結させると想起が罰のトリガーになり学習動機を毀損する(vault指摘の矛盾の最悪形)。火の減衰は**時間経過のみ**に紐づけ、想起は常に「火を強くする」側に置くこと。誤答は薪が小さいだけ、にする。
3. **連続没入前提のペーシング**。ADRはオフライン進行なし・開きっぱなし前提で4.5時間を設計している。分散学習(1日数回×3分)に移植するなら、開示の単位を「経過時間」でなく**「セッション数・正解数」基準**に組み直す必要がある。そのまま移植すると初日に物語を食い尽くすか、逆に火が消え続けて萎える。
4. **終盤ジャンルシフトの認知コスト**。ローグライク遠征のような高負荷ミニゲームは想起と認知資源を奪い合う。ADRの「章=ルール変更」は、学習アプリでは「章=出題形式や報酬構造の小変化」程度に減衰させるべき。
5. **4択は受動認知、ADRの快感は能動探索**。4択の単調さ自体をADR構造は救わない。救うのは4択の**あと**——正解が何を解錠し、どの見出しを書き換えるか、という意味付けの層である。

## 5. 盗むべき設計1つ / やってはいけない失敗1つ

**盗むべき1つ: 「最初の画面はボタン1個」から始まる動詞アンロック+見出し書き換えの無言進行表現。**
初回起動はチュートリアルも説明もなく、押せるものが1つだけ(例:「思い出す」)。正解の蓄積で動詞・画面・名詞が増殖し、拠点や称号のラベルが習得語数に応じて無言で書き換わる(a dark room→a firelit room 方式)。数字バーでなく**言葉の変化で成長を語る**。単語学習アプリとのテーマ整合が完璧で、実装コストが最も低く、かつ「次は何が現れる?」という構造的好奇心(vaultのルール増殖論)をテキストだけで実現できる。アンロック密度は ADR に倣い**最初の3分に集中投下**(3分で動詞1→4)。

**やってはいけない失敗1つ: 手書き物語を進行の主燃料にしたまま尽きさせる(+終盤の別ゲー接ぎ木)。**
ADR自身の弱点がこれ。物語が尽きた瞬間にリプレイ価値がほぼ消え、4.5時間で関係が終わる。さらにTownsendは「飽きたからリリースした」と認めており、終盤の宇宙パートは唐突な操作系シフトで蓄積資産を無意味化し、最も批判された。学習アプリは数ヶ月走らせる前提なので、**物語開示は有限のスパイス、無限の背骨は想起×乗数成長**という主従を最初から固定すること。終盤に高負荷の別メカニクスを足して「単語を思い出すゲーム」から逸脱させないこと。
