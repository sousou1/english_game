# Melvor Idle 深掘り分析メモ(スキルUI/目標可視化/オフライン精算)

対象バージョン: v1.3.1(2024-10-30)。既存のジャンル分析(idle-incremental.md)との重複を避け、Melvor固有の実装・実数値・2024年以降の評判に絞る。

---

## 0. 前提: Melvorの骨格を3行で

20超のスキル(伐採・採掘・調理・戦闘…)を持つが、**同時に進行できるアクションは常に1つだけ**。プレイヤーの仕事は「次の24時間、何を回すか」の選択であり、ゲームの実体は機会費用のパズル。グラフィックはアイコンと数値とプログレスバーのみで、3Dどころかアニメーションすらほぼない。

---

## 1. テキストだけで面白さが成立する仕組みの分解

**(a) 「1アクション制約」が画面を戦略に変える。** Cookie Clicker系は全生産が並列で走るため数値は風景になるが、Melvorは1つしか動かせないため、スキル一覧UIそのものが「次の一手を選ぶ盤面」になる。テキストUIで成立するのは描画力が不要だからではなく、**意思決定の密度が視覚情報を代替している**から。

**(b) 目標が全部「固有名詞」である。** 「生産+5%」のような抽象目標ではなく「Lv10でオークの木」「Lv60でイチイの木+ルーンの斧」と、レベル表に具体名詞が並ぶ(Milestones画面で全段階を最初から閲覧可能)。テキストゲーの報酬は名詞の魅力で決まる。RuneScape由来の名詞群(ドラゴン装備、魔法の木)が事前知識なしでも「強そう」の序列を直感させる。

**(c) 進捗バーの多層化。** 1画面に常時5種類以上のバーが見える: ①現在アクションの進行バー(3〜40秒で1周)、②スキルXPバー、③アイテム別マスタリーバー、④マスタリープール(チェックポイント付き)、⑤装備/ショップの次の購入。どのバーかが常に「あと少し」。これを全部テキスト+矩形で実現している。

**(d) 数値の信頼性。** オフラインもオンラインも完全に同一の計算(後述)。「この数値は嘘をつかない」という信頼が、スプレッドシート的UIを「読む快楽」に変えている。Steamレビューでも「ミニマルだが情報リッチなUI」が一貫して長所として挙がる。

**(e) ドロップテーブルのテキスト的スロット。** 伐採中に0.5%で鳥の巣、レアでペット。文字列のログに突然レア名詞が混ざる瞬間が可変報酬として機能。演出ゼロでもアイテム名+確率だけでガチャの興奮を作れる実証例。

## 2. アンロックと数字成長のリズム(実数値、Woodcutting基準)

XPテーブルはRuneScape型指数(約7レベルごとに必要XPが倍、**Lv1→92のXP合計 = Lv92→99と同量**)。木こりの基礎XP/sから逆算した無補正の所要時間:

| 到達Lv | アンロック内容 | 前の節目からの所要 |
|---|---|---|
| 10 | オークの木+鋼の斧 | **約6分** |
| 15/20 | 巻物/黒の斧(ショップ) | 各10〜20分 |
| 25 | 柳の木 | 約30分 |
| 35 | チークの木+ミスリル斧 | 約55分 |
| 45 | カエデの木 | 約2.2時間 |
| 55 | マホガニー | 約5.8時間 |
| 60 | イチイ+ルーン斧 | 約5時間 |
| 75 | 魔法の木+炎の斧巻物 | **約39時間** |
| 90 | レッドウッド | 100時間超(装備補正で半減) |
| 99 | スキルケープ(称号装備) | さらに実質4〜7日の放置 |

リズムの設計則:
- **最初の1時間に名詞付きアンロックが4〜5個**(6分→20分→40分→60分)。
- 中盤は2〜6時間間隔=「1日2〜3回開けば毎回何か起きる」帯。
- 終盤は日〜週単位=「寝る前にセットして朝回収」帯に移行。
- ここで重要なのが**横展開によるリセット**: 1スキルの報酬間隔が日単位に伸びた頃、別スキルはまだ「6分で初アンロック」帯にある。20スキルあるので序盤の快感曲線を20回再生できる。縦のインフレではなく横の本数でドーパミン密度を維持するのがMelvorの解。
- さらにアイテム別マスタリー(各木にLv1-99)が第2層として走る。**マスタリー10レベルごとに「+5%二倍ドロップ」等の固有ボーナス**、マスタリーXPは取得式の構造上「育てた項目ほど加速」する(取得量が項目マスタリーLvと総マスタリーに比例)。スキルXPの間隔が伸びても、マスタリー側の節目が分〜時間単位で挟まり続ける。
- プール・チェックポイント(10/25/50/95%)がスキル単位の中期目標。例: 木こりは50%で「丸太売却金+50%」。

## 3. プレイヤーが戻ってくる理由の構造

**(a) オフライン上限24時間 = 日次リズムの強制発振器。** 24時間を超えた分は消失するため「1日1回は開く」が損失回避で固定される。これが復習アプリのDaily Streakと同じ周期を、義務感でなく回収欲で作る。

**(b) 精算画面が「セッションの初手=報酬」を保証する。** Welcome Backポップアップに不在時間、スキルXP/マスタリーXP、入手アイテム一覧、消費(食料・ルーン・矢・祈りポイント・召喚タブレット)、討伐数、スレイヤータスク完了数が列挙される。テキストの羅列だけだが、**完全シミュレーション(オンラインと1tickも違わない)**であることが知られているため、数値の羅列が「不在中も世界が動いていた」証拠として機能する。擬似計算や広告2倍ブーストの類を一切入れていない。

**(c) 出発前の「仕込み」ゲーム。** オフライン戦闘・盗みは死亡リスクがあり、食料の量と成功率を確認してから閉じる必要がある。「閉じる前の装備チェック」という能動行為がセッションの締めの儀式になり、回収時の結果(生きてた/食料が尽きてた)に物語性が生まれる。閉じる行為自体がプレイになっている稀有な例。

**(d) 次の精算予告が常に具体的。** 「魔法の木は20秒/100XP」のように全数値が公開されているため、プレイヤーは閉じる時点で「明日の朝には約4,320回伐採、Lv76到達、巣が約21個」と暗算できる。コミュニティで「Melvor ETA」(レベル到達までの残り時間を表示するMod)が定番化している事実は、**「あとXP」より「あと何時間」の表示が本質的需要**であることの証拠。本家は距離をXPでしか出さないが、新規設計なら時間表示をネイティブに持つべき。

**(e) 完成欲のメタ層。** Completion Log(全体達成率%)、スキルケープ(Lv99の称号)、ペット(超低確率)、全スキル99のMaximum Cape。一枚岩の「100%」がある買い切りゲームとして、終わりが見えることが逆に長期動機になっている。

## 4. 「英単語の想起(4択)×忘却曲線」との構造的相性

(ジャンル一般論は既存メモ §4 にあるので、Melvor固有構造に限定する)

### 相性が良い部分

1. **アイテム別マスタリー = 単語別の記憶強度、の直訳が可能。** Melvorの「スキルLv(広さ)×アイテムマスタリーLv(深さ)」の2層は、「語彙帯の進行度 × 単語ごとの定着度」にそのまま写像できる。単語タイルにマスタリーバーを付け、10レベルごとに固有ボーナス(その単語が生む資源+5%等)を置く設計は実装コストも低い。
2. **マスタリープール=復習クレジット。** 「全単語の想起が共通プールに25%還流し、任意の単語のテコ入れに使える」構造は、苦手単語への重点配分をゲーム内資源操作として表現できる。チェックポイント(プール10/25/50/95%で帯全体のボーナス)は「この語彙帯は全体的に仕上がっている」の可視化になる。
3. **アクション粒度の一致。** Melvorの1アクション3〜40秒は4択1問(5〜10秒)とほぼ同じ粒度。さらにMelvorは「時間が長いアクションほどXPが大きい」原則を持つ。これは「難しい単語(長考・低正答率)ほど高報酬」にそのまま転用でき、易問高速周回の搾取を式レベルで防げる。
4. **24時間キャップ=SRSの日次レビューと同周期。** 精算画面を「不在中に熟した復習キュー」の提示に流用すれば、「回収=今日の復習開始」が一体化する。
5. **固有名詞ラダー(Milestones)。** 「Lv25で柳」を「Lv25で〈金属加工語彙100語〉が開く」に置換すれば、語彙学習の見えにくい遠景に名詞付きの里程標を立てられる。

### 相性が悪い部分

1. **Melvorのマスタリーは絶対に減衰しない。** 忘却曲線は減衰そのものであり、Melvor文法(損失ゼロ)と正面衝突する。単語マスタリーを時間減衰させると「Melvorの快感の核=積み上げの不可逆性」を壊す。→ 減衰させるのは単語側でなく**産出レート側**(忘却間際の単語は「収率が落ちる」が、想起1回で全回復)にする等の翻訳が必要。
2. **マスタリーの上げ方が「同一アクションの数千回反復」。** 同じ単語を1000回連打しても学習効果はないため、回数ベースのマスタリーXPは流用不可。間隔を空けた成功想起のみマスタリーXPが入る(=SRSスケジューラがXP式の中に入る)よう式を改造する必要がある。
3. **「1アクション放置」の最適解が学習と逆。** Melvorの正解は「最高効率の1つを選んで放置」だが、語学の正解は「広く分散して想起」。放置枠に置けるのは想起ではなく加工・変換工程のみ(既存メモの「想起=採掘、放置=精錬」と同結論だが、Melvorの1アクション制約UIは『今日はどの語彙帯を精錬するか』の選択画面として優秀)。
4. **4択は偶然正解25%を含む。** Melvorは行為=成果が保証された世界で、その確実性が数値の信頼を支える。当て推量の正解に満額報酬を出すと数値の意味が腐る。連続正解・応答速度で報酬を変調し「まぐれ」を式で減価する必要がある。

## 5. 盗むべき設計1つ / やってはいけない失敗1つ

**盗む: 「ロックされた目標を最初から全部、固有名詞と必要条件つきで見せる」二層可視化。** スキル画面ではLv90のレッドウッドがLv1の時点からグレーアウトで見えており、Milestones画面に全アンロックの一覧表、サイドバーに全スキルの「現在Lv/99」が常駐する。距離(あとXLv)と報酬(具体名詞)が常にワンセット。これに加え各アイテムのマスタリーバーが「全タップが必ずどこかのバーを動かす」状態を保証する。英単語ゲームでは: 単語1語ずつにマスタリーバー、語彙帯ごとに固有名詞の里程標一覧、ホーム画面に全帯のLv常駐、を最初の実装スプリントから入れる価値がある。なお距離表示は本家の弱点(XP表示のみ)を改良し、**残り時間/残り問題数**で出すこと(ETA Modの人気が需要の証拠)。

**やってはいけない: Township型の「文法違反サブシステム」。** Melvor最大の失策はTownship(街運営スキル)のリワークで、①稼いだ金を吸うだけで見返りのループが閉じない、②既存プレイヤーの蓄積を強制変換で毀損、③エンドゲームでは産出物(XP消費財)が無価値、という三重の契約違反でSteam直近レビューが92%→84%に急落した。教訓は「全アクションが可視の見返りを持つ」という自ゲームの核契約を、後付けシステムにも例外なく適用すること。英単語ゲームなら: 学習で稼いだ資源を消費するだけの経営ミニゲーム、既習データの仕様変更による価値の書き換え、学習が完了した帯で無意味になる報酬、の3つを設計レビューの禁止項目にする。
