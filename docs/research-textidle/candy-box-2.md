# Candy Box 2 分析メモ

# 分析メモ: Candy Box 2 — ASCIIアートと隠し要素による好奇心駆動設計

対象: Candy Box 2(aniwey、2013年10月、ブラウザ/TypeScript、GPLv3公開)
位置づけ: 既存メモ(idle-incremental.md)が扱う「数値成長×報酬スケジュール」型とは別系統。**CB2の牽引力は数字ではなく好奇心**であり、本メモはその機構を分解する。既存メモとの重複(ドーパミンループ一般論、セッション構造一般論)は省く。

---

## 1. テキストだけで面白さが成立する仕組みの分解

**(a) 極小の開幕 = 期待値の人為的な底下げ**
初期画面は「You have 1 candy」とボタン1〜2個のみ。グラフィックゼロ、説明ゼロ。この「何もなさ」が以後のすべての発見の倍率装置になる。期待値が地を這っているため、タブが1個増えるだけで報酬として成立する。リッチなUIで始めるゲームはこの倍率を最初に捨てている。

**(b) ASCIIアートの二重機能: 描画であり、同時にUIである**
CB2の核心的発明。村の家のASCII絵の中の特定の文字(茂みの中の1文字、壁の刻印など)が実はクリック可能で、アイテムが隠れている。つまり**画面上の全テキストが「読むもの」かつ「触るもの」**になり、プレイヤーは絵を「鑑賞」ではなく「捜索」する読み方を学習する。ビットマップ絵では「どこがインタラクティブか」はUI規約(ボタン形状)で示すしかないが、文字の均質な海では規約が存在しないため、全てが疑わしくなる。この「全面が疑わしい」状態こそが探索の快楽の源泉。

**(c) 数字カウンタ自体が演出装置**
42個で画面に小さなキャラが一瞬現れる、1337個で表示が「leet candies」に変わる、捨てたキャンディが累計580個で画面上の人物がメッセージ末尾の記号を破壊する——**プレーンな数値表示そのものがイベントの舞台**。プレイヤーは「数字を見ている」のではなく「数字を見張っている」。インクリメンタルの最も退屈な要素(カウンタの凝視)を可変報酬スロットに転用している。

**(d) UIの増殖 = ルールの増殖の可視化**
新メカニクスは新タブ/新ボタンとして物理的に画面に増える。「ゲームが大きくなっていく」ことが視覚的事実として積み上がる。既存メモの「ルール自体の増殖」(§3c)のテキスト版だが、CB2はこれを開始60秒から毎分単位で行う点が異なる。

**(e) 制作コストの低さが投入頻度を買っている**
作者aniwey本人が明言:「グラフィックが苦手」「ASCIIが好き」「機能追加のたびに絵を直さなくて済むのでゲームプレイに集中できる」。テキストの限界費用ゼロが、「驚きの密度」という贅沢(灯台のパズル、釣り、醸造、横スクロール自動戦闘クエスト、商人、謎かけリス…全部ミニ機構)を個人開発で可能にした。**驚き駆動のゲームは驚きの単価が勝負**であり、ASCIIはその単価を一桁下げる。

**(f) 第四の壁までコンテンツ**
HTMLソース冒頭に「Hey, you're looking at the source code, cheater! ;)」、ソース末尾にASCIIキャンディ、最終ボスは「The Developer(開発者本人)」。**ゲームの外側(ブラウザ、ソース、タブ)まで遊び場**だと宣言することで、「どこまでが仕込み範囲か分からない」=探索範囲の心理的な無限化に成功している。

## 2. アンロックと数字成長のリズム(実時間)

基礎生産は1キャンディ/秒・固定(タブを開いている間のみ。オフライン進行なし——CB2は厳密には「放置ゲー」より「待ちゲート付きアドベンチャー」)。

| 時間帯 | 起きること | 数字の規模 |
|---|---|---|
| 0〜60秒 | キャンディ10個で「投げる」、30で状態バー、40でセーブ、50〜60でHPとマップ系UI。**最初の1分でUI要素が3〜4個増殖** | 数十 |
| 1〜10分 | 村の発見。店・鍛冶屋(木の剣150個=約2.5分)・家々(クリック秘密)・地下室クエスト(ネズミ退治、1〜2分の自動戦闘) | 数百 |
| 10分〜1時間 | ワールドマップ解放。砂漠・橋のトロル・洞窟・リスの謎かけ(6問)・ロリポップ農場開設。**ここで成長が2レーン化**: 放置の1/秒+農場の複利(風車に餌やり: 21キャンディで1ロリポップ/秒→23,046で100/秒、フカヒレ×5・貝殻粉×3等の乗数) | 数千〜数万 |
| 1〜5時間 | 大型装備(2,000〜15,000)、醸造(大釜)、灯台のサイクロプスのパズル、ポゴスティックで穴の底へ、海への潜水(無限スクロールクエスト、深いほど強敵・高額ドロップ=**能動周回レーン**) | 数万〜数十万 |
| 1〜2日(実質5〜10時間) | 大鎌5,000,000個(放置1/秒では58日かかる=クエスト周回か農場複利が必須という無言の教育)、地獄、The Developer戦、キャンディ箱の開封=エンディング。クリア後にゲーム内コンソール(チートコマンド)解禁 | 数百万 |

**リズムの要点:**
- 序盤の閾値間隔は30秒〜3分。**「次の何か」が常に3分以内**にあり、しかもそれが数値増加でなく「新しい種類の何か」である確率が高い。
- アンロックの紐付け先が価格表ではなく**「地理(マップ上の場所)」**。次の目標が「あと N 個」ではなく「あの行ってない場所」として見える。空間メタファーは残量計算を曖昧にし、過剰な最適化を防ぐ。
- 知識があれば30〜40分で走破可能(スピードラン)、初見は5〜10時間を2〜4日に分割。**分割の原因は放置ゲートではなく「行き詰まり→翌日ふと試したら開く」という謎解きの消化サイクル**である点が普通の放置ゲーと決定的に違う。

## 3. プレイヤーが戻ってくる理由の構造

Cookie Clicker系の再訪動機は「数字が貯まったから回収」。CB2は**「未消化の好奇心残高」**で戻らせる。残高を作る装置は4つ:

1. **前科による全面的猜疑心**: 序盤10分で最低2回「まさかそれが触れるとは」(家の絵の中の文字、数字カウンタのイベント)を経験させる。以後プレイヤーは全画面を疑い続け、**「見つけていない何かが必ずある」が信念ではなく実績に基づく確信になる**。これが「まだ何かある」感の正体。演出ではなく、裏切りの履歴。
2. **可視の未解決物**: 地図上の入れない場所、用途不明のアイテム(ポゴスティック入手→「どこで使う?」)、明らかに何かありそうな X 印。**謎は隠すが、謎の存在は隠さない**。「未知の未知」ではなく「既知の未知」を画面に常駐させる。ツァイガルニク効果の好奇心版。
3. **行き詰まりの非対称性**: 詰まりの解消手段が「キャンディを待つ」(時間が解決)と「何かを試す」(発想が解決)の2系統あり、どちらかが常に開いている。待ちゲートは再訪の**口実**、未解決の謎は再訪の**動機**——役割が分離している。
4. **外部コミュニティが「全部は見ていない」を証明する**: wiki・攻略の存在自体が「自分の知らない秘密がある」ことの証拠になる。秘密ベースのゲームはネタバレ媒体と共生関係にある。

**弱点(重要)**: この構造は**一回性**。CB2にはプレステージも周回もなく、秘密は一度見たら消費済み。同時代の批評(Medium/Fasce)はCB2自身が前作比で既にこの問題を露呈していたと指摘——前作は「飽きる寸前に新要素」の時限投入が完璧だったが、CB2のオープンワールド化は発見の密度を薄め「割と早く興味が切れる」。**好奇心駆動は密度が落ちた瞬間に死ぬ**。数字駆動より高出力だが燃費が悪い。

## 4. 「英単語の想起(4択)×忘却曲線」との構造的相性

### 相性が良い部分
1. **テキストが世界そのものという思想**。英単語ゲームの素材は文字列。CB2の「文字が絵であり、UIであり、秘密の容器」という設計は、**単語自体をマップの部品・クリック可能な秘密・カウンタイベントの素材にできる**ことを意味する。単語リストとゲーム画面を分離せず、語が世界に埋まっている状態を作れる。
2. **「見えているのに気づかない→働きかけて初めて開く」は想起の構造そのもの**。受容語彙(見れば分かる)と発信語彙(自力で出せる)の差を、「眺めるだけの絵」と「クリックして開く秘密」の差に写像できる。
3. **1〜3分の自動戦闘クエスト=1復習セッションの粒度と一致**。CB2のクエストは装備(=事前の蓄積)で成否が決まる。これを「直前の4択セッションの成績が装備になる」に置き換えれば、想起力→戦闘力の変換が1ステップで閉じる。
4. **アンロック=地理**は語彙レベル帯と自然に対応(CEFR帯=新エリア)。「あと500語」ではなく「あの森の先」として進捗を見せられる。
5. **カウンタ節目イベント**(42、1337型)は総想起回数・連続日数カウンタにそのまま移植可能。SRSの退屈な反復回数に可変報酬を低コストで注入できる。
6. **待ちゲートと謎ゲートの2系統詰まり解消**は、SRSの「次の復習期日まで待つ」を退屈な待機ではなく「もう一つの進行レーン(探索)がある」状態に変換するヒント。

### 相性が悪い部分
1. **秘密の一回性 vs 忘却曲線の無限反復**。CB2の快楽は「二度と同じ驚きはない」こと。SRSは同じ単語への何度もの再訪を要求する。**発見を学習報酬の主軸に置くと、2回目の復習から報酬がゼロになる**。発見は新規語の初回提示にのみ使い、反復側の報酬は数値成長系(既存メモ§4)に分担させるべき。
2. **4択は総当たり可能、隠し要素クリックも総当たり可能——悪い方向で同型**。CB2の探索も後半は「とりあえず全部クリック」に堕ちる(攻略wikiを見たら終わり)。4択で「試行が無料」だと想起せずに当てるのが最適戦略化する。CB2から学べるのは逆教師: **試行コストゼロの探索は想起を殺す**。誤答に学習的意味づけ(再出題予約)を入れる既存メモの解法と併せ、選択前の数秒を強制する等の摩擦設計が要る。
3. **「待てば開く」ゲートの汚染**。CB2では待ち(キャンディ蓄積)が進行手段として正当。学習ゲームで同じ顔のゲートを置くと「学ばなくても待てば進む」というシグナルになり、想起の必然性が崩れる。待ちゲートは復習期日(待つことが学習科学的に正しい待機)にのみ偽装して使うこと。
4. **手作り秘密はスケールしない**。CB2の秘密密度は10時間ぶんの手彫り。語彙学習は数百時間で、数千語に同密度の仕込みは個人開発では不可能。ASCIIで単価を下げても、秘密は**序盤の好奇心モード形成(最初の1〜2時間)に集中投下**し、以降は数値・周回系に引き継ぐ配分が現実的。

## 5. 盗むべき設計1つ / やってはいけない失敗1つ

**盗むべき: 「最初の10分で2回裏切る」= 猜疑心のインストール**
チュートリアルで機能を説明する代わりに、「触れないはずのもの(画面の飾り文字、数字表示、メッセージ本文)が触れる/変化する」体験を開始10分以内に最低2回仕込む。以後プレイヤーは全画面を探索対象として読み、**「まだ何かある」を運営が約束しなくても自分の実績から確信し続ける**。実装は安い(クリック判定+テキスト差し替え)割に、好奇心モードという持続バフを与える——英単語ゲームなら「例文の中の1単語が実はクリックできて辞書の奥に通路がある」型で序盤に2発。

**やってはいけない: 進行の牽引を新規発見だけに依存させる**
CB2自身の弱点であり、続編批評で名指しされた失敗(「新要素の時限投入が崩れた瞬間に飽きる」「この種のゲームは初回の新規性にしか頼れない」)。反復が本体である学習ゲームで発見を主食にすると、コンテンツ消化速度>制作速度の競争に必ず負け、中盤で枯れる。**発見は香辛料(序盤の引き込みとカウンタ節目)、主食は数値成長とSRSリズム**という配分を最初に固定し、秘密の枯渇がリテンション崩壊に直結しない構造にすること。
